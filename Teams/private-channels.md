# <a name="private-channels-in-microsoft-teams"></a>Microsoft 團隊中的私人頻道

Microsoft 團隊中的私人頻道會在您的小組中建立焦點空間，共同作業。 只有團隊中擁有私人頻道之擁有者或成員的使用者可以存取頻道。 任何人（包括來賓）都可以新增為私人頻道的成員，只要他們已成為小組的成員即可。

如果您想要將共同作業限制在必須知道或想要協助指派給特定專案的一組人員之間通訊，您可能會想要使用私人通道，而不需要建立其他要管理的小組。

例如，在這些案例中，私有通道很有用：

- 小組中的一組人員想要集中共同作業，而不需要建立個別的小組。
- 小組中的人員子集需要私人管道來討論機密資訊，例如預算、資訊、戰略定位等。

[鎖定] 圖示代表 [私人頻道]。 只有專用通道的成員才能查看並參與已加入的專用頻道。

![小組中私有頻道的螢幕擷取畫面](media/private-channels-in-teams.png)

## <a name="what-you-need-to-know-about-private-channels"></a>私人通道所需注意的事項

目前，私有通道支援連接器和索引標籤（除了串流、Planner 和表單之外）。 我們正在使用完整的 app 支援專用通道，包括訊息擴充功能和機器人。

每個團隊最多可以有30個專用通道，每個私人通道最多可以有250個成員。 30個專用通道限制是每個團隊200標準通道限制的補充。

> [!NOTE]
> 我們會持續將功能新增至專用頻道，讓您查看最新的 app 相關資訊、頻道會議，以及大型團隊的伸縮專用頻道。

## <a name="when-to-create-a-private-channel"></a>建立私人頻道的時機

若要判斷專用通道是否適當，請考慮下列問題：需要共同合作的人員，以及共同作業的相關資訊。

|是否已有團隊將這些人員做為小組成員？  |此工作是否需要與其他人保密？  |有多個不同的主題要討論嗎？  |推薦  |
|---------|---------|---------|---------|
|是      |是         |是         |在現有的小組中建立私人頻道，或考慮建立每個主題的專用專用頻道。         |
|是     |是         |否         |在現有的小組中建立私人頻道。         |
|是     |否         |否         |在現有團隊中建立頻道。         |
|否     |否         |否         |考慮建立新團隊。         |
|否     |否         |是         |考慮建立新的團隊，然後根據每個主題的機密性，考慮為每一個主題建立個別的標準或私人頻道。         |
|否     |是         |否         |在現有團隊中建立新團隊或建立新的專用頻道。         |

建立私人頻道之後，該頻道就會連結至上層小組，而且不能移至不同的小組。 此外，無法將專用通道轉換成標準通道，反之亦然。

## <a name="private-channel-creation-and-membership"></a>私人通道建立與成員資格

### <a name="who-can-create-private-channels"></a>誰可以建立私人頻道？

根據預設，任何團隊擁有者或團隊成員都可以建立私人頻道。 來賓無法建立。 您可以在小組階層和組織層級管理建立私人頻道的能力：

- 在團隊的 [**設定**] 索引標籤上，小組擁有者可以關閉或開啟成員建立私人頻道的能力。
- 身為系統管理員，您可以使用[原則](teams-policies.md)來控制貴組織中允許哪些使用者建立私人頻道。

建立私人頻道的人員是私人頻道擁有者，只有私人頻道擁有者可以直接在其中新增或移除人員。 私人頻道擁有者可以將任何小組成員新增至他們建立的私人頻道，包括來賓。 私人通道的成員有安全的交談空間，新增成員後，就可以在該私人頻道中看到所有交談（甚至是舊交談）。

### <a name="what-happens-when-a-team-member-leaves-or-is-removed-from-a-team"></a>小組成員離開或移除小組時會發生什麼情況？

如果小組成員離開或移除團隊，該使用者也會離開或移除團隊中的所有私人頻道。 如果使用者已新增回團隊，則必須將其新增回團隊中的專用頻道。

### <a name="what-happens-when-a-private-channel-owner-is-removed-from-a-private-channel"></a>當私人頻道擁有者從私人頻道中移除時，會發生什麼情況？

如果私人頻道擁有者是一或多個私人頻道的最後一個擁有者，則無法透過團隊用戶端移除。

如果私人頻道擁有者離開您的組織，或從與團隊相關聯的 Office 365 群組中移除，則私人通道的成員會自動升級為私人通道擁有者。

### <a name="what-can-team-owners-and-team-members-see-in-a-private-channel"></a>小組擁有者和小組成員可以在私人頻道中看到哪些內容？

小組擁有者可以查看其小組中所有私人頻道的名稱，也可以刪除小組中的任何私人頻道。 （已刪除的私人通道可以在刪除後的30天內還原）。 小組擁有者無法在私人頻道中看到檔案，或是在私人頻道的 [交談與成員] 清單中查看檔案，除非他們是該私人頻道的成員。

下表顯示誰可以查看私人頻道中的內容。

|選項  |小組擁有者可以查看 |小組成員可以查看 |
|---------|---------|---------|
|名稱和描述    |團隊中的所有私人頻道         |僅限新增的專用頻道         |
|交談與索引標籤     |僅在新增至專用頻道時         |僅在新增至專用頻道時         |
|檔案和內容    |僅在新增至專用頻道時        |僅在新增至專用頻道時         |
|私人頻道擁有者    |團隊中的所有私人頻道        |僅在新增至專用頻道時         |
|上次啟用時間戳  |團隊中的所有私人頻道       |僅在新增至專用頻道時         |

### <a name="manage-private-channel-membership-and-settings"></a>管理私人頻道成員資格與設定

每個專用通道都有自己的設定，包括新增和移除成員的功能、新增索引標籤，以及整個通道的 @mentioning。 這些設定與父團隊設定無關。 建立專用頻道之後，它會從父團隊繼承設定，之後就可以獨立于父團隊設定變更其設定。

私人頻道擁有者可以按一下 [**管理頻道**]，然後使用 [**成員**] 和 [**設定**] 索引標籤來新增或移除成員及編輯設定。

請參閱[管理團隊私人頻道的生命週期](private-channels-life-cycle-management.md)，以取得如何管理組織中專用頻道生命週期的指導方針。 這包括如何控制貴組織中的使用者是否可以建立私人頻道，如何代表小組擁有者建立私人頻道，以及如何取得用於封存和審核用途的所有私人通道訊息清單，以及其他管理工作。  

## <a name="private-channel-sharepoint-sites"></a>私人頻道 SharePoint 網站

每個私人通道都有自己的 SharePoint 網站集合，針對檔案共用和快速資源調配進行優化。 個別的網站集合是確保對私人通道檔案的存取權僅限於專用通道的成員，與小組擁有者可以存取網站集合中所有資產的小組網站的方式不同。 根據預設，這些網站集合是使用文件庫建立的，而且可以透過[網站管理介面](https://support.office.com/article/Enable-or-disable-site-collection-features-A2F2A5C2-093D-4897-8B7F-37F86D83DF04)輕鬆地增強至完整功能的網站集合。 每個網站集合都是在與父團隊的網站集合相同的地理區域中建立。 這些羽量級網站具有自訂的範本識別碼（TEAMCHANNEL # 0），可讓您更輕鬆地透過 PowerShell 與圖形 API 進行管理。

為了適應每個租使用者的網站集合數量上限，此限制已從500000增加到2000000。 私人通道網站集合會同步處理資料分類，並繼承父小組網站集合的來賓存取權。  網站集合擁有者和成員群組的成員資格會與團隊內的私人頻道成員資格保持同步。 在 SharePoint Online 中，對擁有者或成員群組的成員資格所做的任何變更，都會在四個小時內自動還原為私人通道成員資格。 當特定使用者需要存取檔而不需要存取私人通道訊息時，請將其新增至網站上的 [訪客] 群組或與 [擁有者] 和 [成員] 不同的新群組。

團隊會管理私人頻道 SharePoint 網站集合的生命週期。 如果網站集合是在小組外刪除，只要私人通道仍在使用中，背景作業就會在四個小時內復原網站。 如果網站已刪除且已實刪除，就會為私人頻道提供新的網站集合。

如果已還原私人頻道或包含私人頻道的小組，網站集合就會還原。 如果私人通道網站集合已還原，且超過專用通道的30天虛刪除視窗，網站集合就會以獨立的網站集合的方式運作。

## <a name="private-channel-message-compliance-records"></a>私人通道訊息合規性記錄

在私人通道中傳送的訊息記錄會傳送到所有私人通道成員的信箱，而不是群組信箱。 記錄的標題會設定格式，以指出其傳送的是哪一種私人頻道。

如需有關執行電子檔探索搜尋私人通道訊息的詳細資訊，請參閱電子檔探索（[私人頻道](ediscovery-investigation.md#ediscovery-of-private-channels)）。

## <a name="considerations-around-access-in-private-channels"></a>私人通道存取權的考慮

當您在私人頻道中建立新的 OneNote 筆記本時，其他使用者仍然可以取得筆記本的存取權，因為行為與使用者的私人頻道 SharePoint 網站中其他專案的共用存取權相同。

如果使用者有權透過 SharePoint 在私人頻道中存取筆記本，移除團隊或私人頻道中的使用者將不會移除使用者對筆記本的存取權。

如果將現有的筆記本新增為專用頻道的索引標籤，則不會變更對專用通道的存取權。 這表示下列事項：

- 並非私人頻道中的每個人都可預設存取筆記本。 這是因為他們可能無法存取存放筆記本的位置，例如另一個小組的 SharePoint 網站。
- 不是私人頻道成員的使用者可以查看筆記本。
