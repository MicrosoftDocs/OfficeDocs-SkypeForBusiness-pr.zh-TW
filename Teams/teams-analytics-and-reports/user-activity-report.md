# <a name="microsoft-teams-user-activity-report"></a>Microsoft 團隊使用者活動報告

[團隊使用者活動報告] 可深入瞭解貴組織中的使用者在團隊中執行的活動類型。 例如，您可以看到有多少使用者透過未排程的會議（通常稱為1:1 和群組通話）來進行多人的通訊、小組使用者已組織的會議，以及參與團隊使用者參與的會議。 我們會分享螢幕、影片和音訊分鐘的詳細資料，以及聊天通訊統計資料，例如，回復和張貼頻道訊息的使用者數目，以及有多少使用者參與1:1 或群組聊天訊息。

## <a name="view-the-user-activity-report"></a>查看使用者活動報告

您必須是團隊服務管理員，才能進行這些變更。 請參閱 [使用團隊管理員角色管理團隊](https://docs.microsoft.com/microsoftteams/using-admin-roles) ，瞭解如何取得管理員角色和許可權。

1. 在 Microsoft [團隊管理中心] 的左導覽中，按一下 [**分析] & 報告**  >  **使用方式報告**。 在 [ **查看報表** ] 索引標籤的 [ **報表**] 底下，選取 [ **團隊使用者活動**]。
2. 在 [ **日期範圍**] 底下，選取一個範圍，然後按一下 [ **執行報表**]。

    ![[團隊系統管理中心] 的 [小組] 使用者活動報告的螢幕擷取畫面（含標注）](../media/teams-reports-user-activity-with-callouts.png "[團隊系統管理中心] 的 [小組] 使用者活動報告的螢幕擷取畫面（含標注）")

## <a name="interpret-the-report"></a>解讀報表

|圖說文字 |描述  |
|--------|-------------|
|**1**   |您可以在過去7天、30或90天的趨勢中查看團隊使用者活動報告。 |
|**2**   |每個報告都有產生此報告的日期。 報告通常會反映來自啟用時間的24小時延遲時間。 |
|**3**   |<ul><li>圖表上的 X 軸是特定報表的已選取日期範圍。 </li><li>Y 軸是參與活動的使用者數目。</li></ul>將游標暫留在代表指定日期的活動點上，即可查看該活動在該指定日期的實例數。 |
|**4**   |您可以按一下圖例中的專案，篩選您在圖表上看到的內容。 例如，按一下 [ **1:1 通話**]、[ **頻道訊息**] 或 [ **聊天訊息** ]，只查看與每一個相關的資訊。 變更選取範圍不會變更表格中的資訊。 |
|**500**   |此表格提供使用者的使用方式細分。   <ul><li>[**使用者名**] 是使用者的顯示名稱。 您可以按一下顯示名稱，移至 Microsoft 團隊系統管理中心的 [使用者設定] 頁面。</li><li>[**頻道訊息**] 是使用者在特定的時段內，在小組聊天中張貼的唯一訊息數目。</li><li>[**回復訊息**] 是使用者在特定期間內于團隊頻道中張貼的唯一回復訊息數。</li> <li>[**張貼訊息**] 是在指定期間內，使用者在團隊頻道中張貼的唯一張貼訊息數目。</li><li>[**交談訊息**] 是使用者在特定的時段內，在私人聊天中張貼的唯一訊息數目。</li><li>**緊急郵件** 是指使用者在指定的時段內，在聊天中張貼的緊急訊息數目。</li><li>[**總會議** 數] 是使用者在指定的時段內參與的排程和點對點會議總數。</li><li>[**組織會議** 數] 是使用者在指定期間內組織的排程與點對點會議數目。</li><li>[**井然有序排程的會議**] 是使用者在指定期間內組織的排程會議數。</li><li>[組織中的 **會議**] 是使用者在指定的時間期間組織的臨時會議數量。</li><li>[**會議參與**] 是使用者在特定期間內參與的排程和臨時會議數。</li><li>**參與會議的會議** 是指使用者在指定時段內參與的排程會議數。</li><li>[參與中的 **會議**] 是指使用者在特定的時段內參與的 ad hoc 會議數目。</li><li>[ **1:1 通話**] 是使用者在指定期間內參與的1:1 通話數目。</li><li>[**音訊時間**] 是使用者在指定的時段內參與的總音訊時間。</li><li>[**視頻時間**] 是使用者在特定的時段內參與的視頻總時間。</li><li>**螢幕共用時間** 是指使用者在指定的時段內參與的螢幕共用時間總計。</li>  <li>[**上一個活動**] 是使用者參與團隊活動的最後一個日期 (UTC) 。</li><li>當使用者被視為有效，但其值為零的情況下，表示聊天訊息、1:1 通話、通道訊息、會議總數及會議的值為零的 **其他活動** 追蹤。 範例動作是在使用者開啟通道訊息後但不回復，或當使用者收到私人訊息但不回應它時。</li> </ul>請注意， **群組通話** 已由 **會議** 所 **取代，這** 兩個值的總和等於由 **群組通話** 衡量的專案。
