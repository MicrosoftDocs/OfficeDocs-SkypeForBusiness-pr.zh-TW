# <a name="plan-for-teams-auto-attendants-and-call-queues"></a>規劃 Teams 自動撥打和通話佇列

自動語音回應允許您設定功能表選項，以根據來電者輸入來路由通話。 功能表選項，例如「針對銷售，請按 1。  針對服務按 2"，針對自動語音回應，讓組織提供一系列選項，引導來電者快速抵達目的地，而不需要仰賴接線生處理來電。

通話佇列正在等候來電者。 在來電者需要與特定特殊人員聯繫的情況下 ，例如銷售或服務，而非特定人員，您可以使用通話佇列將來電者連接至可以協助他們的代理人群組。 在指派至佇列的代理人可以參與通話之前，來電者會保留。

自動語音回應和通話佇列一起使用時，可以輕鬆地將來電者路由至貴組織中適當的人員或部門。

## <a name="auto-attendants"></a>自動語音應答

自動語音回應的主要目的，是依據來電者在提供之功能表選項的輸入，將來電者引導至適當的人員或部門。 來電者可以導向至貴組織中特定人員、撥打佇列等候與下一位可用的代理人通話，或撥打語音信箱。 您可以針對上班時間、休息時間及假日指定不同的通話路由選項。

功能表提示可以使用文字到語音 (系統產生的提示或) 上傳錄製的音訊檔案來建立。 語音辨識可接受免手流覽的語音命令，但來電者也可使用電話鍵盤來流覽功能表。

每個自動翻譯人員都有特定的語言和時區。 如果您以多種語言或世界多個地區進行商務，您可以建立許多不同的自動語音回應，以容納來電者。

您可以針對每個自動 Attendant 設定運算子。 雖然您可以將運算子通話設定為前往各種目的地，但是運算子功能的設計目的是讓來電者與貴組織的特定人員通話，他們可協助他們。

您可以將自動語音語音機的自動語音訊息機，無論是根據名稱或分機號碼，都允許來電者搜尋貴組織的目錄。 在自動參與中，您可以選擇要包含或排除的使用者群組，指定哪些人可進行目錄搜尋。  (這稱為撥號 *範圍*.) 

來電者可以撥打自動語音機，方法有直接電話號碼 ，如果有，也可以從另一個自動語音機或通話佇列重新導向。

## <a name="call-queues"></a>通話佇列

通話佇列會排入實體大樓的等候室。 來電者等候等候，而通話會路由至佇列中的代理人。 通話佇列通常用於銷售與服務功能。 不過，通話佇列可用於通話數量超過您內部容量的任何情況，例如忙碌設施中的看機員。

當佇列的來電者總數或等待時間超過您指定的限制時，通話佇列會允許特定通話路由。 通話可以路由至特定人員、語音信箱、其他通話佇列或自動語音留言。

和自動撥打的通話一樣，通話佇列都有語言設定。 如果您以多種語言進行商務，您可以使用不同的通話佇列。 代理程式若是多語言，可以是多個佇列的成員。

對於每一個通話佇列，您可以指定佇列中的代理人是否可退出宣告通話，以及通話是否該根據 Teams 中的目前狀態指示路由至通話。

您可以將電話號碼指派給通話佇列，但是通話佇列不會針對休假和假日提供個別的通話路由。 除非您的通話佇列是 24 小時無時無分的員工，否則建議您將電話號碼指派給自動撥打服務，在上班時間重新導向到通話佇列。

## <a name="prerequisites"></a>必要條件

若要設定自動撥打和通話佇列，您需要下列資源：

- 每個自動撥打和每個通話佇列的資源帳戶
- 免費的電話系統 - 每個資源帳戶的虛擬使用者授權
- 針對您想要直接 [撥號](getting-service-phone-numbers.md)的每個資源帳戶，至少一個 Microsoft 服務號碼、直接路由號碼或混合式號碼
 - 服務號碼可能是付費或免付費號碼

從通話佇列接聽來電的代理人必須啟用企業語音線上或內部部署使用者。 此外，如果通話佇列使用直接路由號碼，需要電話會議或轉接電話的代理人也需要：

- 在通話佇列使用轉接模式時指派的線上語音路由策略
- 如果通話佇列使用會議模式，則指派的音訊會議授權或線上語音路由策略

如果您的代理人使用 Microsoft Teams 應用程式進行通話佇列通話，他們必須使用 TeamsOnly 模式。

將電話轉接到外部電話號碼時，資源帳戶會執行轉接 (，也就是說，與自動 Attendant 或通話佇列關聯的資源帳戶) 必須具有電話號碼和 Microsoft 365 電話系統虛擬使用者授權。 此外：

- 針對有通話方案號碼的資源帳戶，請指派 [通話方案](calling-plans-for-office-365.md) 授權。
- 針對具有直接路由號碼的資源帳戶，指派 [線上語音路由策略](manage-voice-routing-policies.md)。

> [!NOTE]
> Microsoft Teams 使用者和通話代理程式只支援自動撥打服務佇列和通話佇列的直接路由服務號碼。<br>
> 不支援在通話方案主幹與直接路由主幹之間傳輸。

## <a name="business-decisions"></a>商業決策

在設定自動參與和通話佇列之前，您應先決定如何在企業使用這些功能。 這些決定將決定您設定自動撥打和通話佇列時所選擇的設定。

記錄您對於這些問題的回答，並提供資訊給執行此配置的系統管理員。

- 您需要哪些語言？ 需要哪些語言 -哪個部門或群組？
- 是否允許來電者的語音輸入或僅允許撥號輸入？
- 您是否需要針對休假時間或假日另行呼叫路由？ 小時和假日是什麼？
- 您想要允許通話佇列的代理人退出宣告接聽電話嗎？
- 您希望通話佇列的代理人或運算子在撥出時，擁有特定的本機號碼？
- 您想要在 [貴組織中啟用](call-park-and-retrieve.md) 通話回收和取回功能，以協助人員或部門之間提供呼叫協助嗎？
- 針對語音提示，您想要錄製您自己的語音或使用系統產生的語音嗎？  (系統產生的語音很容易更新。) 

## <a name="technical-decisions"></a>技術決策

使用自動語音回應和通話佇列將來電者與貴組織人員聯繫時，在開始組排之前，您需要進行一些技術決定。

您可以在通話佇列新增代理程式，方法如下：

- 個別使用者
- 通訊群組清單
- 安全性群組，包括已啟用郵件功能的安全性群組
- Microsoft 365 群組或 Teams

如果需要，您可以針對每個佇列使用這些選項的組合。 具有電子郵件地址的群組可用於語音信箱。 使用 Teams 有許多優點，包括共用檔案儲存空間與代理程式之間的交談、可接收語音信箱的一般信箱，以及可整合至企業行應用程式或 Power Apps 的可擴展平臺。

建議您在開始進行組選之前，先選擇將通話代理程式排入佇列的策略。

如果您有現有的自動撥打和通話佇列基礎結構，且要移轉至 Teams，您需要將現有電話號碼移入新自動撥打和通話佇列的計畫。 您可能需要建立移轉訂單 [，才能](phone-number-calling-plans/port-order-overview.md) 將號碼從另一個提供者移轉。 建議您先暫時取得一或多個新電話號碼，並測試自動參與和通話佇列，然後再切換至您目前使用中的號碼。

*會議模式* 是通話佇列的一個選項，可大幅縮短將 Teams VOIP 通話和 PSTN 通話連接到代理人所花的時間。 若要讓會議模式運作，通話佇列中的代理人必須使用下列其中一個用戶端：

- 最新版本的 Microsoft Teams 桌面用戶端、Android App 或 iOS 應用程式
  - Microsoft Teams 手機版本 1449/1.0.94.2020051601 或更新版本
  
將代理人的 Teams 帳戶設為 Teams -only 模式。 不符合需求的代理人不會包含在通話路由清單中。

如果您的代理人都使用相容的用戶端，建議您為通話佇列啟用會議模式。

> [!NOTE]
> 會議模式不支援忙碌中。 若未啟用目前狀態式路由，非通話佇列通話的代理程式仍可能會顯示通話佇列通話。

## <a name="plan-your-call-routing-flow"></a>規劃您的通話路由流程

在規劃程式進行時，建議您在圖表中找出組織的通話路由。 此圖表可協助判斷使用者如何以最有效率的方式路由到您的組織。 您也可以使用圖表來判斷您需要建立的自動 Attendant 和通話佇列，以及服務號碼、授權和資源帳戶等相關需求。

讓我們看看自動有如自動有聲人員與撥打佇列路由通話一樣如何。

自動參與人員會以下列其中一種方式路由所有通話：

- **立即重新** 導向 - 通話可以立即重新導向至下列其中一 (通話路由目的地) 接聽或首次問候語後立即重新導向。
- **根據撥號選項重新** 導向 - 來電者可以導向至電話鍵盤上號碼 0-9 的選項之間做選擇。 每個撥號鍵都可以指派一個電話路由目的地。
- **按名稱或** 分機撥號給人員 - 來電者可以撥打他們正嘗試在貴組織目錄中聯繫的對方的分機號碼，或是拼字。
- **中斷** 連接 - 自動通話人員可能會掛斷通話。

> [!NOTE]
> 單一自動撥打服務只能支援單一「撥號方式」方法。  若要允許來電者以名稱和號碼撥號，您必須建立一個自動語音語音機，其選項為按名稱撥號，而另一個則選以分機撥號。  這些選項會路由到各個針對這些「撥號方式」案例所配置的自動 Attendant。

當自動參與人員或通話佇列重新導向來電時，您可以選擇下列通話路由目的地：

- **組織中可以接聽** 語音通話的人，這是貴組織的人。 這可以是線上使用者，或使用商務用 Skype Server 託管于內部部署的使用者。
- **語音應用程式** - 另一個自動語音語音機或通話佇列。 選擇與目的地相關聯的資源帳戶。
- **外部電話號碼** - 任何電話號碼。  (外部[移轉技術詳細資料，) 。](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details)
- **語音** 信箱 - 與所指定的 Microsoft 365 群組相關聯的語音信箱。
- **僅** (自動) 運算子 - 自動 attendant 中定義的運算子。 定義運算子是選擇性的。 運算子可以是清單中的其他任何目的地。

針對在上班時間外及假日接到的來電，自動撥打服務會提供個別的通話路由選項。 上班時間通話路由允許上述所有選項，而假日通話路由僅允許重新導向或中斷通話的中斷，但是沒有撥號鍵選項。

通話佇列會保留來電者，直到被指派到佇列的代理人可以接到來電。 有兩種情況可能會將來電者導向至佇列外：

- **通話溢出** - 如果佇列中的通話數量超過您設定的限制，則新的來電者會重新導向到佇列外。
- **通話超時** - 如果來電者處於佇列中的時間超過設定超時設定的時間，則來電者會重新導向至佇列外。

重新導向至佇列以外的通話會傳送至上述所列的任何通話路由目的地，但運算子除外。  (通話佇列沒有運算子，但您可以將來電者重新導向到與針對自動語音機所配置的運算子相同的目的地。) 

下列範例顯示使用自動 Attendant 和通話佇列的通話路由範例。

![使用自動 Attendant 和通話佇列的通話路由圖表](media/attendant-and-queue-call-routing.png)

在上例中：

- 0 (0) 會將來電者重新導向至運算子。 自動參與的運算子已設有組織 **人員。**
- 一對 (金鑰) 來電者重新導向到銷售通話佇列。 此通話佇列會連接到包含指派給佇列的銷售團隊的團隊。
- 這兩 (2) 金鑰會將來電者重新導向至支援通話佇列。 此通話佇列會連接到包含指派給團隊的支援小組的團隊。
- 支援通話佇列會透過中間自動參與提供直接電話號碼。 讓自動回應接聽支援行，可分開上班時間和假日電話路由。
- 這三 (3) 金鑰會將使用者重新導向至公司目錄的另一個自動 Attendant。 公司目錄自動語音回應可讓來電者撥號至組織內的人名或分機。

我們建議您建立類似上述的一或多個圖表，以找出您的通話路由。 請務必在圖表或隨附檔中包含下列內容：

- 哪些自動出席者可透過電話號碼直接存取？
- 每個自動 Attendant 的休假和假日路由需求是什麼？
- 每個通話佇列的成員資格。  (您可以個別新增使用者，或將佇列與不同類型的群組進行地圖。 將佇列與團隊進行比對，可提供最多種的體驗。) 

以下是一些通話路由最佳作法：

- 查看您現有的通話系統，並分析來電的類型和頻率。 使用這項資訊來協助通知自動有線電話機和通話佇列結構。
- 最早在功能表中放入最常見的選項，以儘快路由通話。
- 除非能24/7 使用佇列，否則請避免將服務號碼直接連接至通話佇列。 通話佇列不允許針對休假時間或假日進行個別通話處理。 如果您想要讓佇列有直接號碼，請指派號碼給自動 Attendant，自動在上班時間重新導向到佇列。
- 如果您接到許多電話，要求公司的基本資訊 ，例如上班時間、位置或網址，請考慮建立自動回應，以錄製的訊息回答這些問題。
- 將功能表項目清單保持在五個或五個以下。 來電者可能會無法記住五個選項。 如果需要更多選項以正確路由通話，請使用巢式自動參與。
- 先描述服務，再接著按 (選項，例如：針對 Sales 按 1) 而不是以其他方式 (例如。 按 1 表示銷售) 。
- 來電者會瞭解使用者術語，而非您可能會在內部使用術語。
- 避免經常更新通話路由。 如果您之後變更自動語音機的功能表選項，前 30 天內會從語音提示中呼叫。

## <a name="getting-started"></a>快速入門

完成本文中的規劃工作後，請遵循下列步驟來設定自動撥打和通話佇列：

1. 取得自動 Attendant 和通話佇列所需的服務號碼，您希望組織外部直接撥號才能使用服務號碼。 這可能包括 [從另一個提供者移](phone-number-calling-plans/transfer-phone-numbers-to-teams.md) 轉號碼， [或要求新的服務號碼](getting-service-phone-numbers.md)。

2. 取得 [電話系統 - 您計畫建立的每個](teams-add-on-licensing/virtual-user.md) 資源帳戶的虛擬使用者授權。 這些授權是免費的，因此我們建議您多取得一些授權，以防您日後決定變更資源帳戶。

3. [針對要建立的每個](manage-resource-accounts.md) 自動撥打和通話佇列建立資源帳戶。 為每個帳戶指派電話系統 - 虛擬使用者授權，以及服務號碼 ，可以選擇性地指派。

4. [建立您希望](set-up-holidays-in-teams.md) 自動有個別通話路由的假日。

5. 如果您想要使用 [這項功能來](call-park-and-retrieve.md) 協助來電轉接，您也可以設定通話停駐與取回。

6. 建立您想要使用的群組，以包含通話佇列的通話代理程式。

7. 如果您打算允許以分機號碼撥號，請確保您已新增使用者的分機號碼至他們的 Azure Active Directory 設定檔。

完成上述步驟後，就可以建立自動撥打和通話佇列了。 因為自動有聲人員與通話佇列可以彼此重新導向通話，請參閱您建立工作流程圖表，判斷應該先建立哪一個自動參與者或通話佇列。 在上圖中的範例中，您可以在建立 Contoso 主自動語音機前，先建立銷售與支援通話佇列，因為主要自動語音回應需要將來電者引導至銷售和支援通話佇列。
