# <a name="audio-conferencing-common-questions"></a>音訊會議的常見問題

以下是我們從我們想要使用音訊會議的客戶所取得的主要問題。 
  
## <a name="what-are-the-benefits-of-audio-conferencing"></a>音訊會議有哪些優點？

例如，當人員在路上，且無法使用膝上型電腦或行動裝置上的商務用 Skype 或 Microsoft 團隊 app 出席會議時，通話至會議就是有益的。 但在其他情況下，使用電話參加商務用 Skype 或 Microsoft Teams 會議，會比在電腦上使用應用程式的效果更好：
  
- 當網際網路連線能力受限制時。
    
- 當會議只有音訊時。
    
- 人員嘗試加入商務用 Skype 會議，但失敗。
    
- 撥號品質越好，您就能在撥入時撥打。
    
- 與會者可以使用藍牙裝置以「免持」方式加入會議。
    
- 與會者會發現這對他們而言更為簡單且方便。
    
## <a name="who-can-attend-an-audio-conferencing-meeting-and-who-can-i-hear"></a>誰可以出席語音會議會議？ 我可以聽到哪些人？

任何擁有撥入號碼和會議 ID 的人都可以加入商務用 Skype 或 Microsoft 團隊會議，除非會議召集人已鎖定會議。
  
無論您是使用手機或商務用 Skype 或 Microsoft 團隊 app 撥打電話，您都可以在通話時聆聽其他人，也可以聽到您的聲音。 如果會議召集人不想聽到出席者，就能讓他們「靜音」。 
  
## <a name="can-i-add-a-toll-free-number-for-my-audio-conferencing-users"></a>我可以為我的音訊會議使用者新增免付費電話號碼嗎？

是的，免付費電話號碼 (的服務號碼) 提供，但只能在部分國家/地區使用。 如需可用的號碼清單，請參閱 [音訊會議與通話方案的國家和地區可用性](country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans.md)。
  
## <a name="how-many-local-dial-in-numbers-are-currently-supported"></a>目前支援多少本機撥入號碼？

當您購買音訊會議的授權時，會有指派給您的本機撥入號碼。 電話撥入號碼會包含在會議邀請中。 這些當地電話號碼將只適用于您的組織。 指派給貴組織的電話，該號碼會由該組織中啟用音訊會議的使用者共用。 所以，由使用者 A 和另一個使用者 B 排程的商務用 Skype 或 Microsoft 團隊會議，都會有相同的撥入號碼。
  
當地撥入號碼，而且在某些情況下，您組織所在國家/地區的國際撥入號碼將會包含在會議邀請中。 如果會議出席者使用包含在邀請中的其他號碼，就會是共用的電話號碼。
  
## <a name="how-many-international-dial-in-numbers-does-audio-conferencing-in-microsoft-365-or-office-365-support"></a>Microsoft 365 或 Office 365 支援音訊會議的國際撥入號碼數？

如需國家/地區的目前清單，請參閱 [Microsoft 團隊中的音訊會議電話號碼](phone-numbers-for-audio-conferencing-in-teams.md) 或 [商務用 Skype Online 中的音訊會議](/SkypeForBusiness/audio-conferencing-in-office-365/phone-numbers-for-audio-conferencing)電話號碼。
  
## <a name="can-i-set-up-local-numbers-for-audio-conferencing-from-additional-cities-in-the-country"></a>我可以從國家/地區的其他城市設定音訊會議的當地號碼嗎？

如果您的地區無法使用音訊會議的電話號碼，或不符合貴組織的需求，請在 [SkypeFeedback 論壇](http://www.skypefeedback.com/forums/299910--preview/category/119971-pstn-conferencing)中傳送意見反應給我們。
  
## <a name="what-is-the-maximum-length-of-the-audio-conferencing-meetings"></a>音訊會議會議的長度上限為何？

時間的最大長度取決於會議中的人員，以及他們用來加入會議的驗證類型。
  
|**會議出席者**|**會議結束時間**|
|:-----|:-----|
|使用者已使用商務用 Skype 或 Microsoft 團隊 app 進行加入，或已撥入會議。  <br/> |如果在24小時後沒有任何出席者清單的變更，會議就會結束。  <br/> |
|所有使用者都已撥入會議，但有人使用 PIN 進入會議。  <br/> |會議會在24小時後結束。  <br/> |
|所有使用者都是以撥入方式參與會議，但沒有任何人使用 PIN 進入會議。  <br/> | 如果允許匿名使用者開始會議，會議會在4小時後結束。 如果匿名使用者無法啟動會議，且所有經過驗證的參與者都離開會議，會議將于最後一次驗證參與者留下的90分鐘結束。 <br/> |
   
## <a name="how-many-total-phone-participants-can-i-have-in-meetings"></a>我可以在會議中使用多少總的電話參與者？

請參閱會議中允許的出席者總數 [限制及規範](limits-specifications-teams.md) 。
  
若要瞭解會議限制的相關資訊，請參閱 [商務用 Skype Online 限制](https://technet.microsoft.com/library/skype-for-business-online-limits.aspx#bkmk_Meeting_LyncOnlineLimits)。
  
## <a name="why-did-users-start-receiving-emails-with-their-audio-conferencing-information"></a>為什麼使用者要開始接收含有其音訊會議資訊的電子郵件？

我們新增了一項新功能，讓您（ [管理員](https://support.office.com/article/eac4d046-1afd-4f1a-85fc-8219c79e1504)）傳送及更新音訊會議資訊和在電子郵件中 PIN。 若要深入瞭解它，包括如何停用它，請參閱 [啟用或停用 Microsoft 團隊中的音訊會議設定變更時傳送電子郵件](enable-or-disable-sending-emails-when-their-settings-change-in-teams.md) ，或在 [商務用 Skype Online 中的音訊會議設定變更時啟用或停](/SkypeForBusiness/audio-conferencing-in-office-365/enable-or-disable-sending-emails-when-their-settings-change)用傳送電子郵件。
  
## <a name="can-audio-conferencing-be-used-by-the-users-who-are-part-of-an-on-premises-deployment-of-skype-for-business-server"></a>在內部部署商務用 Skype Server 中，使用者可以使用音訊會議嗎？

首先，透過 [會議](meetings-first.md)，商務用 Skype Server 使用者可從團隊會議和小組音訊會議中受益。 
  
## <a name="can-a-user-get-a-personal-conference-id"></a>使用者可以取得個人的會議 ID 嗎？

商務用 Skype 和 Microsoft 團隊使用者將是隨機指派的會議 Id。 使用者無法保留或設定只有他們可以使用的靜態會議 ID。 
  
## <a name="can-i-use-audio-conferencing-with-skype-meeting-broadcast"></a>我可以使用音訊會議與 Skype 會議廣播嗎？

目前不支援讓音訊會議加入 Skype 會議廣播的使用者。
  
## <a name="can-a-user-get-operator-assistance-during-a-meeting"></a>使用者可以在會議期間取得操作員協助嗎？

否，在會議期間按 * 0，使用者就不能取得任何操作員協助或支援。 如果音訊會議有問題，組織的系統管理員可以與 [Office 365 的 Microsoft 支援](https://support.office.com/article/Microsoft-support-for-Office-365-32a17ca7-6fa0-4870-8a8d-e25ba4ccfd4b)人員取得聯繫。
  
## <a name="how-does-a-user-access-or-change-their-conference-id"></a>使用者如何存取或變更其會議 ID？

商務用 Skype 或 Microsoft 團隊使用者可以在 Outlook 和 Outlook 網頁版中排程會議，以找出指派給他們的會議 ID。 此外，使用者也可以在設定後傳送給他們的電子郵件中找到會議 ID。
  
> [!NOTE]
> 使用者將無法重設其會議 ID。 會議 ID 只能由您在組織的 [管理員](https://support.office.com/article/admin-eac4d046-1afd-4f1a-85fc-8219c79e1504)重設。 
  
我們正在努力解決方案，讓使用者存取及重設會議 ID，而不需要組織管理員的協助。
  
## <a name="how-do-users-access-or-change-their-pin"></a>使用者如何存取或變更其 PIN？

商務用 Skype 或 Microsoft 團隊使用者可以在設定之後，在要傳送給他們的電子郵件中找到 PIN。
  
> [!NOTE]
> 團隊使用者可以變更其 pin。 在會議邀請底部有一個連結，以重設其 pin。 
  
我們正在努力解決解決方案，讓使用者可以在沒有組織管理員的協助下存取及重設 PIN。
  
## <a name="what-in-meeting-dial-pad-commands-are-supported"></a>支援哪些會議中的撥號鍵板命令？

- * 6 (將自己設為靜音/取消靜音) 
    
- * 1 (播放可用的撥號鍵台命令的描述)  
    
## <a name="can-attendees-dial-out-to-international-phone-numbers-when-they-are-in-a-skype-for-business-or-microsoft-teams-meeting"></a>當出席者在商務用 Skype 或 Microsoft 團隊會議中時，可以撥出到國際電話號碼嗎？

是的，出席者可以在國際撥出電話，並邀請其他來電者加入商務用 Skype 或 Microsoft 團隊會議。 請參閱 [從 Microsoft 團隊會議撥出，讓其他人可以加入，](dialing-out-from-a-teams-meeting-so-other-people-can-join-it.md) 或 [從商務用 Skype Online 會議撥出，讓其他人可以加入該](/SkypeForBusiness/audio-conferencing-in-office-365/dialing-out-from-a-meeting-so-other-people-can-join-it)會議。
  
## <a name="how-does-a-skype-for-business-or-microsoft-teams-user-schedule-a-meeting-with-audio-conferencing-meeting-details"></a>商務用 Skype 或 Microsoft 團隊使用者如何使用音訊會議會議詳細資料排程會議？

當使用者被指派 **音訊會議** 授權，且使用者在 Outlook 或 outlook 網頁版中建立新的商務用 Skype 或 Microsoft 團隊會議時，電話撥入電話號碼和會議 id 會自動新增到會議邀請中。
  
## <a name="how-does-a-user-schedule-and-start-a-meeting-when-all-attendees-will-be-using-a-phone-to-dial-in"></a>當所有出席者都要使用電話撥入時，使用者如何排程及開始會議？

排程使用電話撥入的所有出席者所加入的會議，與排程一般線上會議並無不同。 不過，有兩種方式可以讓所有參與者都使用電話撥入的會議：

- **選項 #1**：根據預設，如果會議召集人和所有參與者都使用電話加入會議，會議召集人需要輸入她的音訊會議 PIN 才能開始會議。 來電者在撥打線上會議的電話號碼時，系統會詢問他們是否要作為指定會議的召集人進行驗證。 在召集人開始之前透過撥入方式加入會議的所有參與者都將放在大廳，並會在保留時聆聽音樂。 如果是商務用 Skype 會議，在召集人輸入其音訊會議 PIN 之後，所有參與者都會自動加入會議。 在 Microsoft 團隊會議中，參與者會根據召集人會議原則中的 [自動承認人員] 設定的值來加入會議。

- **選項 #2**： [允許未驗證的呼叫者成為會議中的第一位人員] 設定預設為停用。 如果針對指定的召集人啟用該設定，就可以在召集人輸入音訊會議 PIN 前，啟動由該召集人排程的所有會議。 啟用此設定時，在第一位參與者透過撥入電話號碼加入會議時，就會立即開始會議，而第一個參與者則不會放在大廳中。 如需詳細資訊，請參閱 [在 Microsoft 團隊中管理使用者的音訊會議設定](manage-the-audio-conferencing-settings-for-a-user-in-teams.md) ，或 [管理商務用 Skype Online 中使用者的音訊會議設定](/SkypeForBusiness/audio-conferencing-in-office-365/manage-the-audio-conferencing-settings-for-a-user)。
