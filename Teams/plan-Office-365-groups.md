<a name="plan-for-microsoft-365-groups-when-creating-teams-in-microsoft-teams"></a>在 Microsoft 365中建立團隊時，規劃群組Microsoft Teams
==========================================================

考慮使用群組Microsoft 365或建立團隊時，請考慮團隊會用於哪些用途、誰應具有存取權限，以及團隊預期會取得什麼結果。 請特別注意您建立頻道的數量，因為人員可能會因為內容散佈過細而 (過多頻道) 。

有兩種案例需要討論規劃群組Microsoft 365群組及其 (或) Microsoft Teams：

-   首先，由於客戶可以在 Groups 中擁有現有的投資，我們目前同時支援公用和私人群組，針對目前支援的成員數目，請參閱 Microsoft Teams[的限制和Microsoft Teams。](./limits-specifications-teams.md) 如先前所述，您想要使用用戶端管理團隊Teams成員資格，而不是Microsoft 365 系統管理中心。 根據此案例，如果人員習慣使用 Microsoft 365 群組中的對話對話，則值得注意的是，群組交談基本上是電子郵件，與 Teams 頻道中的聊天訊息不同。 教育您的人員瞭解這一差異，並建議他們採用更靈活的聊天訊息格式，Teams或 OWA 以電子郵件Outlook群組。

-   第二，針對在 Microsoft 365 中尚未定義現有群組的客戶，您可以使用 Microsoft 365 系統管理中心、Teams或桌面用戶端來建立群組。 如先前所述，使用用戶端管理Microsoft 365群組Teams成員資格。 由於團隊成員資格也會定義群組Microsoft 365成員資格，因此您應該為人員進行這項變更做好準備。

## <a name="teams-respects-microsoft-365-groups-naming-policy"></a>Teams組Microsoft 365命名政策

系統Microsoft 365系統管理員所設定的任何群組命名原則，都會在使用者建立Teams編輯小組名稱時套用至群組名稱。 這包括強制首碼或尾碼，以及排除禁止的字詞等專案。

若要深入瞭解，請參閱 Microsoft 365[中的群組命名Teams。](https://support.office.com/article/Office-365-Groups-Naming-Policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552)
