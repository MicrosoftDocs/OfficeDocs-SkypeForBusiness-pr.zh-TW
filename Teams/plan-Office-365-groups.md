<a name="plan-for-microsoft-365-groups-when-creating-teams-in-microsoft-teams"></a>在 Microsoft Teams 中建立團隊時，規劃 Microsoft 365 群組
==========================================================

考慮使用 Microsoft 365 群組或建立團隊時，請考慮團隊會用於哪些用途、誰應具有存取權限，以及團隊預期會取得什麼結果。 請特別注意您建立頻道的數量，因為人員可能會因為內容散佈過細而 (過多頻道) 。

有兩種案例需要討論 Microsoft 365 群組的規劃，以及這些群組對 Microsoft Teams (或) 的影響：

-   首先，由於客戶可以在 Groups 中擁有現有的投資，我們目前同時支援公用和私人群組，針對目前支援的成員數目，請參閱 [Microsoft Teams](./limits-specifications-teams.md)的限制和規格。 如先前所述，您想要使用 Teams 用戶端而非 Microsoft 365 系統管理中心來管理團隊中的人員成員資格。 根據此案例，如果人員習慣在 Microsoft 365 群組中討論討論，則值得注意的是，群組交談基本上是電子郵件，與 Teams 頻道中的聊天訊息不同。 教育您的人員瞭解這一差異，並建議他們採用 Teams 中更靈活的聊天訊息格式，而不是使用 Outlook 或 OWA 以電子郵件方式將群組電子郵件。

-   第二，針對在 Microsoft 365 中尚未定義現有群組的客戶，您可以使用 Microsoft 365 系統管理中心、Teams Web 或桌面用戶端建立群組。 如先前所述，使用 Teams 用戶端管理 Microsoft 365 群組的所有未來成員資格。 由於團隊成員資格也會定義 Microsoft 365 群組的成員資格，因此您應該為人員進行這項變更做好準備。

## <a name="teams-respects-microsoft-365-groups-naming-policy-in-private-preview"></a>Teams 尊重 Microsoft 365 群組命名 (在私人預覽版) 

當使用者建立或編輯團隊名稱時，系統將會套用系統管理員所設定的任何 Microsoft 365 群組命名原則。 這包括強制首碼或尾碼，以及排除禁止的字詞等專案。

> [!NOTE]
> 此功能為私人預覽，這表示如果您不是此預覽的一部分，Teams 尚未遵守此 Microsoft 365 群組命名原則。

若要深入瞭解，請參閱 [Teams 中的 Microsoft 365 群組命名政策](https://support.office.com/article/Office-365-Groups-Naming-Policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552)。
