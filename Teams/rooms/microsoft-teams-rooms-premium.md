# <a name="microsoft-teams-rooms-managed-service"></a>Microsoft Teams 會議室受管理的服務

Microsoft Teams 會議室管理的服務是一種雲端式 IT 管理和監控服務，可讓Microsoft Teams 會議室裝置及其周邊裝置保持在最新狀態，並主動受到監視，支援針對最佳使用者體驗優化的環境。

Microsoft Teams 會議室管理的服務適用于想要改善其會議室體驗的組織，並提供Microsoft Teams 會議室裝置的主動管理，以快速擴展其業務量。 這項服務可讓組織節省時間、金錢和專業知識，否則會花在「自己動手」的聊天室管理上。  

Microsoft Teams 會議室管理服務是以 Microsoft 在支援軟體管理端點、機器學習和智慧自動化方面的專業知識為基礎而建置。 Microsoft Teams 會議室管理的服務可協助組織跨多個遠端網站提供高品質的體驗，並可適應組織可能需要的任何成長速度。

以下是服務的三大層面：  

- 智慧型作業  

   可自動化更新、問題偵測、Microsoft Teams 會議室問題解決方式的軟體和機器學習。  



- 專屬專家  

   提供 24x7 服務作業、層級支援及事件解決協助的專家小組。  



- 增強的深入解析  

   針對許多客戶提供豐富的分析、報告和經驗證的學習。

## <a name="service-prerequisites"></a>服務先決條件

若要讓Microsoft Teams 會議室管理的服務維持服務品質，註冊的會議室必須[Microsoft Teams 會議室支援的裝置](requirements.md#hardware-requirements)，而且您必須使用會議室裝置完全允許和存取的所有[必要Office 365 URL 和 IP 位址範圍](/office365/enterprise/urls-and-ip-address-ranges)來準備您的環境。 若要深入瞭解，請參閱Microsoft Teams 會議室檔準備[您的環境](rooms-prep.md)。

做為受管理服務的提供者，Microsoft 需要與組織中負責的專案關係人合作，讓會議室保持健康並從問題中復原。 您必須負責會議室中所使用的未經授權裝置、您選擇在會議室裝置中部署的協力廠商或自訂軟體，並遵守 Microsoft 建議的變更管理做法。

貴組織或貴組織所簽訂之合作夥伴必須準備好以補救服務偵測到的最後一個里程問題。

## <a name="data-collection"></a>資料收集

Microsoft Teams 會議室管理的服務會監視裝置、收集裝置資料，並允許 Microsoft 遠端存取和管理會議室裝置。 Microsoft Teams 會議室管理的服務無法存取會議的即時內容。

收集的資料是專為聊天室帳戶而非個別使用者。 個別使用者的附隨性參照可能會在使用裝置期間出現在活動記錄檔中。 收集的資料是由 Microsoft 依照 [Microsoft 隱私權聲明](https://aka.ms/privacy)進行處理。
