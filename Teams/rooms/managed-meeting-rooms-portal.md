# <a name="microsoft-managed-meeting-rooms-portal"></a>Microsoft 管理的會議室入口網站

## <a name="overview"></a>概觀

管理型會議室入口 (「會議室入口) 」入口網站提供您會議室健康狀態的視圖。 此入口網站的客戶視圖可顯示您的可見度與意見，並方便您現有的監控工具/做法。

監控範圍為

- 事件檢視
  - 影響您會議室的熱門問題
  - 將會議室還原為健康狀態所需的動作
  - Microsoft 正在調查的問題
- 會議室Microsoft Teams的視圖
  - 在裝置層級上MICROSOFT TEAMS 會議室 (的) 狀態快照
  - 每個裝置的基本歷程記錄與詳細資料

**會議室Microsoft Teams的視圖**

- 在裝置層級上MICROSOFT TEAMS 會議室 (的) 狀態快照
- 每個裝置的基本歷程記錄與詳細資料

> [!Important]
> 請 [**閱將使用者指派給受管理服務系統管理員角色**](enrolling-mtrp-managed-service.md#assign-users-to-the-managed-service-administrator-role) ，並根據您的業務需求確定入口網站存取權受到限制。

## <a name="types-of-incidents"></a>事件種類

事件分為兩種嚴重性類型：

- **重要**：有可能導致會議問題的事件，應優先處理。
- **警告** ： 事件是規劃維護動作的通知。 如果這些問題沒有處理，則隨著時間的過去，會議室更容易發生問題。 警告的目的是要給予您規劃及編排支援的時間。

如果一段時間未注意 **，** 警告可能會轉換為「重要」。

## <a name="responding-to-incidents"></a>回應事件

事件分為三種類別：需要動作、待調查或指派給 Microsoft。

### <a name="needs-action-incidents"></a>「需要動作」事件

狀態設定為「需求動作 **」** 的事件會指派您採取矯正動作。

每個這類事件都會有一個動作欄位，其中會提供 Microsoft 的建議動作，如下所示：

![顯示建議的事件動作的螢幕擷取畫面](../media/rooms-monitor-005.jpg)

- 如果您已經採取動作，您可以在回應方塊中以筆記回應事件，然後在張貼前選擇 「指派給 Microsoft」。
- 根據您的評論，通知也可能不正確。 在這種情況下，請提供該意見，並指派回 Microsoft。
- 最後，如果您想要新增批註，為您自己的小組或 Microsoft 小組提供其他內容，請張貼訊息，而不開啟「指派給 Microsoft」。

>[!NOTE]
>您的矯正動作可能會解決此問題，且受管理聊天室監控會清除清單中的該事件。 在上述情況中，您可能無法解決問題，並指派回 Microsoft。 這個問題將會在未來版本中解決。

### <a name="pending-investigation-incidents"></a>「待調查」事件

針對調查中的事件，描述欄位會保留事件、一般原因和解決方法的資訊，這些資訊可能有助於解決特定問題，因此您可以立即採取行動。

### <a name="assigned-to-microsoft-incidents"></a>「指派給 Microsoft」事件

針對指派給 Microsoft 的事件，「動作」欄位會包含計畫或進度之矯正步驟的簡短詳細資料。 這些步驟可能需要與小組共同合作，並需要透過電子郵件/通話完成延伸共同工作。 這些問題一旦解決，就會從入口網站消失，未來將會有追蹤這類事件及其解決的歷程記錄。

![顯示 MS 校正步驟的螢幕擷取畫面](../media/rooms-monitor-006.jpg)

## <a name="rooms-view"></a>會議室視圖

每個裝置都是會議室及其連接周邊的 Proxy。 健康裝置代表健康的房間，不健康裝置代表會議室在會議期間可能會導致問題。 除了事件檢視之外，受管理會議室入口網站也會提供會議室健康情況概觀，並説明您疑難排解裝置詳細資料，並瞭解事件記錄中的重複失敗。

![顯示聊天室健康概觀的螢幕擷取畫面](../media/rooms-monitor-007.jpg)

**健康、不健康、已中斷連接** 「會議室」視圖的上方面板提供您有多少裝置處於良好狀態 (「健康」) 、受問題 (「不健康」) 影響、有多少裝置未提供遙測 (「已中斷」) ，以及有多少裝置會隱藏以替代) 提醒 (。 會議室會使用不斷演進的準則和啟發式來監控健康情況。 目的是盡可能準確地反映聊天室中的使用者體驗實作，並使其具有可操作性。

**健康/不健康的房間**：

沒有任何嚴重性為「重要」事件的裝置/周邊裝置符合目前健康情況的準則，會標示為健康。 不過，這並不表示入口網站中每一個不健康裝置都有會議室中斷。 事件的描述和行動部分包含此問題的更具體詳細資料，以及使用者經驗的潛在影響。

**已中斷連接裝置：**

部署為 Managed Room 試驗的 Microsoft 監控代理程式已從 Managed Room 雲端服務中斷。 我們並未收到有關聊天室的遙測資料，而且沒有最新的健康情況狀態。 這可能是因為網路問題、防火牆政策變更，或是對裝置影像做了變更。
