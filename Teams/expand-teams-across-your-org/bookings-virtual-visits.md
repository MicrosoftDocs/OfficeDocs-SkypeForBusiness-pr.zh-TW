# <a name="virtual-visits-with-microsoft-teams-and-the-bookings-app"></a>使用 Microsoft Teams 和 Bookings 應用程式進行虛擬會面

## <a name="overview"></a>概觀

Bookings[應用程式](https://support.microsoft.com/office/what-is-bookings-42d4e852-8e99-4d8f-9b70-d7fc93973cb5)Microsoft Teams組織為教職員和出席者安排及管理虛擬約會的簡單方法。 使用它來排程約會，例如醫療保健訪問、財務諮詢、面試、客戶支援、虛擬配件和諮詢、教育用辦公室時數等。

此Bookings應用程式可輕鬆地管理任何組織的複雜排程需求。 排程器可以從單一體驗管理多個部門與員工行事曆，以及與內部和外部出席者之間的通訊。

虛擬約會會透過會議Microsoft Teams，提供強大的視訊會議功能。 例如，醫生可以與病患共用螢幕並檢閱測試結果。 或者，銀行顧問可以在檔上要求電子簽章，讓他們遠端關閉交易。

每個虛擬約會Teams一個會議連結，以電子郵件寄給出席者，他們可以輕鬆地從網頁瀏覽器或在任何裝置Teams加入。 自動電子郵件提醒有助於減少未顯示，並提升客戶和客戶的參與度。

有了Bookings，您獲得專為您的產業量身打造的體驗。 以下是一些如何在組織中使用它的範例：

|行業 | 範例 |
|---------|---------|
|金融服務    |  遠端銷售與服務的虛擬訪問<br/>排程及管理銀行關係管理員、財務顧問及理賠調整員的約會，以提升效率與便利性，為客戶提供服務。  |
|零售   | 虛擬試裝和諮詢 <br/>排程及管理銷售人員、產品專家和設計顧問的約會，以與客戶進行虛擬配件和諮詢。   |
|醫療   |  病患照護的虛擬訪問 <br/>為護理小組成員排程及管理約會，以與病患或其他醫療保健提供者開會，討論醫療保健問題。   |

本文提供您如何使用 Bookings 應用程式排程、管理及執行虛擬Teams。

## <a name="before-you-get-started"></a>開始之前

如果您是系統管理員，請參閱在[Teams](../bookings-app-admin.md)中管理 Bookings 應用程式，以瞭解在 Teams 中使用 Bookings 應用程式的先決條件、如何控制組織中 Bookings 的存取權，以及建議的政策和系統管理設定。

請記住，只有貴組織的排程者必須安裝 Bookings 應用程式Teams。 進行或參與虛擬約會的員工不需要應用程式。 他們從自己的會議Teams或Outlook，或在預約確認電子郵件中使用會議連結加入約會。

## <a name="set-up-a-new-booking-calendar"></a>設定新的預約日曆

### <a name="create-the-booking-calendar"></a>建立預約日曆

在 Teams中 **，前往**  >  Bookings **開始**，然後選取新的 **預約日曆**。 完成表單，並請務必為貴組織選擇相關的商務類型。

:::image type="content" source="../media/bookings-virtual-visits-new-booking-calendar.png" alt-text="顯示商務類型的新預約日曆畫面螢幕擷取畫面":::

如果您是較大的組織，如果您希望出席者收到來自特定部門而非整個組織的預約電子郵件，請考慮建立多個預約日曆。
若要深入瞭解，請參閱建立[Bookings。](https://support.microsoft.com//office/create-a-bookings-calendar-921cfd26-a24d-4aca-9004-561594112148)

> [!NOTE]
> 如果這不是您第一次使用 Bookings App，或您想要使用現有的預約日曆，請在 Bookings 中選取組織名稱旁的下拉箭號，然後選擇現有的預約 **日曆**。 您可以在這裡搜尋您想要的。

### <a name="add-staff"></a>新增教職員

在預約日曆中，前往其他 **選項 (...**) >**設定**，然後選取教 **職員**。 新增教職員成員，並將角色指派給每個新增的人員。 您可以新增最多 100 位教職員成員至預約日曆。

應用程式Bookings應用程式與 Outlook。 新增教職員之後，您就能查看該人員的日曆可用性，並排程他們的預約。 若要深入瞭解，請參閱[新增教職員，並Bookings日曆](https://support.microsoft.com/office/add-staff-and-view-a-bookings-calendar-6c579f61-8adb-4514-9458-021de2023fa0)。  

### <a name="create-appointment-types"></a>建立約會類型

建立特定的約會類型來代表貴組織提供的服務，並量身訂做預約體驗。 排程人員可以使用約會類型來排程拜訪。

在預約日曆中，前往其他 **選項 (...**) >**設定，選取** 約會類型，然後選取新增 **約會類型**。**** 輸入名稱， &mdash; 例如帳戶開啟、開單續約、貸款諮詢、稅務準備 &mdash; ，以及您想要的其他資訊和設定。

每次預約這類約會時，您新增的資訊都會包含在寄給出席者的電子郵件確認中。 您可以設定電子郵件提醒和其他選項，例如出席者是否可以從行動瀏覽器加入[](mobile-browser-join.md)，而不需要下載Teams。

如果您是系統管理員，Bookings每次預約此約會類型時，您最多可以連結四個出席者填寫的表單。 例如，您可能會要求出席者在加入流覽之前先完成註冊表單。 若要連結表單，請選擇連結 **表單**。 輸入表單的 URL，**然後選擇連結。**  (如果這是您第一次連結表單，系統會提示您建立一個儲存表單Microsoft 365群組。 選擇 **建立群組** 以建立群組。 您只需要為預約日曆執行一次。) 

使用表單時，請記住：

- 若要變更已連結至約會類型的表單，請選取約會類型中的表單，或選取位於 Microsoft 365 群組中的表單 [https://forms.office.com](https://forms.office.com) 。
- 當所有出席者都來自同一[](https://support.microsoft.com/office/add-questions-that-allow-for-file-uploads-6a75a658-c02b-450e-b119-d068f3cba4cf)個組織時，支援將檔案上傳到包含檔案上傳問題的表單。

當排程者使用約會類型來排程拜訪時，他們可以選擇包含表單、移除表單，或新增您連結至約會類型的其他表單。 出席者必須先填寫表單，才能加入流覽。

若要深入瞭解，請參閱 [建立約會類型](https://support.microsoft.com/office/create-an-appointment-type-810eac77-6a65-4dc8-964d-c00eadf43887)。

## <a name="schedule-a-visit"></a>排程流覽

在預約日曆中，選取新增 **預約**。 選取約會類型，然後填寫相關資訊。

這包括出席者連絡人資訊、將提供服務的員工成員、只有教職員可以看到的內部記事、電子郵件提醒，以及出席者是否可以從行動瀏覽器加入。 如果表單已連結至約會類型，您可以選擇包含、移除該表單，或新增任何其他連結表單。

寄給出席者的電子郵件確認包含會議連結和附件，以便他們新增虛擬約會至他們的日曆。 教職員也會收到電子郵件確認和會議邀請。 如果約會中包含表單，Bookings系統管理員和排程人員可以查看出席者在拜訪前是否已完成表單，並可以查看出席者的回應。

若要深入瞭解，請參閱[在應用程式Teams Bookings預約](https://support.microsoft.com/office/schedule-a-booking-in-the-teams-bookings-app-e275049d-0d0f-4161-8526-461a9f29439f)。

## <a name="conduct-a-visit"></a>進行流覽

在會議Teams或Outlook中，前往預約，然後選取 **加入或Teams** 連結。 檢查您的音訊和視音訊設定，然後選取立即 **加入**。 若要深入瞭解，請參閱進行[Bookings約會](https://support.microsoft.com/office/conduct-a-bookings-appointment-a86a4007-e26c-4909-9893-f7036e2747cd)。

## <a name="monitor-visits-and-get-real-time-status-updates"></a>監控訪問並取得即時狀態更新

Bookings[中的](https://support.microsoft.com/office/queue-view-in-bookings-3eea2840-a1e0-4bcd-8e09-d3cf51c184d6)佇列視圖提供您的教職員儀表板，可監控當天的所有虛擬約會，並即時更新。 若要查看佇列，請前往 Bookings 中的佇列Bookings。

:::image type="content" source="../media/bookings-virtual-visits-queue.png" alt-text="螢幕擷取畫面顯示 Bookings 應用程式中的佇列Teams" lightbox="../media/bookings-virtual-visits-queue.png":::

排程員可以從佇列中新增預約、查看相關的約會詳細資料，以及查看全天的約會狀態。 當病患加入等候室時，狀態會變更，並且會顯示並追蹤他們的等待時間。 該視圖會自動重新使用色彩編碼的更新，以便輕鬆識別變更。

教職員甚至可以直接從佇列加入及管理約會。

> [!NOTE]
> 目前，Bookings應用程式支援每個預約日曆新增最多 100 個教職員。 如果您用Graph API 來設定並新增員工至預約日曆，可能不會強制執行此限制。 在此情境中 **，無法針對** 超過 100 名教職員的日曆呈現內容。 為了獲得最佳體驗，建議您在預約的日曆中新增不超過 100 名員工。 我們正努力在未來版本中解決此限制。

## <a name="additional-capabilities-with-the-bookings-web-app"></a>使用 Web 應用程式Bookings功能

Bookings Web App 提供您其他功能。 例如，您可以發佈自助線上預約頁面，使用者可以在這裡與員工排程約會。 若要存取Bookings應用程式，請前往其他 **選項** (...) >**開啟Bookings應用程式**。

若要深入瞭解，[請參閱Microsoft Bookings。](/microsoft-365/bookings/bookings-overview)

## <a name="get-insight-into-virtual-visits-usage"></a>深入瞭解虛擬訪問的使用狀況

系統[管理中心中的](../teams-analytics-and-reports/virtual-visits-usage-report.md)虛擬Microsoft Teams使用方式報告，提供系統管理員Teams貴組織虛擬流覽活動概觀。 報告顯示虛擬約會的詳細分析，包括Bookings。

您可以查看重要度量單位，例如大廳等候時間和流覽持續時間。 使用這項資訊來深入瞭解使用趨勢，協助優化虛擬訪問，以提供更好的商務成果。
