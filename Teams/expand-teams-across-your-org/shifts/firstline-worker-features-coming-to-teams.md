# <a name="firstline-worker-features-coming-soon-to-microsoft-teams"></a>第一線員工工作人員功能即將推出給 Microsoft 團隊

> [!IMPORTANT]
> 2019年10月1日生效, Microsoft StaffHub 將停用。 我們正在將 StaffHub 功能組建至 Microsoft 團隊。 今天, 小組包含針對排程管理的倒班應用程式, 而其他功能則會隨著時間推移而推出。 StaffHub 將會停止針對2019年10月1日的所有使用者使用。 任何試圖開啟 StaffHub 的人, 都會顯示一則訊息, 讓他們下載小組。 若要深入瞭解, 請參閱[Microsoft StaffHub 停用](microsoft-staffhub-to-be-retired.md)。

以下是即將推出給小組的第一線員工工作人員的功能清單:

- 公司公告
- Kronos 整合
- 工作
- [主畫面] 登陸頁面
- 在團隊中新增自訂欄位

由於團隊中的唯一平臺結構, 小組中將不提供下列功能:

- 流動連接器
- 針對非 Azure Active Directory (Azure AD) 使用者的倒班或 StaffHub 存取權
- StaffHub 的條件式存取
- StaffHub * 的 Intune MAM *
- 為您的組織新增員工資源 * *

* 您將無法管理 Intune MAM 進行 StaffHub 或班次, 但您可以使用團隊管理 Intune MAM 安全性。 若要深入瞭解, 請參閱[小組中安全性與合規性的概覽](../../security-compliance-overview.md)。

* * 在小組中, 您將無法再與您的第一線員工工作力一起散佈共用的員工資源。 您可以改為在團隊中手動新增檔案或網站的索引標籤, 以快速內容存取。 若要深入瞭解, 請參閱[如何使用 [團隊]](https://office365adoption.com/microsoft-teams-tabs/)索引標籤。 或者, 您可以使用 [團隊 Microsoft Graph Api], programattically 將相同的索引標籤新增至任何需要存取相同共用資源的小組。 若要深入瞭解, 請參閱新增索引標籤[至 [頻道]](https://docs.microsoft.com/graph/api/teamstab-add?view=graph-rest-1.0)。

我們承諾組建能滿足您第一線員工工作力需求的功能, 並想要聽到您的意見, 請傳送[UserVoice](https://microsoftteams.uservoice.com/forums/555103-public-preview/category/182881-developer-platform)的意見反應給我們。
