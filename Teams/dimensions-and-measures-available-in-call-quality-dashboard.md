# <a name="dimensions-and-measures-available-in-call-quality-dashboard"></a>通話品質儀表板中提供的維度與量值

Microsoft 團隊和商務用 Skype Online 的通話品質儀表板 (CQD) 可讓您深入瞭解使用 Microsoft 團隊和商務用 Skype 服務所進行的通話品質。 本主題提供 CQD 所公開之維度與量值的詳細資訊。 若要深入瞭解 CQD 及如何啟用它, 請參閱[開啟與使用 Microsoft 團隊和商務用 Skype Online 的通話品質儀表板](turning-on-and-using-call-quality-dashboard.md)。



## <a name="first-and-second-endpoint-classification"></a>第一和第二個端點分類

通話品質儀表板 (CQD) 中的許多維度與量值標示為第一個或第二個。 下列邏輯會判斷資料流程或呼叫中所涉及的端點標示為「第一」:

- 首先會是伺服器端點 (AV MCU、中繼伺服器等)。如果您的伺服器是在串流/通話中參與。

- 第二個將一直是用戶端端點, 除非資料流程位於兩個伺服器端點之間。

- 如果兩個端點都是相同的類型, 則第一個與第二個端點的順序是以使用者代理程式類別的內部排序為基礎。 這可確保順序一致。

### <a name="notes-on-dimension-data-typeunits"></a>[維度] 資料類型/單位的注意事項:

#### <a name="range"></a>大量

以下列格式顯示以範圍或值群組形式提供的尺寸:

 _\<排序次序字串\> [\<下限全形\> - \<(含上\>界排他)]_

例如, [工期] (分鐘) 維度代表以秒為單位的值, 以值為範圍來表示通話持續時間。

|||
|:-----|:-----|
|**持續時間 (分鐘)** <br/> |**如何解讀** <br/> |
|062: [0-0)  <br/> |資料流程持續時間 = 0 分鐘  <br/> |
|064: [1-2)  <br/> |1分鐘 < = 串流持續時間 < 2 分鐘  <br/> |
|065: [2-3)  <br/> |2分鐘 < = 資料流程持續時間 < 3 分鐘  <br/> |
|066: [3-4)  <br/> |3分鐘 < = 資料流程持續時間 < 4 分鐘  <br/> |
|…  <br/> |…  <br/> |

<sort order string>用於在呈現資料時控制排序次序, 並可用於篩選。 例如, [持續時間 (分鐘)] < "065" 的篩選, 會顯示持續時間少於2分鐘的資料流程 (需要使用前導 "0" 才能如期作業)。

> [!NOTE]
> 排序次序字串的實際值並不重要。

#### <a name="enumeration-pair"></a>列舉對

以 [列舉] 對提供的尺寸會以下列格式顯示:

 _\<從某個終點開始列舉值\> : \<從其他端點的列舉值\>_

列舉值的排序是一致的, 但不會反映第一個或第二個端點的排序。

例如, 網路連線詳細資料對會顯示兩個端點的網路連線詳細值:

|||
|:-----|:-----|
|**網路連線詳細資料對** <br/> |**如何解讀** <br/> |
|有線: 有線  <br/> |第一和第二個端點都使用有線乙太網連線。  <br/> |
|有線: wifi  <br/> |第一個端點使用有線乙太網連線和第二個端點使用 Wi-fi 連線, 或第二個端點使用的是有線乙太網連線, 而第二個端點使用 Wi-fi 連線。  <br/> |
|: wifi  <br/> |第一個端點使用 WiFi 連線, 而第二個端點所使用的網路連線是未知的, 或第二個端點使用 WiFi 連線, 而第一個端點所使用的網路連線是未知的。  <br/> |
|…  <br/> |…  <br/> |

#### <a name="blank-values"></a>空白值

上表列出維度可能為空白的可能原因。 如果 QoE 記錄可用維度為 false, 則許多維度與量值將會是空白的。 這通常是在通話未成功建立時發生。
