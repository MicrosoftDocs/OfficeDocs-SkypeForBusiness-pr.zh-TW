# <a name="dimensions-and-measurements-available-in-call-quality-dashboard"></a>[通話品質] 儀表板中提供的尺寸與測量

Microsoft 團隊和商務用 Skype Online 的通話品質儀表板（CQD）可讓您更清楚地瞭解使用這些服務進行通話的通話品質。 本主題提供透過 CQD 顯示之維度與度量的詳細資訊。 若要深入瞭解 CQD，請參閱[開啟與使用 Microsoft 團隊和商務用 Skype Online 的通話品質儀表板](turning-on-and-using-call-quality-dashboard.md)。

## <a name="first-and-second-endpoint-classification"></a>第一和第二個端點分類

CQD 中的許多尺寸與測量值標示為第一或第二筆。 下列邏輯會判斷資料流程或呼叫中所涉及的端點標示為「第一」：

- 伺服器端點（AV MCU、中繼伺服器等）是在資料流程或通話中參與伺服器時，最先考慮。
- 除非資料流程位於兩個伺服器端點之間，否則會視為第二個用戶端端點。
- 如果兩個端點都是相同的類型，則會根據使用者代理類別的內部排序來設定，以確保順序一致。

例如，以下每個資料列代表一個串流中所涉及的一對使用者代理程式：

|:--- |:--- |:--- |:--- |:--- |
|AV-MCU |OC （商務用 Skype 用戶端） |AV-MCU |OC （商務用 Skype 用戶端） |滿足 |
|OC （商務用 Skype 用戶端） |AV-MCU |AV-MCU |OC （商務用 Skype 用戶端） |虛假 |
|OC （商務用 Skype 用戶端） |OC （商務用 Skype 用戶端） |OC （商務用 Skype 用戶端） |OC （商務用 Skype 用戶端） |虛假 |
|AV-MCU |中繼伺服器 |中繼伺服器 |AV-MCU ||
|中繼伺服器 |AV-MCU |中繼伺服器 |AV-MCU |滿足 |
|OC （商務用 Skype 用戶端） |OC 電話（商務用 Skype IP 電話） |OC （商務用 Skype 用戶端） |OC 電話（商務用 Skype IP 電話） |滿足 |
|OC 電話（商務用 Skype IP 電話） |OC （商務用 Skype 用戶端） |OC （商務用 Skype 用戶端） |OC 電話（商務用 Skype IP 電話） |虛假 |

> [!NOTE]
> 第一和第二種分類是獨立于來電者或被叫商的。 第一個是呼叫方維度可用來協助找出哪個端點是呼叫者或被叫方。

### <a name="notes-on-dimension-data-typeunits"></a>[維度] 資料類型/單位的備忘稿

#### <a name="boolean"></a>Boolean

布林值通常是 True 或 False。 在某些情況下，True 也可以表示為1，而 False 則可以表示為0。

#### <a name="range"></a>大量

以下列格式顯示以範圍或值群組形式提供的尺寸：

 _\<排序次序字串\> [\<下限全形\> - \<（含上限）排他\>_

例如，[工期] （分鐘）維度代表以秒為單位的值，以值為範圍來表示通話持續時間。

|持續時間（分鐘） |如何解讀 |
|:--- |:--- |
|062： [0-0） |資料流程持續時間 = 0 分鐘 |
|064： [1-2） |1分鐘 < = 串流持續時間 < 2 分鐘 |
|065： [2-3） |2分鐘 < = 資料流程持續時間 < 3 分鐘 |
|066： [3 – 4] |3分鐘 < = 資料流程持續時間 < 4 分鐘 |
|  | |

\<排序次序字串> 用於在呈現資料時控制排序次序，並可用於篩選。 例如，[持續時間（分鐘）] < "065" 的篩選，會顯示持續時間少於2分鐘的資料流程（需要使用前導 "0" 才能如期作業）。

> [!NOTE]
> 排序次序字串的實際值並不重要。

#### <a name="enumeration-strings"></a>列舉字串

CQD 所使用的字串通常是從資料檔案衍生而來，而且這些字元幾乎是在允許長度內的任何字元組合。 某些尺寸看起來像是字串，但由於它們只能是預先定義值的簡短清單中的其中一個，因此它們是列舉而不是 true 字串。 某些列舉字串也會成對使用。

#### <a name="enumeration-pair"></a>列舉對

以 [列舉] 對提供的尺寸會以下列格式顯示：

 _\<從某個終點開始列舉值\> ： \<從其他端點的列舉值\>_

列舉值的排序是一致的，但不會反映第一個或第二個端點的排序。

例如，網路連線詳細資料對會顯示兩個端點的網路連線詳細值：

|網路連線詳細資料對 |如何解讀 |
|:--- |:--- |
|有線：有線 |第一和第二個端點都使用有線乙太網連線。 |
|有線： wifi |第一個端點使用有線乙太網連線和第二個端點使用 Wi-fi 連線，或第二個端點使用的是有線乙太網連線，而第二個端點使用 Wi-fi 連線。 |
|： wifi |第一個端點使用 WiFi 連線，而第二個端點所使用的網路連線是未知的，或第二個端點使用 WiFi 連線，而第一個端點所使用的網路連線是未知的。 |
| | |

#### <a name="blank-values"></a>空白值

上表列出維度可能為空白的可能原因。 如果 QoE 記錄可用維度為 false，則許多維度和度量將會是空白的。 這通常是在通話未成功建立時發生。
