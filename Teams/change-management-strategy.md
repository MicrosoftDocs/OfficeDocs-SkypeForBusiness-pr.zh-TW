# <a name="create-a-change-management-strategy-for-microsoft-teams"></a>建立 Microsoft Teams 的變更管理策略

對於使用者意識和採用新技術的變更管理策略，對於在組織中成功推行任何新技術而言至關重要。 成功完成這項工作可確保使用者採用和滿意的新體驗。  根據貴組織的規模而定，下列可變更的步驟，但確保與專案關係人、冠軍、IT 系統管理員和使用者持續保持持續的通訊層級，是成功變更的關鍵。  

Teams 變更管理的最佳做法：

1.  找出您重要的專案關係人、冠軍和使用者設定檔

2.  識別&選取您的商務案例

3.  進行包含商務使用者、冠軍和 IT 專業人員的試驗。  

4.  設計、啟動和管理您的採用行銷活動。 下載我們的 [客戶成功套件](./planning-workshop-practical-guide.md) 做為起點。 良好的採用行銷活動包括：

    a.  內部意識資料，例如海報、數位簽章和活動

    b.  單一位置的自助和訓練資訊

    C。  已定義的意見反應機制

    D。  預先定義的成功量值 (解決方案採用、重要資料檢視、課程出席) 

4.  與您的服務部署一起建立冠軍計畫

5.  提供標準的意見反應方法

6.  衡量&分享成功

7.  根據意見反應調整您的訊息和方法，重複這些動作。

## <a name="identify-key-stakeholders-champions-and-user-profiles"></a>找出重要的專案關係人、冠軍和使用者設定檔

成功採用的第一個規則是建立一個由重要專案關係人和適當的人員組成的動態團隊，以推動和影響其他人的變更。 成功採用策略的開頭是由一組致力於代表貴組織跨區段的人員開始。 重要的專案關係人角色包括高階贊助商、服務擁有者、IT 專業人員和冠軍。

## <a name="user-profiles"></a>使用者設定檔

瞭解環境中的使用者類型同樣重要。 您是否有主要行動裝置的使用者？ 有部分人員正在進行常數會議或進行簡報？ 您知道哪些使用者最難以使用現有的共同作業解決方案嗎？ 區段您的使用者社群可協助您尋找最開放變更的群組。 它們通常是您早期商務試驗的最佳目標，而他們的意見反應極為有價值。

瞭解使用者生命週期將可協助您排定業務成果的優先順序、適合您部署的設計採用目標，以及隨著時間持續使用。

## <a name="champions"></a>冠軍

識別可以成為共同作業之星的人員，可為您提供實作的延伸支援小組。 為您的計畫提供早期深入見解和意見反應，以建立社群。 您在此社群中所做的任何投資，無論是時間、注意力或獎勵，都會透過他們的支援和宣傳，回到您的實作中。

若要瞭解如何建立冠軍計畫，請取得 [「建立冠軍計畫」](https://go.microsoft.com/fwlink/?linkid=854665)。

## <a name="identify-and-select-your-business-scenarios"></a>找出並選取您的商務案例

一旦您找到重要的專案關係人和使用者區段後，識別您的商務案例會變得更容易。 以下是一些商務成果範例：

-   簡化銷售程式

-   客戶問題的快速解決方式

-   增加員工參與度

-   縮短產品開發週期

您的銷售人員無法進行協調以提供絕佳的體驗嗎？ 您是否需要在提供產品或服務時進行更好的協調？ 人力資源 () 員工的相關體驗呢？ 每個組織都會找到自己的優先順序。 我們建議您在初始試驗中選取不超過三個以解決此問題，並包括這些特定組織的使用者和商務專業人員。 

也建議您考慮這些案例中內嵌的使用者。 它們是否已開啟以進行變更？ 他們是行動使用者還是在您的設施內？ 他們離支援他們之高階贊助者和冠軍的樞紐有多近？他們是否有任何現有的社群和通訊方式，您可以使用這些方法來宣傳這項變更？  

當人們很少或完全不注意人為變更管理需求時，技術專案就會變得難以解決。 遵循這些建議，您將提高成功的機會！

## <a name="design-launch-and-manage-your-adoption-campaign"></a>設計、啟動和管理您的採用行銷活動

對於初始試驗和全公司推行的正式發行，您的內部通訊應成為優先順序。 這些專案包括：

-   內部意識資料，例如海報、數位簽章和活動。

-   將自助和訓練資訊整合在單一位置。

針對您的試驗階段，以下是成功的最低步驟：

-   定期與專案專案關係人開會進行更新。

-   挑選意見反應方法，從參與試驗的人員收集資訊。 較小的組織可以使用 Teams 內的公用頻道，讓使用者加入並提供意見反應。 在大型組織中，考慮啟動 Yammer 社群以提供產品支援。  

-   提供自助檔，包括產品影片。

-   與試驗使用者舉行啟動會議，讓他們對參與感到興奮。 建立充滿熱情的社群意識是無法被底線的。

一旦您完成初始試驗階段後，上述步驟便可擴充至整個組織。 視您的大小而定，這可能需要一些時間，而且會依地區、使用者設定檔或組織階段進行調整。 在這個規模較大的推行中，運用貴組織中溝通者的專業人才是非常實用的。 我們建議這些個人在您運用 [可用的採用資料](https://go.microsoft.com/fwlink/?linkid=854598) 或自行設計時，提早參與並經常參與。

如果貴公司有新聞、資訊或支援的中央內部網路入口網站，您可以利用這個中心來取得這項推出的資訊。提供廣泛可用的自助資訊、訓練和書面指導方針，可讓使用者快速登入產品。 許多使用者只要在可用後就跳入，我們鼓勵您這麼做。 我們也知道每個人都能以不同的方式學習，因此擁有中央資訊入口網站將支援貴組織內的所有學習樣式。

如同採用行銷活動&任何通訊一樣，您會想要在最前面找出您的成功衡量點。 考慮量值，例如：

-   產品中的作用中使用者

-   資訊網頁的檢視

-   使用者社群中的問題

-   訓練影片的檢視

-   出席學習活動

## <a name="measure-and-share-success"></a>衡量並分享成功

在您的專案小組中，應該是您組織的 IT 系統管理員。 這些個人將能存取內建于Microsoft 365 系統管理中心體驗中的 Microsoft 365 或Office 365 Microsoft Teams 使用方式報告。 此資訊可以顯示您公司中 Microsoft Teams 的使用方式。 我們也建議您直接從您的使用者取得有關其 Teams 體驗的意見反應，以及他們對於使用的其他想法。 與您的使用者社群分享來自您公司內部的這些真實範例，對於隨著時間持續地改變，是非常有價值的。  

在Microsoft 365 系統管理中心中Microsoft 365 系統管理中心[Microsoft Teams 使用方式報告中深入瞭解 Microsoft Teams 使用方式報告](/microsoftteams/teams-analytics-and-reports/user-activity-report)。

> [!TIP]
> 使用您的公用小組或 Yammer 社群來收集使用方式的深入解析。 使用#TeamsStories標籤共用，以便輕鬆擷取。  在內部網路上每月共用#TeamsStory，以醒目提示接受變更以提升您環境中生產力及共同作業的員工。  

## <a name="ongoing-adoption"></a>持續採用

在組織中推動變更是進行中的活動，而不是活動。 您的啟動會議只是讓人員更有效效率地共同作業的開頭。 請務必規劃此旅程中強調成功、介紹新功能，以及與整體 Microsoft 365 或Office 365服務策略保持聯繫，這是 IT 部門會打造的。 確保您的使用者參與而非不知所措，可減少他們改變的疲勞，並支援他們放開舊習慣，例如透過電子郵件寄送附件等，以產生新事物，例如將檔儲存在雲端。

在 Microsoft 技術社群網站上的[「推動採用](https://techcommunity.microsoft.com/t5/Driving-Adoption/ct-p/DrivingAdoption)」社群中，分享有關 Teams 與 Microsoft 365 或 Office 365 服務採用的問題。 我們一直希望聽到您的心聲，並協助您進行採用之旅。
