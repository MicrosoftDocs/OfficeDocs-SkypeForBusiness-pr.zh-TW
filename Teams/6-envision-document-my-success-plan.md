# <a name="document-my-success-plan"></a>記錄我的成功計畫

本文概述正確記錄雲端語音部署的需求。 在規劃雲端語音部署時，定義並記錄所有決策點和下一個步驟，以確保所有專案關係人與專案小組成員都一致，以取得成功的結果。 

## <a name="execution-planning"></a>執行規劃 

定義在組織中使用通話方案解決方案電話系統音訊會議或音訊會議的方法之後，您需要規劃執行執行專案。

如果貴組織只有一或兩個網站，您可能不需要完成本文提供的所有詳細資料，但您應該閱讀本文以引導您的方法。

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>部署模型 

就像任何改變組織人員工作方式的技術實現一樣，選擇適當的部署方式將會大幅影響您的雲端語音實現的成功。

可能的部署模型包括下列專案：

-   **每個網站：** 此模型適用于貴組織地理位置分散且分公司員工人數相當多的情況。 不過，此部署模型可能會中斷部門員工分散在數個位置的部門內通訊。

-   **每個部門**：此模型通常是中型企業的更好選項，並確保相關部門擁有相同的體驗。

-   **一次全公司：** 此模型通常是小型企業的最佳選項，所有員工從部署的第一天起，都獲得相同的體驗。

## <a name="quarterly-execution-plan"></a>每季執行計畫

若要在可管理的工作區塊中執行雲端語音部署，建議您根據目標關鍵結果 (OKRs) 、您所選擇的部署模型，以及貴組織的專案執行能力，建立每季執行計畫。

如此一來，您可以每季追蹤進度、必要時修改計畫，以及根據貴組織的執行能力部署雲端語音功能。

如果貴組織只有一或兩個網站，您可能不需要每季執行計畫，因為您預期會在短時間內完全部署。

## <a name="finalize-my-success-plan"></a>完成我的成功計畫

成功計畫是在構想階段建立的檔摘要。

成功計畫為專案小組提供足夠資訊 ，讓專案小組 ，包括 FastTrack 或部署合作夥伴，充分的資訊，以使用通話方案服務來電話系統音訊會議或電話電話系統達成貴組織的目標。

一般而言，成功計畫包含下列主要區段，許多章節會透過構想階段進行：

-   商務案例

-   服務整備

-   服務決策

-   執行計畫

-   採用方案

-   操作方案

### <a name="business-case"></a>商務案例

商務使用案例、專案關係人清單、OKRs 和 KSIs、風險登記簿和專案時程表一般是商務案例所需的大部分資訊。 您應該將這些檔記錄為成功計畫的一部分。

### <a name="service-readiness"></a>服務整備

您的環境評估提供判斷貴組織是否準備好使用通話方案進行音訊會議和/或電話系統所需的初始資訊。

此處包含您的服務整備狀況評定，以及解決您透過環境評定所發現之需要補救之地區的計畫。

### <a name="service-decisions"></a>服務決策

記錄您規劃音訊會議或電話系統通話方案服務技術實現。

### <a name="execution-plan"></a>執行計畫

記錄您規劃專案的執行方式，以在整個組織中執行解決方案。

### <a name="adoption-plan"></a>採用方案

執行採用就緒性評定之後，專案小組必須針對啟動前、啟動和啟動後採用活動，提供一組完整的通訊計畫、訓練計畫及計畫。

找出支援採用活動的資源，例如傳單、歡迎電子郵件及訓練教材，以及任何您需要的自訂專案，以滿足貴組織的需求。

從客戶成功套件下載[Microsoft Teams採用活動的範本](https://aka.ms/TeamsCustomerSuccess)。

### <a name="operational-plan"></a>操作方案

執行操作角色的繪圖會建立角色與職責，以及指派給每個操作角色的團隊，這些團隊必須支援音訊會議的執行工作。

您必須完成這項作業，並納入營運計畫，做為成功計畫的一部分，以確保解決方案的營運準備狀態。
