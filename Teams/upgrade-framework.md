# <a name="about-the-upgrade-framework"></a>關於升級架構

為了協助您排除升級旅程的猜測，我們採用了實作變更的實證架構。 如下所示，架構中的每一個步驟都是在前一步建立，為了獲得最佳結果，建議您依序執行步驟。  

首先，將適當的專案關係人整合在一起，並定義升級計畫 (例如範圍、目標和時程表) 。 有了計畫，請確認您的技術環境和使用者已準備好進行Teams。 然後，分階段實作升級，準備就緒時從試驗升級到整個組織的升級。 一旦貴組織開啟Teams，請建立監控品質的營運計畫，並加快使用者採用速度。

![升級旅程架構的圖例。](media/upgrade-banner-main.png "請確定您的專案已與正確的專案小組一起設定成功。定義專案範圍、目標和時程表。確認技術與使用者整備。執行您的推行計畫。保持最大化結果的動力。")

在相關頁面上尋找此架構圖形，以識別您在升級程式中的位置。

## <a name="sample-upgrade-timeline"></a>升級時程表範例

升級之旅會在您開始規劃變更時開始。 以下利用成功架構做為指導方針，是一個範例時程表，可引導您從升級前階段開始規劃並準備升級，包括升級和升級之後的營運階段，其設計目的是維持和擴充您的成果。 

> [!NOTE]
> 我們瞭解，您Teams之旅可能涉及在不同時間運用多種[模式](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)和升級使用者群組，這可讓您控制使用者升級體驗，同時保持Teams的動力。  

為了協助示範您的升級旅程如何符合您的需求，我們提供了下方的範例計畫，其中定義從 商務用 Skype Online 到島嶼模式到僅Teams的旅程。 此外，範例計畫概述已將其使用者分成四個升級群組或一組的組織。 使用此範本做為範本，自訂計畫以包含您前往Teams的特定旅程，並結合您將使用的各種[模式](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)，以及您將使用者分成的升級群組數目。 

## <a name="pre-upgrade"></a>升級前

**為貴組織準備Teams**。 為了協助確保順利升級至Teams，請務必配置適當的準備時間。 貴組織不僅能夠快速開始瞭解Teams的價值，您也可以在Teams準備好時立即加速從商務用 Skype升級。 如果您已同時啟用Teams商務用 Skype，請使用這些升級前活動做為檢查點，以在您將使用者升級至Teams之前驗證貴組織的整備狀態。 

> [!TIP]
> 除了範例升級專案計劃和試驗測試計劃之外，還可下載範本使用者整備資料的 [升級成功套件](https://aka.ms/UpgradeSuccessKit) ，例如通訊和使用者問卷。 套件中可用的專案在下列清單中會以星號 (*) 標示。

## <a name="upgrade"></a>升級 

**正式移至Teams**。 當您升級使用者時，您將使用者移至僅Teams模式。 Teams成為聊天、會議、通話和共同作業的主要應用程式，且商務用 Skype應用程式的存取權已停用。 雖然此階段的技術層面相當簡單，但請考慮變更對使用者體驗的影響，並讓使用者有時間正式將其活動從商務用 Skype轉換為Teams。 若要減少使用不同用戶端之不同體驗的使用者，請嘗試將端對端升級視窗限制為 45 天左右。

## <a name="post-upgrade"></a>升級後

**使用Teams最大化商務價值**。 在貴組織完全升級至Teams之後，請花時間根據您的目標評估您的成功，並實作計畫以繼續前進動力。
