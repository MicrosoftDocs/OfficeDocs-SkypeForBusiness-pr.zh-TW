# <a name="steering-committee"></a>指導委員會

## <a name="steering-committee-overview"></a>指導委員會概觀

指導委員會是一組重要的專案關係人和專案負責人，他們彙聚在一起，以引導專案或計畫達成其定義的商業成果。 指導委員會對於專案的傳遞方式概不負責，而是專案交付給企業的內容。**

每個專案都需要一致同意的願景和架設。 若要傳遞專案所需的成果，必須清楚定義願景，並加以監控與維護。 這會成為指導委員會的責任：推動決策、提供建議、提供策略性監督、擔任專案計劃的組織客服員，以及在必要時移除封鎖程式。

您的組織應該為指導委員會進行大量思考。 委員會必須確保專案達成您為推動整個組織變更所定義的業務目標、定期開會討論專案的目前脈衝，並協助移除旅程中所遇到的任何障礙。

委員會應定義其圖表以包含一些主要目標：

-   讓專案小組與執行贊助者或執行領導人保持緊密的一致。

-   向高階贊助商或執行領導人提供專案狀態的深入解析。

-   允許執行贊助者或執行領導小組為專案提供方向和意見，並透過調整專案計劃、目標關鍵結果 (OKRs) 及其他專案活動，確保專案符合整個商務目標。

指導委員會會在專案的整個生命週期內開會，以確保組織領導與專案小組之間能保持一致。 這個重要會議可確保專案方向具有領導者的完整支援，並將領導人員提供的任何意見反應整合到專案中，以推動成功。 委員會會使用這些會議來深入瞭解專案狀態，以及：

-   同意符合商務案例的業務成果，並確保專案正推動這些成果的傳遞。

-   檢查及核准專案，以取得商務案例的正確性與合規性。

-   檢閱並確認對商務案例所做的變更，可能會影響任何已定義的結果。

-   針對專案交付專案的優先順序做出策略性決策，並核准臨時交付專案。

-   找出、管理及減輕間距、風險，以及委員會需要額外影響的問題。

-   針對需要升級、優先處理及解決專案關係人業務單位之間的衝突的問題，收集高階贊助者或執行領導小組的支援。 

-   如有適用，請提供正式的意見反應和建議給主管領導、變更建議面板或其他商務和 IT 專案關係人。

## <a name="identify-members-of-the-steering-committee"></a>識別指導委員會的成員

委員會只有其參與者具影響力才能生效。 管理委員會必須同時維護商務與技術決策者的代表性，以及參與解決方案的任何核心專案小組領導和合作夥伴團隊。

每個指導委員會都需要強大的領導才能和明確定義的角色和職責。 若要設定委員會績效及其成員參與的實際期望，您必須記錄並同意指派的角色和職責。 您絕對必須在此委員會中擁有回應靈敏的執行贊助者和商務領導，才能在整個組織中提供有效影響，並確保各種不同的觀點。

> 根據您的組織和專案小組設計來調整角色和責任。 在較小的組織中，這些角色可能會重迭。

## <a name="schedule-steering-committee-meetings"></a>排程指導委員會會議 

雖然每個組織和每個專案小組的結構略有不同，但指導委員會會議不應只是另一個專案待命會議。 考慮指導委員會會議的目標：藉由獲得高階贊助者或高階主管領導的支援和指示，讓專案步入正軌是您的機會。 請確定每個人都參加此會議，讓整個專案小組與執行贊助者或執行領導人之間能直接傳送訊息，而訊息轉送時也不會出現溝通不順暢的情形。

排程委員會會議時，請考慮專案和組織的頻率。 請根據主管贊助者的喜好設定和貴組織的步調，嘗試每週或每隔一周排程一次。

> [!TIP]
> 如果您的會議環境非常快速，請不要讓太多時間在會議之間經過，因為這可能會導致更頻繁的會議所避免的問題。 不過，在步調較慢的環境中，每週召開一場會議可能會導致會議疲勞，進而降低指導委員會的有效性。 使用您的經驗和判斷判斷適當的頻率，並記得您可以變更排程。 別為了開會而開會。

### <a name="prepare-for-the-meeting"></a>準備會議

若要充分利用每個人的時間，您必須為會議做準備。 核心小組可以開始準備指導委員會會議的材料。 您可以建立PowerPoint簡報，以協助準備會議。

下列資產應完成並包含在會議的簡報組中：

-   後續追蹤先前的委員會動作專案

-   指導委員會PowerPoint，專案高層級狀態報表

    -   從每月/每季/每年輸入檢閱查看專案。

    -   找出重要的專案里程碑：

        -   討論里程碑的重要性與影響 (這些里程碑應該在風險登錄) 中擷取。

        -   根據準備來決定即將到來的移轉，尤其是商務決策者。

    -   標注專案執行日期和高影響力活動。

    -   對於委員會認為重要的 OKR 報告;請務必包含使用量、使用者情緒、品質及可靠性指派。

-   指導委員會PowerPoint、專案每週狀態報表

    -   包含醒目提示和低光燈、重大問題和動作專案，以及後續步驟。

    -   包含更專注于日常問題的技術詳細資料，並減少整體專案狀態。

-   問題記錄檔和 [風險登錄](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)

    -   分享委員會正在追蹤之任何問題的狀態。

        -   準備好提供任何和所有錯誤的更新。

    -   找出任何可能會影響企業成功成果的新專案。

        -   找出它們對專案的影響。

        -   找出受影響的個人或業務單位。

        -   找出後續步驟，以及可能需要委員會成員的協助。

-   品質概觀：使用[準備網路以進行Teams](prepare-network.md)的指導方針。

    -   請準備好深入說明這些品質報告的結果，並提供建議以補救所記錄的任何問題。

    -   調整每週的品質報告，以著重在下欄區域：

        -   組織的整體品質趨勢

        -   已知問題網站或發生問題的網站的狀態，以顯示網站的標題方向，並在理想情況下回報補救工作的正面結果

        -   根據專案進度最近啟用的網站品質狀態

接下來，安排通話，為指導委員會會議做準備。 此通話僅應包含核心專案小組中至關重要的小組成員，這可能包括來自網路、桌面、作業或其他小組的資源，而這些小組的專案將在委員會會議期間處理。 您打算在指導委員會通話期間涵蓋的任何資料，不應該讓核心專案小組成員措手不及。

這項通話是指導委員會開會之前進行同步處理的絕佳方式。 這也是確保專案小組中任何開啟專案、不一致或任一問題被曝光的時候了。 這是為了協助促進開放溝通，並防止任何人在會議期間感到十分驚訝。

與核心小組及合作夥伴專案關係人檢閱該周的報告，並解決任何疑慮或問題。 請在小組內根據您的討論來調整、新增或移除任何內容，或同意向指導委員會要求其他說明。 指派將展示每個節的小組成員，以協助確保流程和連續性。

### <a name="hold-the-meeting"></a>主持會議 

會議本身應該是核心小組所完成準備工作的自然延續。 請記住，這些會議的目的是要營造溝通和共同作業的氣氛，以達成專案的願景。 請使用下列建議來協助確保會議成功：

1.  排定面對面會議。 找出會議室，其中的表格夠大，讓每個人都坐在一旁。 確定會議室具備視訊功能。 雖然最好讓會議室中的每個人都加入會議室，但並非一定都能使用，因此請確定會議邀請包含至少音訊的會議室和線上會議資訊，但最好是音訊、視訊及共用。 鼓勵所有遠端參與者在加入會議時使用視訊。

2.  請嘗試繼續處理議程，但還是會出現需要解決的干擾問題。 如果可能 (視其重要性) ，請將這些專案新增至「車位」區段，稍後再處理。 如果主題太重要而無法延後，請同意繼續討論，但發現您必須將一些其他議程專案移至未來的會議。 同意撥出一些時間 (5 到 10 分鐘，) 處理任何需要委員會立即注意的按下專案。

3.  準備好討論任何開啟的支援案例。 請盡可能為本會議保持討論的高層級，包括票證數量、票證類型、主要開放專案。 您應該準備好解決所有問題，即使這些問題是次要問題，也請瞭解對業務的影響以及對專案的影響

4.  委派記事記錄器。 在您發表簡報或進行熱烈的熱議時，要記錄筆記太難了。 請確定已針對委員會中的任何決策記錄詳細資料。 準備好將這些決定和記事作為後續電子郵件的一部分傳送出去。

5.  如有) ，請一律嘗試宣傳有關人員工作努力的好消息，以及使用者的意見反應 (。 訴說並銷售故事。 如果意見反應是負的，請準備好討論意見反應，並說明解決負面使用者體驗問題所要採取的動作。

    a.  利用機會為個別小組成員提供回饋，讓他們有良好的工作。 請務必辨識小組成員，尤其是在其領導小組前完成的出色、硬工。

6.  請務必儲存會議結束時間，以同意後續任何步驟，並識別在會議期間所同意之動作專案的擁有者。

7.  傳送包含您所建立之簡報的出席者後續追蹤電子郵件，並附加任何記錄。 請務必包含未參加的任何人。

## <a name="next-steps"></a>後續步驟

檢閱指導委員會的概觀之後，現在該採取行動了。專案小組應該開始他們的工作，以將指導委員會實作到專案的時程表，並準備一般的指導委員會會議頻率。 在理想情況下，在專案開始之前會先成立委員會，並在整個階段提供指導方針，但如果情況並非如此，請成立委員會以引導專案小組取得成功的成果。

1.  找出指導委員會成員。 向您自己詢問：

    -   我們是否有來自企業與領導人員的正確標記法？

    -   我們識別的成員是否能夠影響其業務單位或其他單位的同儕？

2.  建立通訊以傳送給指導委員會的每位成員：

    -   包含角色資料表。

    -   出售委員會的重要性。

3.  設定第一個會議來討論角色與職責、定義委員會之召集人，以及驗證或定義 OKR。 議程專案也可以包括：

    -   討論會議頻率。

    -   驗證該函數和 OKRs 正在推動有意義的商業成果。

    -   確認該策略與 OKR 一致。

4.  在每一個交談中開始提高駕駛品質：

    -   闡述擁有高品質之冠軍的重要性。

    -   找出將擔任「品質之冠軍」角色的人員或團隊。

管理委員會成功並非幸運。 選擇具有最佳效能的適當成員，仍可能產生不佳的結果。 若要協助推動指導委員會的成功，您需要確保您擁有下列專案：

-   專案一致同意的願景和範圍，其中包含委員會將提意的明確、可衡量的商業成果。

-   組織完善的議程可推動有目的的會議。

-   開放溝通的一種環境。

    -   開放式溝通和共同作業將能讓委員會更上一層賦予權力。 建立一個可以聆聽所有語音並尊重意見的環境。 獨特的觀點可以帶來最大的影響。

    -   當領導者採取行動促進成員共同作業和通訊時，可以啟用開放的環境。 以下是一些指導方針：

        -   所有委員會成員都應該是提供給組織之變更的強力支援者。

        -   將委員會大小限制為絕對需要提供正確圖片的人員，以及能夠在各種業務單位中做出或影響決策的人員。 當封鎖程式出現時，瞭解組織的階層和每個成員的直接影響線將非常重要。

        -   與專案經理和小組負責人一起參與委員會討論和工作。 請準備好邀請其他核心小組成員，以清楚瞭解特定領域可能需要的深入討論。

在解決方案的生命週期期間，委員會的角色將會變更。 如預期和上線移至磁片磁碟機值階段，委員會可能需要進行幾項調整：

-   調整會議的頻率。 查看組織的動態，並判斷會議是否需要較不頻繁地進行。 當組織變更開始減少上層通訊的頻率時，可能會減少。 如果適當，請調查將會議變更為每月或每季會議的頻率

-   調整會議中的代表。 當解決方案進入磁片磁碟機值階段時，核心專案小組很可能會重新指派給新專案。 您必須更新回報為支援情況的小組，而作業將會成為解決方案狀態的潛在客戶。

-   針對營運 OKR 定義和報告。 主要結果和成功指標將會隨著解決方案的生命週期而改變。 您需要的業務成果應該會繼續追蹤並回報。 繼續回報在推動企業成果的情況下，語音解決方案的品質及可靠性。 其他指標和關鍵結果，例如使用者情緒、使用者採用和支援指標，將會成為成功的首要指標。

-   使用較大的運動委員會保護套來涵蓋多個解決方案的磁片磁碟機值階段。 尋找能更有效率地使用指導委員會的機會。 使用委員會提供跨更大領域解決方案的指導方針。
