# <a name="steering-committee"></a>指導委員會

## <a name="steering-committee-overview"></a>指導委員會概觀

指導委員會是由重要專案關係人與專案領導者所共同管理，共同引導專案或計畫取得已定義之業務成果的一個管理群組。 指導委員會不直接負責專案的交付方式，而是專案給企業提供的專案。 

每個專案都需要一致願景和章程。 若要交付專案想要的結果，必須清楚定義願景，必須監控及維護願景。 這會成為指導委員會的責任：推動決策、提供意見、提供策略性監督、為組織宣傳專案計劃，並在必要時移除封鎖者。

貴組織應該對組成指導委員會進行重大思考。 委員會必須確保專案達到您為推動整個組織進行變更所定義的商業目標，定期開會討論專案的目前脈動，並協助移除在過程中遇到的任何障礙。

委員會應定義其章程，以納入一些關鍵目標：

-   在專案小組與主管贊助者或主管領導之間保持一致。

-   向主管贊助者或主管領導提供專案狀態的深入資訊。

-   允許執行贊助者或主管領導小組為專案提供方向和意見，並調整專案計劃、目標關鍵結果 (OKRs) 及其他專案活動，確保專案符合整體的業務目標。

指導委員會會在整個專案生命週期中週期性開會，以確保組織領導與專案小組之間保持一致。 此重要會議可確保專案的方向獲得領導階層的完全支援，並將領導階層提供的任何意見回饋納入專案中，以推動成功。 委員會使用這些會議來深入瞭解專案狀態，以及：

-   同意與業務案例一致的商務成果，並確保專案能夠推動這些成果的交付。

-   檢查並核准專案是否正確且符合商務案例。

-   檢查及驗證對商務案例進行變更，可能會影響任何已定義的結果。

-   針對專案交付專案的優先順序做出策略決策，並核准臨時交付專案。

-   找出、管理及減少需要委員會額外影響之差距、風險及問題。

-   針對需要升級、優先處理及解決專案關係人業務單位之間任何衝突的問題，向執行贊助者或主管領導小組收集支援。 

-   在適用時，向主管領導階層、變更顧問委員會或其他企業與 IT 專案關係人提供正式的回饋和建議。

## <a name="identify-members-of-the-steering-committee"></a>識別指導委員會的成員

只有參與者具有影響力，委員會才能有效率。 指導委員會必須維持來自企業與技術決策者的代表性，以及參與解決方案的任何核心專案小組領導和合作夥伴團隊。

每個指導委員會都需要強大的領導才能，以及明確定義的角色與責任。 若要設定對委員會績效及其成員參與的實事求是的預期，您必須記錄並同意指派的角色與職責。 您絕對必須在這個委員會中擁有回應迅速的主管贊助人及企業領導，才能在整個組織中提供有效影響力，並確保有各種不同的觀點。

## <a name="schedule-steering-committee-meetings"></a>排程指導委員會會議 

雖然每個組織和每個專案小組的結構都稍有不同，但指導委員會會議絕不應變成另一個專案獨立會議。 考慮指導委員會會議的目標：這是您收集主管贊助者或主管領導人的支援與指示，讓專案保持進度的機會。 請確定每個人都參與此會議，讓整個專案小組與執行贊助者或主管領導之間能直接傳送訊息，且轉遞訊息時不會傳達錯誤訊息。

排程委員會會議時，請考慮到專案和組織進度。 根據您的主管贊助者喜好和貴組織的速度，嘗試每週或每隔一周排程。

> [!TIP]
> 如果您的環境是一個快速移動的環境，請不要在會議之間花太多時間，這可能會導致較常開會可避免的問題。 不過，在速度緩慢的環境中，每週召開一次會議可能會導致會議疲勞，這會降低指導委員會的效率。 使用您的經驗與判斷來判斷適當的步頻，並永遠記住您可以變更排程。 不要為了開會而開會。

### <a name="prepare-for-the-meeting"></a>準備會議

若要充分利用每個人的時間，您必須為會議做好準備。 核心小組可以開始準備指導委員會會議的材料。 您可以建立PowerPoint簡報，協助準備會議。

下列資產應已完成，並包含在會議簡報區中：

-   對先前的指導委員會動作專案進行後續追蹤

-   指導PowerPoint，專案高層級狀態報表

    -   從每月/每季/每年查看專案的所有內容，並輸入評論。

    -   識別重要的專案里程碑：

        -   討論里程碑的重要性和 (，這些應記錄在風險) 。

        -   根據準備，尤其是與商務決策者一起，決定進行即將進行的移轉到/否移轉到。

    -   找出專案執行日期和影響較高的活動。

    -   已識別為委員會重要之 OKRs 的報告;請務必包含使用方式、使用者情緒、品質和可靠性度量。

-   指導委員會PowerPoint，專案每週狀態報表

    -   包含高亮和低照度、重大問題和行動專案，以及下一個步驟。

    -   包含更多專注于日常問題的技術詳細資料，而較少關注整體專案狀態。

-   問題記錄與 [風險註冊](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)

    -   分享委員會正在追蹤的任何問題的狀態。

        -   準備好提供任何及所有錯誤更新。

    -   找出任何可能會影響成功達成商務成果的新增專案。

        -   找出他們對於專案的影響。

        -   識別受影響的個人或業務單位。

        -   找出下一個步驟，以及可能需要來自成員哪些協助。

-   品質概觀：使用準備[您的網路以Teams。](prepare-network.md)

    -   準備好深入說明這些品質報告的結果，並包含建議以補救所發現的任何問題。

    -   調整每週的品質報告，以專注于下欄區域：

        -   組織的整體品質趨勢

        -   已知問題網站或發生問題的網站狀態，以顯示網站標題的方向，並報告補救工作的正面結果

        -   根據專案進度，最近啟用的網站品質狀態

接下來，排程通話以準備指導委員會會議。 此通話應只包含核心專案小組的重要小組成員，這可能包括來自網路、桌面、作業或其他團隊的資源，這些團隊的專案將在委員會會議期間處理。 核心專案小組成員絕不應受到您計畫在指導委員會通話期間涵蓋的任何資料所遮住。

這個準備式通話是一種在指導委員會開會前進行同步處理的很好方式。 這也是確保專案小組中任何未開啟的專案、不一致或不滿情緒曝光的時候。 這是為了協助促進開放式溝通，避免任何人在會議期間感到意外。

與核心小組和合作夥伴關係人一起審查本周的報告，並解決任何疑慮或問題。 在小組中根據您的討論調整、新增或移除任何內容，或同意向指導委員會要求其他說明。 指派將呈現每個節的小組成員，協助確保流程與持續性。

### <a name="hold-the-meeting"></a>召開會議 

會議本身應該是核心小組所做的準備工作的自然延續。 請記住，這些會議的目標是促進溝通和共同合作的環境，以達成專案願景。 請使用下列建議，協助確保會議成功：

1.  將會議排程為面對面。 找出一個會議室，其表格夠大，每個人都可以坐在一起。 確保會議室具有視音訊功能。 雖然讓每個人都在會議室中是理想的選擇，但並非一定可行，因此請確保會議邀請包含會議室和至少含有音訊的線上會議資訊，但最好是音訊、視像和共用。 鼓勵所有遠端參與者在加入會議時使用視像。

2.  請儘量保留議程，但肯定會出現需要解決的干擾。 如果可能 (視其重要性) ，請新增這些專案到「駐車地」區段，稍後再處理。 如果主題太重要，無法延後，請取得一致以繼續討論，但請注意，您必須將一些其他議程專案移至未來的會議。 同意撥出一些時間 (五至十分鐘) 處理任何需要委員會立即注意的急迫專案。

3.  準備好討論任何開啟的支援案例。 嘗試讓此會議的討論保持在盡可能高的水準，包括票證數量、票證類型、主要開啟專案。 您應該準備好解決所有問題，即使問題很小，也請瞭解對業務的影響及對專案的影響

4.  委派記事員。 在進行報告或進行激烈爭論時，記筆記太難了。 請確認詳細資料已記錄在委員會中做出的任何決定。 準備在後續電子郵件中傳送這些決策和筆記。

5.  請一直嘗試宣傳有關人員工作投入和使用者意見反應的正面消息， (提供) 。 講述並銷售故事。 如果意見反應完全為負面，請準備好討論，並說明處理負面使用者體驗問題所執行的工作。

    a.  請利用這個機會，為個別小組成員提供他們所做的良好工作。 當小組成員完成出色的、勤勞的工作時，尤其是他們的領導小組面前時，一點很重要。

6.  永遠儲存會議結束時間，以同意任何下一個步驟，並找出會議期間所同意之動作專案的擁有者。

7.  傳送包含您建立之簡報的出席者後續電子郵件，並附加任何記錄。 請務必包含未參加的任何人。

## <a name="next-steps"></a>後續步驟

在審查指導委員會概觀之後，現在該採取行動了。專案小組應開始其工作，將指導委員會納入專案時程表，並準備一般指導委員會會議進行。 在理想情況下，委員會會先在專案開始之前組建，並在其所有階段提供指引，但如果情況並非如此，請組成委員會來引導專案小組取得成功的結果。

1.  識別指導小組成員。 詢問您：

    -   我們是否擁有來自企業與領導階層的合適代表？

    -   我們識別的成員是否能夠影響其他單位的業務單位或同儕？

2.  建立通訊傳送給指導委員會的每個成員：

    -   包含角色資料表。

    -   銷售委員會的重要性。

3.  設定第一個會議以討論角色與職責、定義委員會章程，以及驗證或定義 OKRs。 議程專案也可以包括：

    -   討論會議頻次。

    -   確認章程和 OKRs 正在推動有意義的商務成果。

    -   確認策略與 OKRs 一致。

4.  開始每一個交談的駕駛品質：

    -   表達擁有品質優等獎的重要性。

    -   找出將擔任品質優等獎人角色的人或小組。

指導委員會的成功不是靠運氣。 選擇適當的成員 ，並擁有最佳意圖，仍然會產生不佳的結果。 若要協助推動指導委員會的成功，您必須確保您擁有下列專案：

-   為專案達成一致願景和範圍，並擁有明確定義、可測量的商業成果，這是委員會所宣傳的。

-   井井有條的議程，可推動有目的的會議。

-   開放通訊的氣息。

    -   開放式溝通和共同合作將賦予委員會權力。 建立一個能聽取所有聲音並尊重意見的環境。 獨特的觀點可以產生最大的影響。

    -   當領導者採取促進成員共同合作與溝通的作用時，可以啟用開放式環境。 以下是一些指導方針：

        -   所有成員都應該是組織所推行之變更的強項宣傳者。

        -   將委員會規模限制在絕對需要提供正確圖片的人，以及可以跨業務單位做出或影響決策的人。 瞭解組織的階層和每個成員的直接影響線，在封鎖者出現時至關重要。

        -   讓專案經理和小組領導者參與委員會討論和討論。 準備好邀請其他核心小組成員，以清楚說明特定領域可能需要的深入討論。

在解決方案的生命週期中，委員會的角色將會變更。 當構想和上移進入磁碟機值階段時，可能需要對委員會進行數項調整：

-   調整會議頻次。 查看組織的動態，並判斷會議是否需要較少發生。 隨著組織變更開始減少，高層次溝通的頻率可能會減少。 調查將會議變更為每月或每季的會議步調

-   調整會議代表。 當解決方案進入磁碟機值階段時，核心專案小組很可能會重新指派給新專案。 需要更新報告為支援的團隊，而營運會成為解決方案狀態上的潛在客戶。

-   定義及針對可營運的 OKRs 進行報告。 關鍵結果和成功標記將在解決方案的生命週期中變更。 應該繼續追蹤並報告想要的商業成果。 繼續報告語音解決方案的品質和可靠性，以取得業務成果。 其他度量和關鍵結果 ，例如使用者情緒、使用者採用和支援量度，會成為成功的主要指示器。

-   使用較大的轉向委員會保護套，以涵蓋多個解決方案的磁碟機值階段。 尋找更有效率地使用指導委員會的機會。 使用委員會在更大的解決方案領域提供指引。
