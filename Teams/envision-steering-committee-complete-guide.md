# <a name="steering-committee"></a>籌畫指導委員會

## <a name="steering-committee-overview"></a>籌畫指導委員會概覽

「指導委員會」是一組主要的專案關係人，也就是專案領導人，我們已將專案或計畫導向其定義的商業成果。 指導委員會不會直接負責專案的交付 *方式* ， *而是將* 專案傳達給業務。

每個專案都需要商定的願景與章程。 若要傳送專案的預期結果，遠景必須明確定義，它必須受到監視及維護。 這會成為籌畫指導委員會的責任：若要推動決策、建議、提供戰略監管，以充當專案計劃的組織，以及在必要時，請移除封鎖器。

貴組織應該將重要的想法放入籌畫指導委員會。 該委員會必須確保專案達到您在整個組織中所定義的業務目標，並定期開會以討論專案目前的脈衝，並協助您移除在旅程期間遇到的任何障礙。

委員會應該定義其宗旨，以包括一些主要目標：

-   在專案小組與主管主管或高層領導之間保持緊密的關聯性。

-   深入瞭解專案對於主管主管或主管領導的狀態。

-   可讓主管主管或高層主管小組針對專案提供方向與輸入，並確保其符合商業目標，只要調整專案方案、目的金鑰結果 (OKRs) 及其他專案活動。

籌畫指導委員會在專案的整個生命週期中都符合週期性間隔，以確保組織領導與專案小組之間的關聯。 這項重要的會議可確保專案的方向具有領導能力的完整支援，並將由領導提供的任何意見反應納入專案中，以促進成功。 該委員會會使用這些會議，深入瞭解專案狀態，也可以執行下列動作：

-   同意與業務案例相符的商業結果，並確保專案能夠傳送這些結果。

-   檢查並核准專案，以保證其準確性並符合商業案例的規範。

-   審查並驗證對業務案例所做的變更，可能會影響任何已定義的結果。

-   針對專案交付優先順序進行戰略決策，以及核准暫時交付專案。

-   找出、管理及減輕從委員會所需要的其他影響力所需的缺口、風險及問題。

-   從管理層主管或主管領導小組收集客戶支援，以解決需要上報的問題、排定風險承擔者企業單位之間的任何衝突並加以解決。 

-   提供正式的意見反應和建議給高層領導、變更建議委員會，或其他業務和 IT 干係人（如果適用的話）。

## <a name="identify-members-of-the-steering-committee"></a>找出籌畫指導委員會的成員

委員會只與參與者有影響的人一樣有效。 籌畫指導委員會必須維持來自業務與技術決策者的代表，以及參與方案的任何核心專案小組領導和合作夥伴小組。

每個指導委員會都需要強大的領導能力和明確定義的角色和職責。 若要設定委員會效能及其成員參與的實際預期，您必須對指派的角色和職責進行記錄並同意。 您絕對必須在這個委員會上具備回應性的主管及業務領導能力，才能為整個組織提供有效的影響力，並確保不同的觀點。

## <a name="schedule-steering-committee-meetings"></a>排程指導委員會會議 

雖然每個組織與每個專案小組的結構略有不同，但您永遠不應該只成為另一個 project standup 會議。 考慮指導委員會會議的目標：您可以透過從總經理主管或主管領導層，收集客戶支援和方向，協助您保持專案順利進行。 請確認每個人都出席此會議，讓整個專案小組與主管或主管領導之間有直接訊息，而且郵件是在中繼郵件時沒有 miscommunication。

排程委員會會議時，請考慮專案與組織節奏。 根據您的主管的喜好設定和組織節奏，嘗試每週或每隔一周排程一次。

> [!TIP]
> 如果您是一種快速移動的環境，請不要在會議之間花費太多的時間進行，這可能會導致會議更加頻繁地避免問題。 不過，在進度較慢的環境中，召開會議的每週都可能導致會議 fatigue，這可能會降低指導委員會的效能。 使用您的經驗與 judgement 來判斷適當的節奏，並隨時記住您可以變更排程。 不要僅僅為了進行會議而要開會。

### <a name="prepare-for-the-meeting"></a>準備會議

若要充分利用每個人的時間，您必須準備好進行會議。 核心小組可以開始準備用於籌畫指導委員會會議的內容。 您可以建立 PowerPoint 簡報，協助準備會議。

下列資產應該已完成，且包含在簡報投影片的會議中：

-   前一本指導委員會行動專案的後續追蹤

-   籌畫指導委員會 PowerPoint，專案高層次狀態報表

    -   從每月/每季/每年類型的審閱中查看專案。

    -   找出主要專案里程碑：

        -   討論里程碑 (應該在風險登記) 中捕獲這些里程碑的重要性與影響。

        -   根據準備，決定是否要使用 [移至]/[不執行] 遷移，特別是在企業決策者中。

    -   撥出項目執行日期和高影響力的活動。

    -   針對委員會所識別為重要的 OKRs 報告;請務必包含用法、使用者觀點、品質與可靠性指標。

-   籌畫指導委員會 PowerPoint，project 的每週狀態報表

    -   包含醒目提示與 lowlights、主要問題、動作專案，以及後續步驟。

    -   包含更多針對日常問題的技術詳細資料，並不會降低整個專案的狀態。

-   問題記錄和 [風險登記簿](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)

    -   共用該委員會所追蹤之任何問題的狀態。

        -   準備好提供任何及所有錯誤的更新。

    -   找出可能會影響取得業務結果成功的任何新專案。

        -   找出其對專案的影響。

        -   找出受影響的個人或公司單位。

        -   識別後續步驟，以及委員會成員可能需要的協助。

-   品質概覽：使用指導方針 [為小組準備您的網路](prepare-network.md)。

    -   準備深入說明這些品質報告的結果，並加入修正任何所述問題的建議。

    -   每週調整品質報告，以將焦點集中在下欄區域：

        -   組織的整體品質趨勢

        -   已知問題網站或發生問題的網站的狀態，以顯示該網站的標題方向，也就是 [理想]-從修正工作報告正面結果

        -   以專案進度為基礎的最近啟用網站品質狀態

接下來，排程通話以準備進行籌畫指導委員會會議。 這個通話應只包含核心專案小組所不重要的小組成員，這可能包含來自網路、桌面、作業或其他團隊的資源，這些專案在委員會會議期間將會解決這些問題。 核心專案小組成員永遠不應受到任何您打算在籌畫指導委員會通話時所涵蓋的任何內容的防護。

這個準備通話是在籌畫指導委員會進行同步處理的絕佳方式。 這也是確保專案小組中任何開啟的專案、disagreements 或 grievances 的時間，都是一種小的時機。 這是協助鼓勵開啟的通訊，並防止任何人在會議期間 unpleasantly 驚訝。

複習該周的報表與核心小組與合作夥伴的專案關係人，並解決任何擔心或問題。 在小組中工作，根據您的討論來調整、新增或移除任何內容，或同意要求籌畫指導委員會進一步澄清。 指派將提供每個區段的小組成員，協助確保流程與連續性。

### <a name="hold-the-meeting"></a>召開會議 

會議本身應該是核心小組完成的 [準備] 工作的自然接續。 請記住，這些會議的目標是鼓勵溝通與共同作業，以達到專案的願景。 請使用下列建議，協助確保會議成功：

1.  安排面對面會議。 找出會議室，其中的表格要足夠大，讓每個人都能坐在周圍。 確定會議室有視頻功能。 雖然您可以將所有人都放在聊天室中，但這並不是很好的做法，所以請確定會議邀請包含會議室，以及至少有音訊的線上會議資訊，但最好、音訊、影片和共用。 鼓勵所有遠端參與者加入會議時，使用影片。

2.  請嘗試保留議程，但不會發生任何需要解決的干擾。 如有可能 (視其重要性) 而定，請將這些專案新增至 [停車場] 區段，並在稍後進行處理。 如果主題太重要，無法繼續進行討論，但請注意，您需要將其他議程專案移至未來的會議。 同意設定一段時間 (5 到10分鐘) ，以解決任何需要委員會立即注意的按下專案。

3.  準備好討論任何開啟的支援案例。 嘗試將討論保持在此會議的高層次：票證數量、票證類型、主要開啟的專案。 您應該準備好解決所有問題，即使它們很輕微，瞭解對企業的影響，以及對專案的影響。

4.  委派記事。 在您進行簡報時，或在您對人對人較直熱的情況下，記筆記很困難。 請確定已針對在委員會中所做的任何決策記錄詳細資料。 準備好將這些決策和記事傳送成後續電子郵件的一部分。

5.  如果有) 的話，請務必先宣傳與人員的工作成果和使用者意見反應 (的積極消息。 講述並銷售該故事。 如果意見反應完全是否定的，請準備好進行討論，並說明已完成的動作，以解決消極的使用者經驗問題。

    a.  您可以利用商機，將 kudos 給個別小組成員，以取得他們所做的工作。 我們必須認識的是，在小組成員（尤其是領導團隊成員）完成時，請務必辨識。

6.  永遠都要儲存會議的結束，以同意任何後續步驟，並識別會議期間已商定之動作專案的擁有者。

7.  傳送包含您所建立之簡報的出席者後續電子郵件，並附加所拍攝的任何筆記。 請務必包含任何沒有參與的人員。

## <a name="next-steps"></a>後續步驟

在審查指導委員會的概況之後，現在可以開始採取行動。專案小組應該開始進行工作，將籌畫指導委員會實施到專案的時程表，並準備週期性指導委員會會議節奏。 理想的是，在專案開始前會形成委員會，並在整個階段提供指導方針，但如果不是這樣，請在委員會中，將專案小組指引至成功的結果。

1.  識別指導委員會成員。 自問：

    -   我們有正確的商業和領導表現方式嗎？

    -   我們認識到的成員是否能夠在其他單位中影響其營業單位或對等？

2.  建立通訊以傳送給籌畫指導委員會的每個成員：

    -   包含角色資料表。

    -   宣傳委員會的重要性。

3.  設定第一個會議來討論角色和職責、定義委員會的宗旨，以及驗證或定義 OKRs。 [議程] 專案也可以包含：

    -   討論會議節奏。

    -   確認您的章程與 OKRs 都正在推動有意義的商業成果。

    -   確認策略已與 OKRs 對齊。

4.  在每個交談中開始駕駛品質：

    -   闡明擁有品質擁護者的重要性。

    -   找出將參與品質擁護者角色的人員或小組。

我們好運不會發生籌畫指導委員會成功的情況。 選擇正確的成員（具有最佳意圖），仍會產生不佳的結果。 為了協助推動籌畫指導委員會的成功，您必須確保您具備下列專案：

-   已商定的專案，以及該委員會將會 advocating 之明確定義且可衡量之業務結果的專案範圍。

-   組織良好的議程，可促進 purposeful 會議。

-   開放式通訊的大氣。

    -   開放式通訊與共同作業將可為委員會進行準備。 建立一個可聽取所有聲音並遵守意見的環境。 獨特的觀點會受到最大的影響。

    -   在領導作用是促進成員共同作業及通訊時，可以啟用開啟的環境。 以下是一些指導方針：

        -   所有的委員會成員對組織所帶來的變更而言，都應該有有力的主張。

        -   將委員會的大小限制為絕對需要的人員，以提供正確的圖片，以及可在整個企業單元中產生或影響決策的人員。 瞭解組織的階層，以及當封鎖者顯示時，每個成員的直接影響行為都是至關重要的。

        -   在委員會討論與 deliberations 中，讓專案經理和小組領導參與。 準備好邀請其他核心小組成員，讓特定區域中可能需要的深入討論更清楚。

在方案的整個週期中，委員會的角色會變更。 隨著構想與加入將移至 [磁片磁碟機] 值階段，委員會可能需要進行數個調整：

-   調整會議的節奏。 看看組織的動態，並判斷會議是否需要不太頻繁。 隨著組織的變更開始 dwindle，可能會降低高層通訊的頻率。 如果適用，請調查將會議變更為每月或每季會議節奏

-   調整會議中的代表。 隨著方案移至 [磁片磁碟機] 價值階段，可能會將核心專案小組重新指派給新的專案。 需要更新報告為支援的小組，且作業將會成為解決方案狀態的潛在客戶。

-   針對操作 OKRs 的定義和報告。 關鍵結果和成功指示會在方案的週期中變更。 您應該繼續追蹤並報告所需的商業結果。 在推動商業結果的內容中，繼續報告語音解決方案的品質與可靠性。 其他衡量標準與重要的結果（例如，使用者觀點、使用者採納及支援度量）會成為成功的最佳指示器。

-   使用較大型的指導委員會傘來涵蓋多個解決方案的磁片磁碟機價值階段。 尋找更有效率地使用籌畫指導委員會的機會。 使用委員會來提供更多較大的解決方案球體的指導方針。
