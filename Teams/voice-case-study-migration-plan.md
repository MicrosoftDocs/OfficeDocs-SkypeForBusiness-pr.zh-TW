# <a name="contoso-case-study-teams-upgrade-plan"></a>Contoso 案例研究：Teams升級計畫

在決定要從商務用 Skype移轉到Teams時，Contoso 想要為使用者提供輕鬆的轉換體驗。 與其同時將所有人切換到Teams，他們決定設定混合式連線能力，並使用重迭功能方法將使用者移至Teams。 這允許Teams和商務用 Skype內部部署的使用者共用目前狀態和通訊。 當使用者輸入電話系統試驗時，他們已移至 [僅Teams模式]。

若要瞭解有關升級、方法和模式的基本概念，Contoso 請閱讀下列文章：

- [開始升級您的 Microsoft Teams](upgrade-start-here.md)
- [適用于 IT 系統管理員的升級策略](upgrade-to-teams-on-prem-implement.md) 
- [移轉與互通性指導方針](migration-interop-guidance-for-teams-with-skype.md)
 
Contoso 也參與 Ignite 2019 會話[，設計從商務用 Skype到Teams的路徑](https://myignite.microsoft.com/archives/IG20-OD251)。 Contoso 已瞭解：

- 互通性、同盟和升級行為等基本概念 

- 以 TeamsUpgradePolicy 為基礎的共存模式和管理 

- 使用者體驗： 

  - 聊天和通話 

  - 會議排程 

  - Teams用戶端中的共同作業功能可用性 

若要規劃和設定混合式連線能力，將內部部署環境移至雲端的第一步，Contoso 閱讀 [規劃混合式聯](/SkypeForBusiness/hybrid/plan-hybrid-connectivity) 機能力和設定 [混合式聯](/SkypeForBusiness/hybrid/configure-hybrid-connectivity) 機，以瞭解如何： 

  - 將其內部部署環境服務設定為與Office 365同盟。 

  - 設定他們的內部部署環境以信任Office 365，並啟用與 Office 365 共用的 SIP 位址空間 

  - 在其Office 365租使用者中啟用共用 SIP 位址空間。

  - 在技術試驗期間使用群島模式。

  - 在使用者啟用電話系統後，將使用者切換到 TeamsOnly 模式。 通話方案和直接路由需要 TeamsOnly 模式。
