# <a name="prepare-my-users"></a>準備我的使用者

本文提供準備使用者使用雲端語音服務的需求概觀。 正確準備使用者後，您就會確保為貴組織帶來最大的價值。

<!--ENDOFSECTION-->

## <a name="preparing-for-user-adoption"></a>準備使用者採用

在您保護專案小組並定義專案的範圍和目標之後，下一個步驟就是確保組織與使用者準備就緒，這是一項與技術整備同時完成的活動。 若要從部署中實現價值，使用者必須實際使用及採用。 只要啟用解決方案並無法保證您能夠達成目標。 此外，使用者有不同的使用案例和不同的學習樣式，而且會以不同的速度來適應新技術。 好消息是，管理變更並不那麼複雜，但確實需要專注努力。 以下包含的引導式討論是專為協助您瞭解使用者基礎，並準備適當的教育程度，以促進和加速使用者採用所設計。

活動擁有者：採用變更管理員做為潛在客戶、Project主管、主管贊助人、行銷/訓練/支援管理員

> [!NOTE] 
> 下列提供的評定活動可適用于貴組織的任何變更計畫。 只要根據專案範圍來回答問題。 在這些活動的討論中，「新解決方案」可適用于「音訊通話」、「音訊會議」，或「從商務用 Skype升級Teams。

<!--ENDOFSECTION-->

## <a name="create-your-vision-to-value-statement"></a>建立願景到價值陳述

設定專案願景可讓您的主要專案關係人看到全域。 這通常等同于企業推動者，例如成本節省、營運效率，以及提高生產力，並符合您先前在專案中設定 &mdash; &mdash; 的目標。 對於使用者來說，同一個專案可能會被視為已取走熟悉的工具、生產力中斷，以及管理缺乏愛心。 花一點時間讓自己熟悉使用者，將專案願景轉換成吸引他們的價值陳述。 在通訊和訓練活動中運用這個價值陳述，讓使用者Teams如何為使用者工作。

### <a name="what-percent-of-users-fall-into-each-bucket"></a>每一個資料桶的使用者百分比是百分比？

| **使用者類型**                                                             | **%** |
|------------------------------------------------------------------------------|-------|
| *早期採用者* 在解決方案提供之前先要求解決方案。                  |       |
| *通知使用者* 一旦顯示其值，即接受解決方案。    |       |
| *落後者* 拒絕解決方案，即使被推入變更。                 |       |


> [!TIP]
> 早期採用者是試驗部署及支援計畫很好的候選者。 如果您有一大部分的使用者，請花一些時間專注于為這些使用者傳遞價值訊息，並舉手協助他們進行變更。

### <a name="what-percent-of-users-fall-into-each-competency"></a>每個能力的使用者百分比是什麼？

| **學習者類型**                                                        | **%** |
|----------------------------------------------------------------------------|-------|
| *自我推動的* 學習者會尋找資源，學習方式也一樣。                         |       |
| *團隊玩家享受* 群組和互動式訓練;他們會和同事一起合作。 |       |
| *握持求職者* 需要「白色手套」或一對一協助。                   |       |

 
> [!TIP]
> Learning大小不一樣。 瞭解使用者如何學習最佳，您可以建立強大的訓練計畫，協助加快他們的學習速度。 如果有許多使用者是自我推動的，連結至影片和使用者指南可能就夠用。 對於需要特殊注意的使用者，請務必排程一對一的時間，或請您的領軍者協助這些使用者完成 &mdash; 學習曲線。

## <a name="assess-your-user-base"></a>評估您的使用者基礎

人員層級分析可讓您瞭解使用者角色、群組及使用案例。 只要花時間瞭解使用者是誰，以及使用者如何工作，就可以確保他們擁有成功執行變更所需的專案。 這可能包括目標訊息、訓練，甚至是耳機和網路網路等裝置。 例如，主管可能會預期進行一對一訓練，而出差的現場代表可能會受益于他們空閒時可以觀看的訓練影片。 此外，您的人員層級分析可協助識別誰應該立即開始使用新功能，以及哪些人最好等候。

<tr><td><img src="media/audio_conferencing_image9.png" alt="An icon depicting the next steps"/><br/>後續步驟</td><td><ul><li>地圖使用者角色。 首先，將使用者群分成人員/群組 (主管、道路鬥士、共同合作工作者、零售商等) 。 為了易於管理，請列出不超過八個群組。  (針對範例交談入門者，請參閱下方。) 使用您的發現來進一步設計您的使用者準備活動，並針對可能受益于日後升級的使用者細分試驗使用者。
</li><li>地圖使用案例。 在您找出將受益于這項新技術的核心使用者群組之後，請考慮您的一般使用者群組。 瞭解跨小組的使用案例和案例，您可以視覺化您的組織使用此新技術。 此外，在通訊和訓練計畫中新增使用案例範例，確實有助於讓使用者瞭解這些範例。</li></ul></td></tr>
</table>

針對每個人員群組，與您的專案小組討論下列事項：

-   這個群組的核心特性 (，也就是說，什麼是典型的「一天」生活) ？

-   此群組會如何受益于新解決方案？ 它們需要哪些功能有效？

-   他們需要哪些裝置 (，例如耳機或喇叭) ，而且它們已經擁有嗎？

-   若要在這個群組中建立開始使用新解決方案之需求，還需要哪些額外的價值訊息 ？。

-   此群組接收的最佳方式是什麼：

    -   認識 (例如電子郵件、海報、部門主管) ？

    -   訓練 (例如線上訓練、使用者指南、白手套) ？

    -   支援 (自助、服務台、支援) ？  

-   這個群組適合試驗或早期採用者計畫嗎？ 如果是這樣，請制定計畫，向這些使用者取得購買，以參與試驗。 有關成功推行試驗計畫之詳細資訊，請參閱試驗Teams[試驗商務用 Skype。](pilot-essentials.md)

下圖是一個人物形象表的範例。

![列出人員詳細資料的值表格](media/prepare-my-users-image2.png "列出角色及其共同角色的表格，以及通訊和共同合作解決方案相依性。主管人員形象會以詳細資料 (個人檔案) ，以及主管的需求、裝置及訓練。")

針對每個使用案例，請與您的專案小組討論下列事項：

-   此使用案例/案例的核心特性是什麼？

-   目前解決方案有什麼用？

-   您目前的解決方案會面臨哪些挑戰 ？如果有的話，您面對的是什麼挑戰？

-   新解決方案如何優化使用者體驗？

有關使用案例的範例，請流覽[FastTrack 文件庫](https://fasttrack.microsoft.com/microsoft365/productivitylibrary)。

## <a name="optimize-your-readiness-plan"></a>優化您的準備計畫
瞭解使用者是誰，以及使用者如何在新解決方案中工作之後，您可以設計您的準備計畫。 您的使用者準備計畫會納入您試驗、上載及操作新解決方案時，如何溝通、訓練及支援使用者。 如本文上述步驟所發現，準備狀態並非一成多。 從廣泛的認知和訓練計畫開始，納入您在此程式開始時定義的價值訊息。 然後根據您的定義，為您定義的角色或群組量身訂做您的訊息、訓練和支援。 您的個人化方案可讓使用者更快速地與新解決方案聯繫，同時證明您瞭解他們的需求，這有助於協助他們轉換。
