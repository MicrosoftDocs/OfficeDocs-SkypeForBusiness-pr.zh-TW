# <a name="define-my-success"></a>定義我的成功

本文將說明如何為您的組織定義成功部署音訊會議、包含通話方案的電話系統或電話系統直接路由的需求。 透過正確定義成功的外觀，您可以在進行部署時測量您的結果，並確認您所要的結果是您所要的結果。

<!--ENDOFSECTION-->

[**音訊會議**] 可透過使用傳統的有線電話、私人分支 EXCHANGE （PBX）或行動電話，透過撥入，讓會議參與者透過公用交換電話網絡（PSTN）加入，以提供其他會議的其他進入點。 當召集人或參與者不在電腦上，或當資料連線無法使用或無法支援語音通訊（例如在含 spotty 行動資料覆蓋率的遠端區域，或連線至免費、公用 Wi-fi）時，這會很有用。使用有限頻寬的服務，或當會議參與者想要使用可隨時存取的電話端點撥入會議時。

**含有通話方案的電話系統（「通話方案」）** 可讓組織在其電腦和行動裝置上進行與業務相關的電話通話，提供一種方式來現代化工作場所。 工作場所的現代化可以是任何幾種案例的一部分：以活動為基礎的作業實現、主要的辦公室移動、office 的重新整理、停用舊版 PBX 解決方案、PSTN 服務提供者合約的結論等。 使用通話方案，Microsoft 可協助連接 PSTN。

**[電話系統直接路由] （「直接路由」）** 可為組織提供與呼叫方案相同的優點，除了是由協力廠商提供者（而不是 Microsoft）來加速 PSTN 連線。 這可讓您在無法使用通話方案的國家或地區進行部署，或是在需要維護現有 PSTN 服務提供者合約，或需要與特定內部部署系統進行互通性的部署中進行。 還有一個考慮直接路由的其他案例，就是電話系統互通性。 在將使用者轉換成團隊通話時，某些使用者可能會繼續舊版 Pbx。 直接路由可讓兩種使用案例共存。 舊版系統上的使用者與團隊使用者之間的通話流量，在組織中保持不變。

<!--ENDOFSECTION-->

## <a name="define-business-use-cases-for-audio-conferencing-calling-plans-or-direct-routing"></a>定義音訊會議、通話方案或直接路由的商務使用案例

首先，核心專案的干係人需要定義支援音訊會議、通話方案或直接路由的業務使用案例。

商務使用案例是要定義並記錄預期和可衡量的商業結果，包括下列各項：

-   目前業務程式的描述

-   現有業務程式面臨的挑戰

-   技術可以如何協助克服這些難題

-   若要解決這些難題，預期和可衡量的商業結果

> [!TIP]
> 以下是音訊會議的完整商務使用案例範例：
> 
> |         |
> |---------|
> |**目前業務程式的描述**<br>Contoso 目前依賴于與外部參與方進行的內部會議和會議，由與委任當地電話服務提供者所提供的 PSTN 會議服務，在會議紀要中計費。|
> |**現有業務程式面臨的挑戰**<br>Contoso 在目前的 PSTN 會議服務中每年約 USD1 萬萬美元，而內部會議產生成本的75%。 使用傳統的電話端點來加入由 PSTN 會議服務主持的會議，不會與組織將團隊採用新式通訊與共同作業平臺的方案對齊。|
> |**技術可以如何克服這些難題**<br>透過採用 Microsoft 團隊作為現代化的通訊與共同作業平臺，內部使用者將能利用配備優化耳機和會議室裝置的電腦來加入會議。 音訊會議服務可提供給外部參與者，或支援電腦音訊使用不適用於內部參與者的情況。|
> |**預期、可衡量、商業結果**<br>隨著新式通訊與共同作業平臺，將小組移至團隊，將會大大降低傳送 PSTN 會議服務的成本。|

<br>

> [!TIP]
> 以下是通話方案的完整商務使用案例範例：
> 
> |         |
> |---------|
> |**目前業務程式的描述**<br>Contoso office 工作區的標準設定包含每個書桌的桌面電話。 每個員工都已獲直接撥出電話號碼。 桌面電話已連線至 PBX 系統，並透過會話初始通訊協定（SIP）主幹來連線至 PSTN。 員工只能在其指派的桌面手機撥打及接聽電話。|
> |**現有業務程式面臨的挑戰**<br>桌面手機的使用狀況分析顯示，只有10% 的桌面電話可供使用，而 rest 則會設定為將來電轉接到行動電話，或同時撥打至行動電話。 維護現有的 PBX 系統和相關的桌面電話，可為 Contoso 的每月電話語音成本貢獻20%。|
> |**技術可以如何克服這些難題**<br>通話方案可利用原生 Microsoft 團隊應用程式，讓使用者的個人電腦在資料網路上接收並撥打電話。 這樣就不需要推出及維護桌面手機，而且會開啟停止使用現有 PBX 系統的機會，因為手機服務可以透過雲端在網路上傳送，而在傳統的電話系統上沒有相依性。|
> |**預期、可衡量、商業結果**<br>移除維護需求以及解除舊版 PBX 和桌面手機將可減少20% 的每月電話語音費用。 通話方案可簡化 office 工作區，允許 Contoso 透過建立具有最低前期電話成本的新辦事處來擴充其運作。|

> 以下是直接路由的完整商務使用案例範例：
>

> |---------|

> |**技術可以如何克服這些難題**<br>SIP 中繼提供者合約最近已簽署，並將在三年內生效。 [直接路由] 可讓 SIP 中繼提供者提供 PSTN 連線，也可利用原生 Microsoft 團隊應用程式，讓使用者的個人電腦在資料網路上接收並撥打電話。 這樣就不必再推出及維護桌面電話，並開啟可讓現有 PBX 系統退役的機會，同時維持有限的內部部署會話邊界控制器（SBC）需求量。|
> |**預期、可衡量、商業結果**<br>移除維護需求以及解除舊版 PBX 和桌面手機將可減少20% 的每月電話語音費用。 直接路由可簡化 office 工作區，允許 Contoso 以最少的前期電話成本建立新的辦公室來擴充其運作。|

## <a name="define-okrs-ksis-and-risks"></a>定義 OKRs、KSIs 和風險

在專案干係人組建時，您可以將商務用案例、組織範圍和專案時程表轉換成目標和主要結果（OKRs），而專案成功的度量可以定義為重要的成功指示（KSIs）。

在定義 OKRs 和 KSIs 中，完全參與來自專案的專案關係人，以協助確保他們感覺擁有權，並將這些成功尺度與組織的業務需求相符。

OKRs 包含您在專案開始時所設定的目標，而且您會以季度為單位定義可測量的金鑰結果。 您每月都會查看金鑰結果，以追蹤整個專案的狀態，並根據進度來調整季度方案（視情況而定）。

-   特定：針對特定區域進行改進

-   可測量：量化或至少建議的指示、進度

-   可賦值：指定要執行的工作

-   真實：指出可實際達到哪些結果、給定可用的資源

## <a name="establish-a-steering-committee"></a>建立籌畫指導委員會

「指導委員會」是一組主要的專案關係人，也就是專案領導人，我們已將專案或計畫導向其定義的商業成果。 指導委員會不會直接負責專案的交付*方式*，*而是將*專案傳達給業務。

每個專案都需要商定的願景與章程。 若要從專案中傳送您想要的結果，遠景必須明確定義，而且必須受到監視及維護。 這會成為籌畫指導委員會的責任：若要推動決策、建議、提供戰略監管，以充當專案計劃的組織，以及在必要時，請移除封鎖器。

貴組織應該將重要的想法放入籌畫指導委員會。 該委員會必須確保專案能夠達到您為整個組織所做的變更所定義的公司目標，定期開會以討論專案目前的脈衝，並協助解除封鎖出現在其中的任何障礙一下.

委員會應該定義其宗旨，以包括一些主要目標：

-   在專案小組與主管主管或高層領導之間保持緊密的關聯性。

-   深入瞭解專案對於主管主管或主管領導的狀態。

-   讓總經理主管或高層主管小組為專案提供方向與輸入，並透過調整專案方案、目標金鑰結果（OKRs）及其他專案活動，以確保其符合企業目標。

籌畫指導委員會在專案的整個生命週期中都符合週期性間隔，以確保組織領導與專案小組之間的關聯。 這項重要的會議可確保專案的方向具有領導能力的完整支援，並將由領導提供的任何意見反應納入專案中，以促進成功。 該委員會會使用這些會議，深入瞭解專案狀態，以及：

-   同意與業務案例相符的商業結果，並確保專案能夠傳送這些結果。

-   檢查並核准專案，以保證其準確性並符合商業案例的規範。

-   審查並驗證對業務案例所做的變更，可能會影響任何已定義的結果。

-   針對專案交付優先順序進行戰略決策，以及核准暫時交付專案。

-   找出、管理及減輕從委員會所需要的其他影響力所需的缺口、風險及問題。

-   從管理層主管或主管領導小組收集客戶支援，以解決需要上報的問題、排定風險承擔者企業單位之間的任何衝突並加以解決。 

-   提供正式的意見反應和建議給高層領導、變更建議委員會，或其他業務和 IT 干係人（如果適用的話）。
