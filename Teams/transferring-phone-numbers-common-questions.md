# <a name="transferring-phone-numbers-common-questions"></a>傳送電話號碼常見問題

以下是將電話號碼轉接至 Microsoft 團隊的常見問題。 查看答案之後，您應該已準備好建立埠順序並傳送您的電話號碼。 如需相關指示，請參閱[將電話號碼轉接至 Office 365](transfer-phone-numbers-to-office-365.md) 。
  
## <a name="what-countries-or-regions-support-number-porting"></a>哪些國家或地區支援數位移植？

您可以在所有支援的國家或地區中移植或傳送電話號碼，但提交埠順序要求的方式取決於電話號碼來自哪個國家或地區。 您可以查看國家/地區的清單，其中列出[音訊會議與通話方案](country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans.md)所支援的國家和地區。 

當您執行電話號碼管理工作（例如轉移（移植）號碼或取得無法在商務用 Skype 系統管理中心中使用的電話號碼時，請參閱[管理貴組織的電話號碼](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)。
  
## <a name="what-numbers-can-be-transferred"></a>哪些號碼可以轉移？

 **您可以傳送：**
  
一般來說，您可以從支援的提供者轉移任何電話號碼，包括：
  
- 土地線路電話號碼。

- 行動裝置電話號碼，例如用於行動電話和平板電腦等。

    > [!NOTE]
    > 轉移行動電話號碼只適用于美國和波多黎各。 
  
- 付費電話號碼。

- 免付費電話號碼。

    > [!NOTE]
    > 通用國際 Freephone 號碼（UIFN）無法傳送給我們。 
  
- 服務電話號碼，例如用於會議橋接、自動語音應答等。

- 傳真呼叫號碼，但無法用來進行傳真。 必須將他們指派給使用者。

- 電話供應商（例如 Vonage 或 RingCentral）的 VoIP 電話號碼。

- 商務用 Skype 混合式電話號碼。 如果您想要轉移這些號碼，您需要傳送電子郵件給我們<ptn@microsoft.com>。

  **您無法傳輸：**
  
> [!NOTE]
> 此時，您無法從支援的國家/地區（包括 VoIP 手機提供者的電話號碼）中轉移任何不是來自支援的國家/地區的電話號碼或號碼。 若要查看支援的國家/地區清單，請參閱[音訊會議與通話方案的國家/地區可用性](country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans.md)
  
- 用於資料連線的電話號碼，例如 DSL 線路或寬頻網際網路連線。

- 專用來傳真的電話號碼。

    如果您的現有專用電話號碼是用來傳送傳真的，您*可以*將這些號碼轉接至小組，但您的傳真服務不會如預期的那樣繼續運作。 [傳真服務] 不會提供給小組客戶，即使您有**電話系統**、**國內通話方案**或**國際通話方案**的授權。

    如果您將電話號碼移植至團隊，您可以將此電話號碼指派給組織中的使用者，而不是用來進行傳真。

> [!NOTE]
> 目前在英國（英國）中，我們目前不支援傳輸英國非地理數位，包括區號0843、0844、0845、0870、0871、0872的共用成本數位。 
  
## <a name="what-information-will-i-need-to-provide"></a>我需要提供哪些資訊？

您必須為目前的運營商準備好所有的帳戶資訊。 您必須放在 [埠順序] 中的資訊，主要是在目前服務提供者的最近帳單或發票中找到。 您也必須知道其帳戶的名稱，以及您想要移植的號碼。
  
## <a name="what-are-full-port-and-partial-port-transfers"></a>什麼是完整埠和部分埠傳輸？

當您將電話號碼移植至 Office 365 時，您可以選擇轉移所有號碼或其中一部分。
  
- **全埠**當您將所有數位從目前的服務提供者轉移至團隊時，就是這種情況。 當系統詢問您想要轉移的電話號碼時，您必須在帳戶上*包含*帳單電話號碼以及其他所有電話號碼。

    例如，假設您的帳單電話號碼是 *+ 1 425-555-1234* ，而且您想要將25個電話號碼（*+ 1 425-555-1235 到 1259*）全部移植。 當您遵循下列指示來轉接您的數位時，請輸入： **+ 14255551234-+ 14255551259**。

- **部分埠**這是您只會將部分電話號碼從目前的服務提供者轉移至團隊。 如果您想要將部分電話號碼捆綁到相同帳單電話號碼，您 * **不能包含** * 帳單電話號碼以及您帳戶上的所有其他電話號碼。

    例如，假設您的帳單電話號碼（BTN）是 *+ 1 425-555-1234* ，而且您只想要移植25個電話號碼中的5個（*+ 1 425-555-1235 至 1259*）。 當您遵循下列指示來轉接您的數位時，請輸入： **+ 1 425 555 1235-+ 1 425 555 1239**。
    
## <a name="can-i-submit-a-single-number-porting-request-for-all-of-my-numbers-at-one-time"></a>我可以一次為所有號碼提交一個數位移植要求嗎？
<a name="bkmk_type_1"> </a>

要移植的每個載波和編號類型，都需要唯一的要求。
  
例如，您需要針對下列每個號碼類型提交唯一的號碼移植要求：
  
- 當地付費電話號碼，也稱為訂閱者號碼或地理位置號碼

- 含區號的免費電話號碼：800、844、855、866、877和888

- 行動電話號碼

- 可在 Office 365 中用來進行音訊會議的服務號碼。

以下是針對每個類型的號碼提交編號移植要求的詳細資訊：
  
- 不同的運營商所提供的**電話號碼**，對於每個電信公司而言，都需要有唯一的號碼移植要求。

- 含區號的免費**電話號碼**：800、844、855、866、877和888不能包含在其他類型的數位移植要求中。 若要移植這些免費電話號碼，您必須[手動提交自訂服務要求](/SkypeForBusiness/what-are-calling-plans-in-office-365/manually-submit-a-custom-service-request);無法在商務用 Skype 系統管理中心中提交。 請參閱[管理貴組織的電話號碼](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)。

    若要移植，請務必使用 [國家/地區] 的正確 LOA 以及電話號碼類型。 您可以下載您需要的 LOA，以[下載您在此所需的授權信（LOA）](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)。

- **行動電話號碼**需要 PIN 碼來授權轉接。 因此，它們需要個別的數位移植要求。

- **服務編號**移植要求必須自行提交。 它們無法與其他類型的數位一起提交。

## <a name="how-long-does-it-take-to-port-numbers"></a>埠號碼需要多少時間？
<a name="bkmk_type_1"> </a>

完成 [埠順序] 要求後，將需要7-14 天的時間來處理。 不過，視您的服務提供者而定，可能需要長達30天的時間。 在撥出電話號碼之後，您會收到一封電子郵件，告訴您您已準備好了。
  
您可以移至商務用 Skype 系統管理中心 > 的**語音** > **埠訂單**，來檢查您的埠順序狀態。 狀態會列在 [Status] （**狀態**）欄下的視窗中。
  
## <a name="can-user-subscriber-phone-numbers-be-converted-to-service-numbers"></a>使用者（訂閱者）電話號碼可以轉換為服務號碼嗎？
<a name="bkmk_type_1"> </a>

可以。 您只需要提交服務要求，其中包含您組織的租使用者 GUID 和您想要轉換的電話號碼。 若要這樣做，請參閱[管理貴組織的電話號碼](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)。 
  
## <a name="common-mistakes-to-watch-out-for"></a>需要注意的常見錯誤
<a name="bkmk_type_1"> </a>

數位移植很容易完成。 您的訂單可以向上 messed，不過，當電話服務提供者出現問題時，該訂單不完整且遺失資訊，或發生輸入錯誤。
  
以下是我們在客戶撥打電話號碼時最常看到的錯誤。 自行儲存來電給客戶支援，並仔細檢查這些錯誤。
  
- 請確定您所提供的帳戶資訊，與您的電話聯絡人記錄的內容完全相符。 不匹配的資訊是錯誤最常見的原因，而且會延遲您的埠順序。 確認下列專案成立：

  - 授權名稱正確無誤。

  - 位址正確無誤。

  - 帳戶號碼正確無誤。

  - 帳單電話號碼（BTN）正確無誤。

- 確定在這些電話號碼上沒有任何高級通話控制功能（例如，通話查尋、特殊鈴聲）。

- 確認您還沒有加入任何新的服務訂單，或中斷與目前服務提供者的連線。

- 確定所有號碼都來自同一個承運人和同一個帳戶。

- 請確定您的服務處於作用中狀態。 凍結帳戶會防止帳戶上的載波變更。 授權的使用者必須將訂單提交給目前的運營商，才能移除凍結。 根據運營商的不同，此程式可能需要1-3 周。

## <a name="can-you-transfer-or-port-out-numbers"></a>您可以轉移或移植輸出號碼嗎？
<a name="bkmk_type_1"> </a>

若要將小組的電話號碼轉移或撥*出*給其他電話服務提供者或電信公司，您必須設定 PIN。 在您設定 PIN 之後，當您要求將電話號碼移植到外，您必須將它包含在 PIN 中。若要瞭解如何設定您的 PIN，請參閱[設定您的 pin 以將號碼轉移到新的服務提供者](/SkypeForBusiness/what-are-calling-plans-in-office-365/set-your-pin-for-transferring-numbers-to-a-new-service-provider)。
