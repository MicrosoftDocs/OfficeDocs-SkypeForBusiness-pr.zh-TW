# <a name="direct-routing-media-path-country-codes"></a><span data-ttu-id="bdc57-103">直接路由媒體路徑國家/地區代碼</span><span class="sxs-lookup"><span data-stu-id="bdc57-103">Direct Routing media path country codes</span></span>

<span data-ttu-id="bdc57-104">選擇媒體的路由路徑時, [直接路由] 預設會根據會話邊界控制器 (SBC) 的公用 IP 位址來指派資料中心, 且總是選取最接近 SBC 資料中心的路徑。</span><span class="sxs-lookup"><span data-stu-id="bdc57-104">When choosing a routing path for media, Direct Routing, by default, always assigns a datacenter based on the public IP address of the Session Border Controller (SBC), and always selects the path closest to the SBC datacenter.</span></span>

<span data-ttu-id="bdc57-105">不過, 在某些情況下, 預設媒體路徑可能不是最佳的媒體路徑;例如, 來自美國範圍的公用 IP 可能會指派給位於歐洲的 SBC。</span><span class="sxs-lookup"><span data-stu-id="bdc57-105">However, in some cases the default media path might not be the optimal media path; for example, a public IP from a United States range might be assigned to an SBC located in Europe.</span></span> 

<span data-ttu-id="bdc57-106">透過將-MediaRelayRoutingLocationOverride 參數與新的-CsOnlinePSTNGateway 和 Set CsOnlinePSTNGateway Cmdlet 結合使用, 您就可以指定媒體流量的喜好區域。</span><span class="sxs-lookup"><span data-stu-id="bdc57-106">By using the -MediaRelayRoutingLocationOverride parameter with the New-CsOnlinePSTNGateway and Set-CsOnlinePSTNGateway cmdlets, you can specify the preferred region for media traffic.</span></span> <span data-ttu-id="bdc57-107">例如, 下列命令指定 [喜好] 區域為 [德國]:</span><span class="sxs-lookup"><span data-stu-id="bdc57-107">For example, the following command specifies that the preferred region is Germany:</span></span>

<span data-ttu-id="bdc57-108">CSOnlinePSTNGateway-身分識別 sbc1.contoso.com – MediaRelayRoutingLocationOverride DE</span><span class="sxs-lookup"><span data-stu-id="bdc57-108">Set-CSOnlinePSTNGateway -Identity sbc1.contoso.com –MediaRelayRoutingLocationOverride DE</span></span> 

<span data-ttu-id="bdc57-109">請注意, 如果通話記錄清楚指出媒體路徑中資料中心的預設指派不是使用最接近 SBC 資料中心的路徑, Microsoft 只建議設定此參數。</span><span class="sxs-lookup"><span data-stu-id="bdc57-109">Note that Microsoft only recommends setting this parameter if the call logs clearly indicate that the default assignment of the datacenter for the media path does not use the path closest to the SBC datacenter.</span></span>
