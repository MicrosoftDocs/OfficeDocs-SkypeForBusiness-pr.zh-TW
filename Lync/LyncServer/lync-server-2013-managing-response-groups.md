# <a name="managing-response-groups-in-lync-server-2013"></a>在 Lync Server 2013 中管理回應群組

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**主題上次修改日期：** 2018-02-01_

回應群組是一種通話管理功能，可讓您保留對特定區域所撥出的通話 (例如客服中心)，然後將這些通話路由傳送至指定的人員群組 (稱為 *「代理人」*)。

若要管理回應群組，請設定代理人群組、佇列及工作流程，藉以定義從通話保留至代理人應答這段期間對通話的處理方式。

<div>


> [!NOTE]  
> 如果您有超過300的工作流程在回應群組部署的單一集區中，最好是使用 Lync Server 管理命令介面 Cmdlet 來建立工作流程。 如果您使用「回應群組設定工具」，為內含 300 多個工作流程的集區建立更多工作流程，該網頁需要一段時間載入。 透過佇列間接與工作流程相關聯的代理程式數目，在頁面載入時也會產生比例影響。



</div>

本節中的主題會針對您可以在部署中執行自訂及維護回應群組應用程式的工作，提供逐步程式。
