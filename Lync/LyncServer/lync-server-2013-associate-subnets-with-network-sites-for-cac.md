# <a name="associate-subnets-with-network-sites-for-cac-in-lync-server-2013"></a>在 Lync Server 2013 中，將子網與網路網站建立在 CAC 中

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**主題上次修改日期：** 2012-10-20_

您網路中的每個子網都必須與特定的網路網站相關聯。 這是因為子網資訊是用來判斷端點所在的網路網站。 當會話中雙方雙方的位置都已知時，通話許可控制（CAC）可以判斷是否有足夠的頻寬來建立通話。

[通話許可控制] 不會對將子網與網路網站建立關聯的任何特殊需求。 若要在您的拓撲中的子網和網路網站之間建立關聯，請遵循在[Lync Server 2013 中將子網與網路網站建立](lync-server-2013-associate-a-subnet-with-a-network-site.md)關聯的程式。 若要在 [呼叫許可控制] 的範例網路拓朴中查看網路網站（及其各自的子網），請參閱範例：在規劃檔中，[收集 Lync Server 2013 的通話許可控制需求](lync-server-2013-example-of-gathering-your-requirements-for-call-admission-control.md)。
