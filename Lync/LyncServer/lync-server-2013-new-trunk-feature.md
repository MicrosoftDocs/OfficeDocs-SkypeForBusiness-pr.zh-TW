# <a name="new-trunk-feature-in-lync-server-2013"></a>Lync Server 2013 中的新主幹功能

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**主題上次修改日期：** 2012-09-21_

在 Microsoft Lync Server 2013 中，可以定義在中繼伺服器和閘道之間的多個 trunks。 Microsoft Lync Server 2010 只允許在中繼伺服器與 PSTN 閘道之間的單一主幹中使用。 這個功能提供了定義其他 trunks 的彈性。 主幹是中繼伺服器 FQDN 與偵聽埠與 PSTN 閘道 FQDN 與偵聽埠之間的邏輯關聯。 這項新功能可讓您輕鬆地進行復原（多個轉送伺服器可用於將呼叫路由至同一個 PSTN 閘道），以進行 PBX 互通性，在與之間可以使用多個具有不同關聯原則的 trunks。IP-PBX 與中繼伺服器，以及適用于不同網站的中繼伺服器將 SIP trunks 給同一個運營商 FQDN 所參照之載波的 SIP 幹線配置。
