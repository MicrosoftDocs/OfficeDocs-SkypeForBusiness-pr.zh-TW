<span data-ttu-id="42947-111">如果您選擇在 Lync Server 2013 中啟用監控，您可以啟用 CDR 監控和 QoE 監視，也可以選擇啟用一種監控類型，而讓其他類型保持停用。</span><span class="sxs-lookup"><span data-stu-id="42947-111">If you choose to enable monitoring in Lync Server 2013 you can enable both CDR monitoring and QoE monitoring, or you can choose to enable one type of monitoring while leaving the other type disabled.</span></span> <span data-ttu-id="42947-112">例如，假設您的使用者只使用立即訊息和檔案傳輸，也不會進行音訊或視頻通話。</span><span class="sxs-lookup"><span data-stu-id="42947-112">For example, suppose your users only use instant messaging and file transfers, and do not make audio or video calls.</span></span> <span data-ttu-id="42947-113">在此情況下，可能不需要啟用 QoE 監視。</span><span class="sxs-lookup"><span data-stu-id="42947-113">In that case, there might be little reason to enable QoE monitoring.</span></span> <span data-ttu-id="42947-114">同樣地，Lync Server 也可讓您在部署監控後，輕鬆啟用及停用監控。</span><span class="sxs-lookup"><span data-stu-id="42947-114">Likewise, Lync Server makes it easy to enable and disable monitoring after monitoring has been deployed.</span></span> <span data-ttu-id="42947-115">例如，您可以選擇部署監控，但在開始時仍會停用 QoE 監視。</span><span class="sxs-lookup"><span data-stu-id="42947-115">For example, you might choose to deploy monitoring but initially leave QoE monitoring disabled.</span></span> <span data-ttu-id="42947-116">如果您的使用者開始遇到音訊或視頻通話的問題，您可以啟用 QoE 監視，並使用該資料來協助您疑難排解及解決這些問題。</span><span class="sxs-lookup"><span data-stu-id="42947-116">If your users begin to experience problems with audio or video calls you could then enable QoE monitoring and use that data to help you troubleshoot and resolve those problems.</span></span>

<span data-ttu-id="42947-117">當您安裝 Lync Server 時，並不會有特別優點 (或缺點) 在安裝 lync server 之後安裝監視。</span><span class="sxs-lookup"><span data-stu-id="42947-117">There is no particular advantage (or disadvantage) to installing monitoring at the same time you install Lync Server vs. installing monitoring after Lync Server has been installed.</span></span> <span data-ttu-id="42947-118">需要記住的一點是，在您安裝監控之前，您必須先選取一部電腦來裝載後端監控存放區，而且必須先在該電腦上安裝及設定 SQL Server 的支援版本，該電腦才可用於監視。</span><span class="sxs-lookup"><span data-stu-id="42947-118">The one point to keep in mind is that, before you install monitoring, you must select a computer to host the backend monitoring store, and a supported version of SQL Server must be installing and configured on that computer before that computer can be used for monitoring.</span></span> <span data-ttu-id="42947-119">如果您已在電腦上安裝 SQL Server，而且該電腦已準備好供使用，則您可以在安裝 Lync Server 的同時安裝監視。</span><span class="sxs-lookup"><span data-stu-id="42947-119">If you have already installed SQL Server on a computer and that computer is ready for use then you can install monitoring at the same time you install Lync Server.</span></span> <span data-ttu-id="42947-120">如果您沒有現成的後端電腦，您可以繼續自行安裝 Lync Server，然後在後端電腦準備好使用時安裝監視。</span><span class="sxs-lookup"><span data-stu-id="42947-120">If you do not have a backend computer ready then you can proceed to install Lync Server by itself, then install monitoring whenever the backend computer is ready for use.</span></span>

<span data-ttu-id="42947-121">**您何時要安裝監控？**</span><span class="sxs-lookup"><span data-stu-id="42947-121">**When do you want to install monitoring?**</span></span> <span data-ttu-id="42947-122">您可以在安裝和設定 Lync Server 2013 的同時安裝和設定監視;Lync Server 部署嚮導可讓您有機會在安裝期間，將前端集區與監控資料庫產生關聯。</span><span class="sxs-lookup"><span data-stu-id="42947-122">Monitoring can be installed and configured at the same time you install and configured Lync Server 2013; the Lync Server Deployment Wizard will provide you with the opportunity to associate your Front End pools with a monitoring database during setup.</span></span> <span data-ttu-id="42947-123">或者，您也可以在安裝 Lync Server 本身之後安裝監視;若要使用拓撲產生器，將前端集區與伺服器與監控資料庫產生關聯，然後發佈修訂的拓撲，即可完成此動作。</span><span class="sxs-lookup"><span data-stu-id="42947-123">Alternatively, you can install monitoring after Lync Server itself has been installed; this can be done by using Topology Builder to associate your Front End pools and servers with a monitoring database, and then publishing the revised topology.</span></span>
