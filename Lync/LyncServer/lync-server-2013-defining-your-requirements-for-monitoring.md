<span data-ttu-id="b838f-103">_**主題上次修改日期：** 2012-09-05_</span><span class="sxs-lookup"><span data-stu-id="b838f-103">_**Topic Last Modified:** 2012-09-05_</span></span>

<span data-ttu-id="b838f-104">在 Microsoft Lync Server 2013 中簡化監視的部署和安裝，也可簡化定義貴組織的監視需求所涉及的程式。</span><span class="sxs-lookup"><span data-stu-id="b838f-104">Streamlining the deployment and installation of monitoring in Microsoft Lync Server 2013 has also streamlined the processes involved in defining your organization's requirements for monitoring.</span></span> <span data-ttu-id="b838f-105">不過，在您開始安裝及設定 Lync Server 2013 之前，仍需要解決幾個主要問題：</span><span class="sxs-lookup"><span data-stu-id="b838f-105">Nevertheless, there are still several key issues that should be addressed before you begin to install and configure Lync Server 2013:</span></span>

<span data-ttu-id="b838f-106">**您想要監視何種類型的資料？**</span><span class="sxs-lookup"><span data-stu-id="b838f-106">**What type of data do you want to monitor?**</span></span> <span data-ttu-id="b838f-107">Lync Server 2013 可讓您監視兩種不同類型的資料：通話詳細錄製（CDR）資料和體驗品質（QoE）資料。</span><span class="sxs-lookup"><span data-stu-id="b838f-107">Lync Server 2013 enables you to monitor two different types of data: call detailing recording (CDR) data and Quality of Experience (QoE) data.</span></span> <span data-ttu-id="b838f-108">通話詳細資料錄製提供一種方式，讓您追蹤 Lync Server 功能（例如，語音 over IP 通話）的使用方式;立即訊息（IM）;檔案傳輸;音訊/視頻（A/V）會議;與應用程式共用會話。</span><span class="sxs-lookup"><span data-stu-id="b838f-108">Call detail recording provides a way for you to track the usage of Lync Server features such as Voice over IP (VoIP) phone calls; instant messaging (IM); file transfers; audio/video (A/V) conferencing; and application sharing sessions.</span></span> <span data-ttu-id="b838f-109">這項資訊可協助您瞭解所使用的是哪些 Lync Server 功能（以及哪些功能不是），也會提供這些功能的使用方式的相關資訊。</span><span class="sxs-lookup"><span data-stu-id="b838f-109">This information helps you know which Lync Server features are being used (and which ones are not) and also provides information as to when these features are being used.</span></span> <span data-ttu-id="b838f-110">經驗品質資料可讓您維護貴組織中音訊和視頻通話品質的記錄，包括網路資料包遺失、背景雜色及「抖動」（資料包延遲的差異）等專案。</span><span class="sxs-lookup"><span data-stu-id="b838f-110">Quality of Experience data allows you to maintain a record of the quality of audio and video calls made in your organization, including such things as the number of network packets lost, background noise, and the amount of "jitter" (differences in packet delay).</span></span>

<span data-ttu-id="b838f-111">如果您選擇在 Lync Server 2013 中啟用監視，您可以同時啟用 CDR 監視與 QoE 監視，或者，您也可以選擇啟用單一監視類型，而讓其他類型保持停用。</span><span class="sxs-lookup"><span data-stu-id="b838f-111">If you choose to enable monitoring in Lync Server 2013 you can enable both CDR monitoring and QoE monitoring, or you can choose to enable one type of monitoring while leaving the other type disabled.</span></span> <span data-ttu-id="b838f-112">例如，假設您的使用者只使用立即訊息和檔案傳輸，且不進行音訊或視頻通話。</span><span class="sxs-lookup"><span data-stu-id="b838f-112">For example, suppose your users only use instant messaging and file transfers, and do not make audio or video calls.</span></span> <span data-ttu-id="b838f-113">在這種情況下，可能無需啟用 QoE 監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-113">In that case, there might be little reason to enable QoE monitoring.</span></span> <span data-ttu-id="b838f-114">同樣地，Lync Server 可讓您輕鬆啟用並在部署監視後停用監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-114">Likewise, Lync Server makes it easy to enable and disable monitoring after monitoring has been deployed.</span></span> <span data-ttu-id="b838f-115">例如，您可以選擇部署監視，但不停停 QoE 監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-115">For example, you might choose to deploy monitoring but initially leave QoE monitoring disabled.</span></span> <span data-ttu-id="b838f-116">如果您的使用者開始遇到音訊或視頻通話的問題，您可以啟用 QoE 監視，並使用該資料來協助您疑難排解並解決這些問題。</span><span class="sxs-lookup"><span data-stu-id="b838f-116">If your users begin to experience problems with audio or video calls you could then enable QoE monitoring and use that data to help you troubleshoot and resolve those problems.</span></span>

<span data-ttu-id="b838f-117">在安裝 Lync Server 之後，沒有特殊的優點（或缺點），無法安裝 [在 Lync Server 安裝後進行監視] 與 [安裝]。</span><span class="sxs-lookup"><span data-stu-id="b838f-117">There is no particular advantage (or disadvantage) to installing monitoring at the same time you install Lync Server vs. installing monitoring after Lync Server has been installed.</span></span> <span data-ttu-id="b838f-118">要記住的一點是，在您安裝監視之前，您必須先選取電腦來託管後端監視存放區，而且在該電腦上必須安裝並設定受支援版本的 SQL Server，才能使用該電腦進行監視.</span><span class="sxs-lookup"><span data-stu-id="b838f-118">The one point to keep in mind is that, before you install monitoring, you must select a computer to host the backend monitoring store, and a supported version of SQL Server must be installing and configured on that computer before that computer can be used for monitoring.</span></span> <span data-ttu-id="b838f-119">如果您已在電腦上安裝 SQL Server，且該電腦已準備好使用，您可以在安裝 Lync Server 的同時安裝監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-119">If you have already installed SQL Server on a computer and that computer is ready for use then you can install monitoring at the same time you install Lync Server.</span></span> <span data-ttu-id="b838f-120">如果您沒有已準備好的後端電腦，您可以繼續自行安裝 Lync Server，然後在後端電腦準備好使用時安裝監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-120">If you do not have a backend computer ready then you can proceed to install Lync Server by itself, then install monitoring whenever the backend computer is ready for use.</span></span>

<span data-ttu-id="b838f-121">**您何時要安裝監視？**</span><span class="sxs-lookup"><span data-stu-id="b838f-121">**When do you want to install monitoring?**</span></span> <span data-ttu-id="b838f-122">您可以在安裝和設定 Lync Server 2013 的同時安裝和設定監視;Lync Server 部署嚮導會提供機會，讓您在安裝期間將前端池與監視資料庫建立關聯。</span><span class="sxs-lookup"><span data-stu-id="b838f-122">Monitoring can be installed and configured at the same time you install and configured Lync Server 2013; the Lync Server Deployment Wizard will provide you with the opportunity to associate your Front End pools with a monitoring database during setup.</span></span> <span data-ttu-id="b838f-123">或者，您也可以在安裝 Lync Server 本身之後安裝監視;您可以使用拓撲建立器，將前端池和伺服器與監視資料庫建立關聯，然後發佈已修改的拓撲。</span><span class="sxs-lookup"><span data-stu-id="b838f-123">Alternatively, you can install monitoring after Lync Server itself has been installed; this can be done by using Topology Builder to associate your Front End pools and servers with a monitoring database, and then publishing the revised topology.</span></span>

<span data-ttu-id="b838f-124">**您需要多少個後端監視資料庫？**</span><span class="sxs-lookup"><span data-stu-id="b838f-124">**How many backend monitoring databases do you need?**</span></span> <span data-ttu-id="b838f-125">單一監視資料庫可支援數十個使用者（針對 Microsoft Lync Server 2010，據估計，在監視與封存中，collocated 資料庫可支援240000使用者）。</span><span class="sxs-lookup"><span data-stu-id="b838f-125">A single monitoring database can support tens of thousands of users (for Microsoft Lync Server 2010, it was estimated that a collocated database for both monitoring and archiving could support 240,000 users).</span></span> <span data-ttu-id="b838f-126">此外，單一監視資料庫可由多個前端池使用;如果您的組織有三個前端池，您可以將這三個池與同一個後端書店建立關聯。</span><span class="sxs-lookup"><span data-stu-id="b838f-126">In addition, a single monitoring database can be used by multiple Front End pools; if you have three Front End pools in your organization then you could associate all three of those pools with the same backend store.</span></span>

<span data-ttu-id="b838f-127">這只意味著，對於許多組織而言，資料庫容量不會是決定所需後端監視資料庫數目的決定因素。</span><span class="sxs-lookup"><span data-stu-id="b838f-127">This simply means that, for many organizations, database capacity will not be the deciding factor when determining the number of backend monitoring databases that will be required.</span></span> <span data-ttu-id="b838f-128">相反地，更重要的考慮因素可能是網路速度。</span><span class="sxs-lookup"><span data-stu-id="b838f-128">Instead, a more important consideration could be network speed.</span></span> <span data-ttu-id="b838f-129">假設您有三個前端池，但是其中一個池是在緩慢的網路連線中找到。</span><span class="sxs-lookup"><span data-stu-id="b838f-129">Suppose you have three Front End pools, but one of those pools is located across a slow network connection.</span></span> <span data-ttu-id="b838f-130">在這種情況下，您可能會想要使用兩個監視資料庫：一個資料庫以使用良好的網路連線來為兩個池提供服務，並使用一個單獨的資料庫，以較慢的網路連線提供該池的服務。</span><span class="sxs-lookup"><span data-stu-id="b838f-130">In that case, you might want to use two monitoring databases: one database to service the two pools with the good network connection, and a separate database to service the pool with the slower network connection.</span></span>

<span data-ttu-id="b838f-131">規劃您的監視基礎結構時，您也應該考慮 Lync Server 2013 支援使用鏡像資料庫的情況。</span><span class="sxs-lookup"><span data-stu-id="b838f-131">When planning your monitoring infrastructure you should also take into account that Lync Server 2013 supports the use of mirror databases.</span></span> <span data-ttu-id="b838f-132">「資料庫鏡像」可讓您同時維護資料庫的兩份複本，並讓每個資料庫都駐留在不同的伺服器上。</span><span class="sxs-lookup"><span data-stu-id="b838f-132">"Database mirroring" provides a way for you to simultaneously maintain two copies of a database, with each database residing on a different server.</span></span> <span data-ttu-id="b838f-133">任何時候，都會將資料寫入到主要資料庫，而這種資料也會寫入到鏡像資料庫中。</span><span class="sxs-lookup"><span data-stu-id="b838f-133">Any time data is written to a primary database that same data is also written to the mirror database.</span></span> <span data-ttu-id="b838f-134">如果主資料庫應該失敗或無法使用，您可以使用簡單的 Lync Server PowerShell 命令，將 [容錯移轉] 到鏡像資料庫。</span><span class="sxs-lookup"><span data-stu-id="b838f-134">If the primary database should fail or otherwise become unavailable, you can "fail over" to the mirror database by using a simple Lync Server PowerShell command.</span></span> <span data-ttu-id="b838f-135">例如：</span><span class="sxs-lookup"><span data-stu-id="b838f-135">For example:</span></span>

    Invoke-CsDatabaseFailover -PoolFqdn atl-cs-001.litwareinc.com -DatabaseType "Monitoring" -NewPrincipal "Mirror"

<span data-ttu-id="b838f-136">這對於規劃的目的非常重要，因為鏡像會要求您將所需的資料庫數加倍：除了每個主資料庫之外，您還需要第二個資料庫作為鏡像。</span><span class="sxs-lookup"><span data-stu-id="b838f-136">This is important for planning purposes simply because mirroring will require you to double your required number of databases: in addition to each primary database you will need a second database to act as the mirror.</span></span>

<span data-ttu-id="b838f-137">**您的 Lync Server 網站需要自己的自訂監視配置嗎？**</span><span class="sxs-lookup"><span data-stu-id="b838f-137">**Do your Lync Server sites need their own custom monitoring configurations?**</span></span> <span data-ttu-id="b838f-138">當您安裝 Lync Server 2013 時，您也會安裝 [CDR] 和 [QoE 設定] 的全域集合;這些全域集合讓您能夠將相同的 CDR 與 QoE 設定套用到整個組織。</span><span class="sxs-lookup"><span data-stu-id="b838f-138">When you install Lync Server 2013 you also install global collections of CDR and QoE configuration settings; these global collections give you the ability to apply the same CDR and QoE settings to your entire organization.</span></span> <span data-ttu-id="b838f-139">在許多情況下，這會足夠：您通常會想要為所有的使用者啟用 CDR 監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-139">In many cases, this will be sufficient: often-times you will want, say, to have CDR monitoring enabled for all of your users.</span></span>

<span data-ttu-id="b838f-140">不過，您也可能會想要將不同的設定套用至不同的網站。</span><span class="sxs-lookup"><span data-stu-id="b838f-140">However, there might also be times when you want to apply different settings to different sites.</span></span> <span data-ttu-id="b838f-141">例如，您可能想要在雷德蒙網站中使用 CDR 與 QoE 監視，但只在您的都柏林網站中使用 CDR 監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-141">For example, perhaps you want to use both CDR and QoE monitoring in your Redmond site, but only use CDR monitoring in your Dublin site.</span></span> <span data-ttu-id="b838f-142">同樣地，您可能想要在雷德蒙者網站保留60天的監控資料，但只需要在都柏林網站中維護此類型的資料。</span><span class="sxs-lookup"><span data-stu-id="b838f-142">Likewise, you might want to retain monitoring data for 60 days in the Redmond site but only need to maintain this type of data for 30 days in the Dublin site.</span></span> <span data-ttu-id="b838f-143">Lync Server 2013 可讓您在網站範圍中建立不同的 CDR 及 QoE 設定設定;這可讓您以不同方式管理每個網站。</span><span class="sxs-lookup"><span data-stu-id="b838f-143">Lync Server 2013 allows you to create separate collections of CDR and QoE configuration settings at the site scope; that enables you to manage each site differently.</span></span> <span data-ttu-id="b838f-144">（這包括啟用和停用監控，以及設定管理設定，例如要保留多久資料。）</span><span class="sxs-lookup"><span data-stu-id="b838f-144">(This includes both enabling and disabling monitoring as well as configuring management settings such as how long data is to be retained.)</span></span>

<span data-ttu-id="b838f-145">請注意，您可以在部署監視之前或在部署監視之後進行此決策。</span><span class="sxs-lookup"><span data-stu-id="b838f-145">Note that you can make this decision before you deploy monitoring or after you deploy monitoring.</span></span> <span data-ttu-id="b838f-146">例如，您可以部署監視，然後使用 [全域設定] 管理整個組織。</span><span class="sxs-lookup"><span data-stu-id="b838f-146">For example, you can deploy monitoring and then manage the entire organization by using the global settings.</span></span> <span data-ttu-id="b838f-147">如果您稍後改變主意，您可以針對雷德蒙者的網站建立一組不同的設定，然後使用這些設定來管理雷德蒙者的監視。</span><span class="sxs-lookup"><span data-stu-id="b838f-147">If you later change your mind, you can create a separate collection of settings for, say, the Redmond site, and then use those settings to manage monitoring for Redmond.</span></span> <span data-ttu-id="b838f-148">（在網站範圍套用的設定，總是優先于在全域範圍中套用的設定。）如果您再次改變主意，您可以直接刪除套用至雷蒙德網站的設定設定。</span><span class="sxs-lookup"><span data-stu-id="b838f-148">(Settings applied at the site scope always take precedence over settings applied at the global scope.)
