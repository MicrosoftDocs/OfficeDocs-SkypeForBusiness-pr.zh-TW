# <a name="documentation-in-lync-server-2013"></a>Lync Server 2013 中的檔

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**主題上次修改日期：** 2015-05-15_

MOF 模型是由許多服務管理功能所組成。 有關如何以及何時執行工作的檔，可以與相同團隊或其他團隊的成員共用。 儲存及共用檔的方法可能會根據函數類型而有所不同。 例如，系統管理的程式可能會儲存為 Word 檔，因為它們可能會經常列印並加以參照。 建構管理資訊可能會自動產生並儲存在資料庫中，以方便搜尋和編制索引。 有些檔可能是敏感的，應該受到限制。

<div>

## <a name="document-management-systems"></a>檔管理系統

檔管理系統是檔的中央文件庫，可協助確保只有最新版本的檔可供使用。 您也可以考慮封存較舊版本的檔以供參考。 Lync Server 提供適合此任務的功能。

</div>

<div>

## <a name="databases"></a>資料庫

我們討論了幾個適用于使用資料庫的工具和管理功能。 建構管理程式可能會使用自動程式儲存大量需要編制索引及搜尋的資料。 支援人員可能會在針對新問題進行疑難排解時搜尋過去問題和解決方式的資料庫。

可能會有不同的資料庫被用於不同的用途。 決定這些資料庫是否應該連結或合併。 例如，如果服務台發現常見主題有幾個問題（例如新的軟體造成特定網路介面卡的問題），支援人員可以查詢設定資料庫，以預測可能會受到影響的電腦數。

</div>
