## <a name="pstn-gateways"></a>PSTN 閘道

公用交換電話網絡（PSTN）閘道是協力廠商硬體元件，可直接或透過連線至 SIP trunks，在企業語音結構和 PSTN 之間轉換信號和媒體。 在其中一個拓撲中，閘道會終止 PSTN。 閘道會隔離在自己的子網中，並透過轉送伺服器連線到商業網路。

具有多個網站的企業通常會在每個網站上部署一個或多個閘道。 分支網站可以透過閘道或透過 Survivable 分支裝置連線到 PSTN，這會將閘道與伺服器結合在單一方塊中。 如果分支網站使用閘道，則會在網站上需要註冊機構和中繼伺服器，除非 WAN 連結是復原的。 一或多個在前端伺服器上 collocated 的中繼伺服器可以在每個網站上傳送一或多個閘道的呼叫。 我們建議您將網站上所需的註冊機構、中繼伺服器和閘道部署為 Survivable 分支裝置。

確定 PSTN 閘道的數目、大小及位置，可能是規劃企業語音基礎結構時必須做出的最重要且昂貴的決策。

以下是要考慮的主要問題。 請記住，這些問題的答案全都相互依賴

  - 需要多少 PSTN 閘道？ 答案取決於使用者數目、預期的同時通話數（流量負載），以及網站數量（每個網站需要一個）。

  - 閘道應該是什麼大小？ 答案視網站上的使用者數目和流量負荷而定。

  - 閘道應該位於何處？ 答案是在拓撲的一部分，以及貴組織的地理位置分佈所組成。

您也應該考慮閘道拓朴選項（如需詳細資訊，請參閱本主題稍後的閘道拓撲）。

## <a name="mn-trunk-support"></a>M:N 幹線支援

中繼伺服器可透過多個閘道、網際網路電話服務提供者所提供的會話邊界控制器（SBCs），或這兩者的組合，路由呼叫。 此外，池中的多個轉送伺服器可以與多個閘道互動。 在中繼伺服器和閘道之間定義的邏輯路由稱為*主幹*。 當內部使用者進行 PSTN 通話時，前端池上的輸出路由邏輯會選擇要路由超出所有可能可供路由該特定呼叫的組合。 使用 DNS 負載平衡時，如果由於池中的特定中繼伺服器問題而導致呼叫無法接通閘道，該呼叫將會重試至池中的備用轉送伺服器。

如需規劃多個閘道的詳細資料，請參閱[Lync Server 2013 中的 M:N 幹線](lync-server-2013-m-n-trunk.md)。

如需其他輸出路由增強功能的詳細資料，請參閱[Lync Server 2013 中的語音路由](lync-server-2013-voice-routes.md)。

## <a name="gateway-topologies"></a>閘道拓撲

當您考慮閘道部署的基本問題時，請遵循下列步驟：

1.  使用企業語音來計算您想要提供 PSTN 連線性的網站數目。

2.  估計每個網站的流量（使用者數及每個使用者每小時的平均通話數）。

3.  在每個網站部署一或多個閘道，以處理預期的流量。

產生的分散式閘道拓撲如下圖所示。

**分散式閘道拓朴**

![分散式閘道拓撲圖表](images/Gg398899.f0f65a0b-a462-491a-878b-4d4bf0a96f6d(OCS.15).jpg "分散式閘道拓撲圖表")

在這個拓朴中，每個網站的工作人員之間以及網站之間的呼叫都是在您的內部網路上路由。 對 PSTN 的呼叫會在企業 IP 網路上路由至最接近目的地號碼位置的閘道。但是，如果您的組織支援數十或幾百或甚至上千個以上的網站散佈在一或多個洲，那麼許多金融機構和其他大型企業都有這麼做？ 在這種情況下，在每個網站上部署個別的閘道並不是不切實際的。

若要解決此問題，許多大型公司都想要部署一個或幾個大型電話中心網站，如下圖所示。

**電話中心網站拓撲**

![資料中心閘道拓撲](images/Gg398899.927f4808-bf74-405a-be20-2cd9cd87af6d(OCS.15).jpg "資料中心閘道拓撲")

在這個拓朴中，有幾個大型閘道足以容納預期的使用者負載，就是在每一個中央網站部署。 企業中的所有使用者呼叫都是由公司的電話服務提供者轉寄至中心網站。 中央網站上的路由邏輯可決定是否要在內部網路或 PSTN 路由通話。

## <a name="gateway-location"></a>閘道位置

閘道位置也可以決定您選擇的閘道類型，以及它們的設定方式。 有許多 PSTN 通訊協定，都不是世界標準。 如果您的所有閘道都位於單一的國家/地區，這不是問題，但如果您在多個國家/地區中找到閘道，則必須根據該國家/地區的 PSTN 標準進行設定。 此外，經認證的閘道（例如加拿大），可能無法在印度、巴西或歐盟進行認證。

## <a name="gateway-size-and-number"></a>閘道大小與數位

大多陣列織都要考慮部署範圍的 PSTN 閘道，其大小必須是2到960埠。 （甚至是更大的閘道，但主要是由電話服務提供者使用。）估計貴組織所需的埠數時，請遵循下列指導方針：

  - 使用輕型電話使用的組織（每個小時每個使用者一個 PSTN 通話）應該為每15個使用者分配一個埠。 例如，如果您有20個使用者，就需要一個具有兩個埠的閘道。

  - 具有中等電話使用方式（每個使用者每小時兩個 PSTN 通話）的組織應該為每10個使用者分配一個埠。 例如，如果您有100使用者，您將需要在一或多個閘道間分配10個埠。

  - 具有大量電話使用的組織（每小時每位使用者有三個或更多個 PSTN 通話）應該為每五位使用者指派一個埠。 例如，如果您有47000使用者，您需要在至少10個大型閘道之間分配的9400埠總數。

  - 在貴組織中的使用者數目或流量增加時，可以取得額外的埠。

針對您必須支援的任何特定使用者數，您可以選擇部署較少、較大的閘道或較小的閘道。 就規則而言，建議組織至少有兩個閘道，才能在一個閘道失敗時維持可用性。

您部署的每個 PSTN 閘道都必須至少有一個相對應的產生轉送伺服器。
