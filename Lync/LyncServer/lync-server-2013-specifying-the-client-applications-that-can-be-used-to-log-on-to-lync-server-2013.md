# <a name="specifying-the-client-applications-that-can-be-used-to-log-on-to-lync-server-2013"></a>指定可用於登入 Lync Server 2013 的用戶端應用程式

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**主題上次修改日期：** 2012-12-11_

Lync Server 2013 可讓您指定環境中支援的用戶端版本。 使用用戶端版本原則可協助減少支援多個用戶端版本相關的成本。 它也可以改善整體使用者經驗，因為當舊版用戶端與更新版本互動時，舊版的用戶端就可以限制可用的功能。

用戶端版本控制有三個元件：

  - 用戶端版本設定可用來開啟或關閉用戶端版本控制，不論是全域性或針對特定網站。

  - 用戶端版本原則是用來全域指派一組規則，或指派給特定網站、集區或使用者群組。

  - 用戶端版本原則規則組成用戶端版本原則，用來定義當使用者嘗試以特定用戶端和用戶端版本登入時應採取的動作。

<div>


> [!NOTE]  
> 由於匿名使用者不會與使用者、網站或服務產生關聯，因此匿名使用者只會受全域層級的原則影響。
