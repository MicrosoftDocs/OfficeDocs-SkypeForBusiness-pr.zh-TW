### <a name="understanding-the-types-of-ip-network-services--basic-ip-and-mpls"></a><span data-ttu-id="4bc43-736">瞭解 IP 網路服務的類型- 基本 IP 和 MPLS</span><span class="sxs-lookup"><span data-stu-id="4bc43-736">Understanding the types of IP Network Services- Basic IP and MPLS</span></span>

<span data-ttu-id="4bc43-737">原始 IP 封包轉送以「盡力」原則運作。</span><span class="sxs-lookup"><span data-stu-id="4bc43-737">The original IP packet forwarding operated on the principle of "best effort."</span></span> <span data-ttu-id="4bc43-738">這表示轉送這些 IP 封包的路由器會盡力將 IP 封包傳送至目的地，但絕對無法保證何時或何時到達目的地。</span><span class="sxs-lookup"><span data-stu-id="4bc43-738">That meant that the routers forwarding those IP packets would do their best to deliver them to their destinations, but there was absolutely no guarantee with regard to when or if they would arrive at their destinations.</span></span> <span data-ttu-id="4bc43-739">這就是基本網際網路服務 ，包括您的家用網際網路連接，今天如何工作。</span><span class="sxs-lookup"><span data-stu-id="4bc43-739">That is how basic Internet services, including your home Internet connection, work today.</span></span> <span data-ttu-id="4bc43-740">其構想是，如果特定應用程式需要可靠性，它會在通訊協定堆疊的較高層級提供。</span><span class="sxs-lookup"><span data-stu-id="4bc43-740">The idea was that if reliability was required for a particular application, it would be provided at a higher level in the protocol stack.</span></span> <span data-ttu-id="4bc43-741">可靠的傳遞機制是 TCP (傳輸控制) 。</span><span class="sxs-lookup"><span data-stu-id="4bc43-741">The reliable delivery mechanism is the Transmission Control Protocol (TCP).</span></span> <span data-ttu-id="4bc43-742">用於即時語音和視 (的使用者資料包通訊協定) UDP 協定是不可靠 (，即「盡力」) 傳遞機制。</span><span class="sxs-lookup"><span data-stu-id="4bc43-742">The User Datagram Protocol (UDP), which is used for real-time voice and video, is the unreliable (i.e. "best effort") delivery mechanism.</span></span> 
  
<span data-ttu-id="4bc43-743">多通訊協定標籤切換 (MPLS) 是網路服務提供者提供 IP 服務，保證延遲、抖動和封包遺失的一個方法。</span><span class="sxs-lookup"><span data-stu-id="4bc43-743">Multi-Protocol Label Switching (MPLS) was developed as a means for network service providers to offer an IP service with performance guarantees for delay, jitter and packet loss.</span></span> <span data-ttu-id="4bc43-744">為了提供這些性能保證，MPLS 會採用傳統 IP 的一些不可預測性。</span><span class="sxs-lookup"><span data-stu-id="4bc43-744">To deliver on those performance guarantees, MPLS takes some of the unpredictability out of traditional IP.</span></span> <span data-ttu-id="4bc43-745">首先，MPLS 不會讓每個封包以路由器到路由器的方式到達其目的地 (其結果可能是每個封包從來源使用不同的路由到目的地) ，而是使用稱為標籤交換路徑 (LSP) 的固定路由，路由 「虛擬回路」連接上的所有封包。</span><span class="sxs-lookup"><span data-stu-id="4bc43-745">First, rather than having each packet find its way router-to-router to its destination (the result of which could be that each packet takes a different route from the source to the destination), MPLS routes all of the packets on a "virtual circuit" connection with a fixed route called a Label Switched Path (LSP).</span></span> <span data-ttu-id="4bc43-746">如果路徑中的其中一個連結失敗，使用該連結的所有 LSP 會快速重新路由。</span><span class="sxs-lookup"><span data-stu-id="4bc43-746">If one of the links in that path fails, all of the LSPs using that link are quickly rerouted.</span></span>
  
<span data-ttu-id="4bc43-747">當封包被送入 MPLS 網路時，網路服務提供者的邊緣路由器會附加額外的標頭至封包，其中包含用於轉寄至適當 LSP 的標籤。</span><span class="sxs-lookup"><span data-stu-id="4bc43-747">When a packet is sent into the MPLS network the network service provider's edge router appends an additional header to the packet that includes a label that is used to forward it over the appropriate LSP.</span></span> <span data-ttu-id="4bc43-748">MPLS 網路另一端的邊緣路由器會從標籤上除掉。</span><span class="sxs-lookup"><span data-stu-id="4bc43-748">The label is stripped off by the edge router at the other end of the MPLS network.</span></span>
