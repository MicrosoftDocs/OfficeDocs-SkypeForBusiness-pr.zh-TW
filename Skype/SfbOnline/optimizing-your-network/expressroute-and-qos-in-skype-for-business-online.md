### <a name="understanding-the-types-of-ip-network-services--basic-ip-and-mpls"></a><span data-ttu-id="2304c-736">瞭解 IP 網路服務的類型-基本 IP 與 MPLS</span><span class="sxs-lookup"><span data-stu-id="2304c-736">Understanding the types of IP Network Services- Basic IP and MPLS</span></span>

<span data-ttu-id="2304c-737">以「最佳做法」為原則運作的原始 IP 資料包轉寄。</span><span class="sxs-lookup"><span data-stu-id="2304c-737">The original IP packet forwarding operated on the principle of "best effort."</span></span> <span data-ttu-id="2304c-738">這表示將這些 IP 資料包轉寄的路由器可以盡最大的目的來傳送給他們的目的地，但在這些情況下或是否會到達目的地時絕對沒有任何保證。</span><span class="sxs-lookup"><span data-stu-id="2304c-738">That meant that the routers forwarding those IP packets would do their best to deliver them to their destinations, but there was absolutely no guarantee with regard to when or if they would arrive at their destinations.</span></span> <span data-ttu-id="2304c-739">這就是如何使用基本的網際網路服務，包括您的家用網際網路連線、今天的工作。</span><span class="sxs-lookup"><span data-stu-id="2304c-739">That is how basic Internet services, including your home Internet connection, work today.</span></span> <span data-ttu-id="2304c-740">其思想是，如果特定應用程式需要可靠性，就會在通訊協定堆疊的較高層級提供此功能。</span><span class="sxs-lookup"><span data-stu-id="2304c-740">The idea was that if reliability was required for a particular application, it would be provided at a higher level in the protocol stack.</span></span> <span data-ttu-id="2304c-741">可靠的傳遞機制是傳輸控制通訊協定（TCP）。</span><span class="sxs-lookup"><span data-stu-id="2304c-741">The reliable delivery mechanism is the Transmission Control Protocol (TCP).</span></span> <span data-ttu-id="2304c-742">[使用者資料包協定] （UDP）是用來進行即時語音和影片的，是不可靠（亦就是「最大努力」）傳遞機制。</span><span class="sxs-lookup"><span data-stu-id="2304c-742">The User Datagram Protocol (UDP), which is used for real-time voice and video, is the unreliable (i.e. "best effort") delivery mechanism.</span></span> 
  
<span data-ttu-id="2304c-743">多協定標籤切換（MPLS）是針對網路服務提供者所開發的一種方式，提供可保證延遲、抖動及資料包遺失的 IP 服務。</span><span class="sxs-lookup"><span data-stu-id="2304c-743">Multi-Protocol Label Switching (MPLS) was developed as a means for network service providers to offer an IP service with performance guarantees for delay, jitter and packet loss.</span></span> <span data-ttu-id="2304c-744">為提供這些效能保證，MPLS 會從傳統 IP 中取得一些 unpredictability。</span><span class="sxs-lookup"><span data-stu-id="2304c-744">To deliver on those performance guarantees, MPLS takes some of the unpredictability out of traditional IP.</span></span> <span data-ttu-id="2304c-745">首先，不是讓每個資料包找到其目的地的路由器對路由器的方式，而是使用與固定路由的 [虛擬線路] 連線中的所有資料包，稱為「標籤交換路徑（LSP）」的路由方式，而不是每個資料包的結果。</span><span class="sxs-lookup"><span data-stu-id="2304c-745">First, rather than having each packet find its way router-to-router to its destination (the result of which could be that each packet takes a different route from the source to the destination), MPLS routes all of the packets on a "virtual circuit" connection with a fixed route called a Label Switched Path (LSP).</span></span> <span data-ttu-id="2304c-746">如果路徑中的其中一個連結失敗，所有使用該連結的 Lsp 都會快速重新路由。</span><span class="sxs-lookup"><span data-stu-id="2304c-746">If one of the links in that path fails, all of the LSPs using that link are quickly rerouted.</span></span>
