# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="ba57f-103">在商務用 Skype Server 中管理伺服器間驗證（OAuth）與合作夥伴應用程式</span><span class="sxs-lookup"><span data-stu-id="ba57f-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>
 
<span data-ttu-id="ba57f-104">**摘要：** 在商務用 Skype Server 中管理 OAuth 與合作夥伴應用程式。</span><span class="sxs-lookup"><span data-stu-id="ba57f-104">**Summary:** Manage OAuth and partner applications in Skype for Business Server.</span></span>
  
<span data-ttu-id="ba57f-105">商務用 Skype 伺服器必須能夠與其他應用程式和伺服器產品進行安全且嚴密的通訊。</span><span class="sxs-lookup"><span data-stu-id="ba57f-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="ba57f-106">例如，您可以設定商務用 Skype Server，讓連絡人資料和/或歸檔資料儲存在 Microsoft Exchange Server 2013;不過，只有當商務用 Skype 伺服器與 Exchange 能夠安全地與其他人通訊時，才能執行此動作。</span><span class="sxs-lookup"><span data-stu-id="ba57f-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="ba57f-107">同樣地，您可以在 Office Web Apps 伺服器中排程商務用 Skype Server 會議;同樣地，只有在兩個伺服器（SharePoint 和商務用 Skype 伺服器）相互信任時，才能執行這項操作。</span><span class="sxs-lookup"><span data-stu-id="ba57f-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="ba57f-108">雖然您可以使用一種驗證機制來在商務用 Skype 伺服器與 Exchange 之間通訊，但在商務用 Skype Server 和 SharePoint 通訊時是一種不同的機制，所以更好且更有效的方法是使用所有伺服器對伺服器驗證與授權的標準化方法。</span><span class="sxs-lookup"><span data-stu-id="ba57f-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="ba57f-109">使用單一、標準化的伺服器到伺服器驗證方法，就是商務用 Skype Server 所採取的方法。</span><span class="sxs-lookup"><span data-stu-id="ba57f-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="ba57f-110">開始使用 Office server 2013 版本、商務用 Skype 伺服器（以及其他 Microsoft Server 產品（包括 Exchange Server 和 SharePoint Server），支援伺服器對伺服器驗證的 OAuth （開放授權）通訊協定，以及授權.</span><span class="sxs-lookup"><span data-stu-id="ba57f-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="ba57f-111">使用 OAuth （由許多主要網站使用的標準授權通訊協定），使用者認證和密碼不會從一部電腦傳送到另一部電腦。</span><span class="sxs-lookup"><span data-stu-id="ba57f-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="ba57f-112">相反地，驗證與授權是以安全權杖的交換為基礎;這些標記會針對特定的一組資源，授予存取權。</span><span class="sxs-lookup"><span data-stu-id="ba57f-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="ba57f-113">OAuth 驗證通常涉及三個參與方：單一授權伺服器，以及需要彼此通訊的兩個領域。</span><span class="sxs-lookup"><span data-stu-id="ba57f-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="ba57f-114">（您也可以執行伺服器對伺服器驗證，而不使用授權伺服器，本檔稍後會討論的程式。）安全權杖是由授權伺服器（也稱為安全權杖伺服器）頒發給需要通訊的兩個領域;這些權杖會確認源自某個領域的通訊應該受到其他領域的信任。</span><span class="sxs-lookup"><span data-stu-id="ba57f-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) Security tokens are issued by the authorization server (also known as a security token server) to the two realms that need to communicate; these tokens verify that communications originating from one realm should be trusted by the other realm.</span></span> <span data-ttu-id="ba57f-115">例如，授權伺服器可能會頒發標記，以驗證來自特定商務用 Skype 伺服器領域的使用者可以存取指定的 Exchange 領域，反之亦然。</span><span class="sxs-lookup"><span data-stu-id="ba57f-115">For example, the authorization server might issue tokens that verify that users from a specific Skype for Business Server realm are able to access a specified Exchange realm, and vice-versa.</span></span>
  
> [!NOTE]
> <span data-ttu-id="ba57f-116">領域只是一個安全性容器。</span><span class="sxs-lookup"><span data-stu-id="ba57f-116">A realm is simply a security container.</span></span> <span data-ttu-id="ba57f-117">根據預設，商務用 Skype Server 會使用預設的 SIP 網域作為其 OAuth 領域。</span><span class="sxs-lookup"><span data-stu-id="ba57f-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="ba57f-118">其他 SIP 命名空間會新增至 OAuth 憑證中的 [Subject 替換名稱] 清單。</span><span class="sxs-lookup"><span data-stu-id="ba57f-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="ba57f-119">商務用 Skype 伺服器支援三個伺服器對伺服器驗證案例。</span><span class="sxs-lookup"><span data-stu-id="ba57f-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="ba57f-120">在商務用 Skype Server 中，您可以：</span><span class="sxs-lookup"><span data-stu-id="ba57f-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="ba57f-121">在商務用 Skype Server 的內部部署安裝與 Exchange 與/或 SharePoint Server 的內部部署安裝之間，設定伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="ba57f-122">在一對 Office 365 元件（例如，在 Microsoft Exchange Server 與商務用 Skype Server 之間，或商務用 Skype Server 和 SharePoint 之間）設定伺服器對伺服器的驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-122">Configure server-to-server authentication between a pair of Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="ba57f-123">在跨內部部署環境中設定伺服器對伺服器驗證（也就是在內部部署伺服器與 Office 365 元件之間的伺服器對伺服器驗證）。</span><span class="sxs-lookup"><span data-stu-id="ba57f-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and an Office 365 component).</span></span>
    
<span data-ttu-id="ba57f-124">請注意，在此時間點，僅限 Exchange 2013、SharePoint Server、Lync Server 2013、商務用 skype Server 2015，以及商務用 Skype 2019 支援伺服器對伺服器驗證;如果您沒有執行其中一個伺服器，您將無法完全實現 OAuth 驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="ba57f-125">您也應該指出該伺服器對伺服器驗證是選用的：如果商務用 Skype 伺服器不需要與其他伺服器通訊（例如 Exchange），則可以完全略過伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="ba57f-126">如果已為 Lync Server 2013 和其他應用程式設定伺服器對伺服器驗證，就不需要針對商務用 Skype Server 重新執行此操作。</span><span class="sxs-lookup"><span data-stu-id="ba57f-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="ba57f-127">不過，如果您想要使用商務用 Skype Server （例如「整合連絡人存放區」）中的部分功能，則需要伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="ba57f-128">使用 [整合連絡人存放區] 時，商務用 Skype Server 連絡人資訊會儲存在 Exchange 中，而不是商務用 Skype Server 中;這可讓使用者擁有一組可在商務用 Skype、Outlook 或 Outlook Web Access 中輕鬆存取的連絡人。</span><span class="sxs-lookup"><span data-stu-id="ba57f-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; this enables users to have a single set of contacts that is readily accessible from within Skype for Business, Outlook, or Outlook Web Access.</span></span> <span data-ttu-id="ba57f-129">因為「整合連絡人存放區」需要商務用 Skype 伺服器與 Exchange 共用資訊，所以您必須使用伺服器對伺服器驗證，才能部署該功能。</span><span class="sxs-lookup"><span data-stu-id="ba57f-129">Because the unified contact store requires Skype for Business Server to share information with Exchange, you must use server-to-server authentication in order to deploy the feature.</span></span> <span data-ttu-id="ba57f-130">如果您選擇使用 Exchange 封存，且在其中將立即訊息會話的腳本儲存為 Exchange 電子郵件，而不是個別的資料庫記錄，也需要伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-130">Server-to-server authentication is also required if you choose to use Exchange archiving, in which the transcripts of instant messaging sessions are saved as Exchange emails rather than as individual database records.</span></span>
  
<span data-ttu-id="ba57f-131">若要將 Office 365 版本的商務用 Skype 伺服器與其 Exchange 對應專案通訊，商務用 Skype 伺服器必須先從授權伺服器取得安全權杖。</span><span class="sxs-lookup"><span data-stu-id="ba57f-131">For the Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="ba57f-132">商務用 Skype 伺服器然後使用該安全權杖來識別 Exchange 本身。</span><span class="sxs-lookup"><span data-stu-id="ba57f-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="ba57f-133">Office 365 版本的 Exchange 必須經過同一個處理常式，才能與商務用 Skype 伺服器進行通訊。</span><span class="sxs-lookup"><span data-stu-id="ba57f-133">The Office 365 version of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="ba57f-134">不過，對於兩個 Microsoft 伺服器之間的內部部署伺服器與伺服器驗證，不需要使用協力廠商的權杖伺服器。</span><span class="sxs-lookup"><span data-stu-id="ba57f-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="ba57f-135">伺服器產品（例如商務用 Skype Server 和 Exchange）有內建的權杖伺服器，可用於針對支援伺服器對伺服器驗證的其他 Microsoft 伺服器（例如 SharePoint Server）進行驗證。</span><span class="sxs-lookup"><span data-stu-id="ba57f-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="ba57f-136">例如，商務用 Skype 伺服器可以自行發出並簽署安全權杖，然後使用該權杖與 Exchange 進行通訊。</span><span class="sxs-lookup"><span data-stu-id="ba57f-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="ba57f-137">在這種情況下，不需要使用協力廠商的權杖伺服器。</span><span class="sxs-lookup"><span data-stu-id="ba57f-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="ba57f-138">若要為商務用 Skype Server 的內部部署實現設定伺服器對伺服器驗證，您必須執行兩個動作：</span><span class="sxs-lookup"><span data-stu-id="ba57f-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
  
- <span data-ttu-id="ba57f-139">將憑證指派給內建的商務用 Skype Server 權杖頒發者。</span><span class="sxs-lookup"><span data-stu-id="ba57f-139">Assign a certificate to the built-in Skype for Business Server token issuer.</span></span>
    
- <span data-ttu-id="ba57f-140">設定商務用 Skype 伺服器將與「合作夥伴應用程式」通訊的伺服器。</span><span class="sxs-lookup"><span data-stu-id="ba57f-140">Configure the server that Skype for Business Server will communicate with to be a "partner application."</span></span> <span data-ttu-id="ba57f-141">例如，如果商務用 Skype 伺服器需要與 Exchange 通訊，您必須將 Exchange 設定為合作夥伴應用程式。</span><span class="sxs-lookup"><span data-stu-id="ba57f-141">For example, if Skype for Business Server needs to communicate with Exchange, you will need to configure Exchange to be a partner application.</span></span>
