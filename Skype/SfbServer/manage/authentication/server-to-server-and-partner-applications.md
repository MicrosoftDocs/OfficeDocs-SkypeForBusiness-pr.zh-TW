<span data-ttu-id="612b8-104">**摘要：** 管理商務用 Skype Server 中的 OAuth 和夥伴應用程式。</span><span class="sxs-lookup"><span data-stu-id="612b8-104">**Summary:** Manage OAuth and partner applications in Skype for Business Server.</span></span>
  
<span data-ttu-id="612b8-105">商務用 Skype 伺服器必須能夠與其他應用程式和伺服器產品進行安全、順利的通訊。</span><span class="sxs-lookup"><span data-stu-id="612b8-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="612b8-106">例如，您可以設定商務用 Skype Server，使連絡人資料和（或）封存資料儲存在 Microsoft Exchange Server 2013 中;不過，只有在商務用 Skype 伺服器和 Exchange 能夠安全地相互通訊時，才可以進行此項操作。</span><span class="sxs-lookup"><span data-stu-id="612b8-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="612b8-107">同樣地，您可以從 Office Web Apps Server 中排程商務用 Skype Server 會議;同樣的，只有在兩部伺服器 (SharePoint 和商務用 Skype Server) 彼此信任時，才可以這麼做。</span><span class="sxs-lookup"><span data-stu-id="612b8-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="612b8-108">雖然您可以使用一種驗證機制來通訊商務用 Skype Server 與 Exchange，但適用于商務用 Skype Server 和 SharePoint 通訊的個別機制，但更好且更有效的方法，就是使用標準化的方法來進行所有伺服器對伺服器驗證和授權。</span><span class="sxs-lookup"><span data-stu-id="612b8-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="612b8-109">使用單一標準化的伺服器對伺服器驗證方法是商務用 Skype Server 所採用的方法。</span><span class="sxs-lookup"><span data-stu-id="612b8-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="612b8-110">開始使用 Office server 2013 版本、商務用 Skype Server (及其他 Microsoft Server 產品（包括 Exchange Server 和 SharePoint Server) 支援 OAuth (開放式授權) 通訊協定，以進行伺服器對伺服器驗證和授權。</span><span class="sxs-lookup"><span data-stu-id="612b8-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="612b8-111">藉由 OAuth (一些主要網站皆使用的標準授權通訊協定)，使用者認證及密碼並不會在電腦間傳遞。</span><span class="sxs-lookup"><span data-stu-id="612b8-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="612b8-112">驗證及授權是基於安全性權杖的交換，這些權杖會授與一段特定時間內一組特定資源的存取權。</span><span class="sxs-lookup"><span data-stu-id="612b8-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="612b8-113">OAuth 驗證通常涉及三方：單一授權伺服器及兩個需要互相通訊的領域。</span><span class="sxs-lookup"><span data-stu-id="612b8-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="612b8-114"> (您也可以執行伺服器對伺服器驗證，而不需使用授權伺服器，本檔稍後將會討論此程式。授權伺服器（也稱為安全性權杖伺服器）發出 ) 的安全性權杖， (也稱為安全性權杖伺服器) 至需要通訊的兩個領域;這些權杖會驗證來自某個領域的通訊是否應該由其他領域所信任。</span><span class="sxs-lookup"><span data-stu-id="612b8-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) > [!NOTE]
> <span data-ttu-id="612b8-116">領域只是安全性容器。</span><span class="sxs-lookup"><span data-stu-id="612b8-116">A realm is simply a security container.</span></span> <span data-ttu-id="612b8-117">根據預設，商務用 Skype 伺服器會使用預設 SIP 網域作為其 OAuth 領域。</span><span class="sxs-lookup"><span data-stu-id="612b8-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="612b8-118">其他 SIP 命名空間會新增至 OAuth 憑證中的主體替代名稱清單。</span><span class="sxs-lookup"><span data-stu-id="612b8-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="612b8-119">商務用 Skype 伺服器支援三種伺服器對伺服器驗證案例。</span><span class="sxs-lookup"><span data-stu-id="612b8-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="612b8-120">在商務用 Skype Server 中，您可以：</span><span class="sxs-lookup"><span data-stu-id="612b8-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="612b8-121">在內部部署商務用 Skype Server 與 Exchange 和/或 SharePoint 伺服器的內部部署安裝之間設定伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="612b8-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="612b8-122">設定一對 Microsoft 365 或 Office 365 元件之間的伺服器對伺服器驗證 (例如，在 Microsoft Exchange Server 與商務用 Skype Server 之間，或在商務用 Skype Server 與 SharePoint) 之間。</span><span class="sxs-lookup"><span data-stu-id="612b8-122">Configure server-to-server authentication between a pair of Microsoft 365 or Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="612b8-123">在跨單位環境中設定伺服器對伺服器的驗證 (也就是說，內部部署伺服器與 Microsoft 365 或 Office 365 元件之間的伺服器對伺服器驗證) 。</span><span class="sxs-lookup"><span data-stu-id="612b8-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and a Microsoft 365 or Office 365 component).</span></span>
    
<span data-ttu-id="612b8-124">請注意，目前只有 Exchange 2013、SharePoint Server、Lync Server 2013、商務用 Skype Server 2015 和商務用 Skype 2019 支援伺服器對伺服器驗證;如果您未執行這些伺服器之一，您將無法完全執行 OAuth 驗證。</span><span class="sxs-lookup"><span data-stu-id="612b8-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="612b8-125">您也應該指出，伺服器對伺服器驗證是選用的：如果商務用 Skype 伺服器不需要與其他伺服器通訊 (例如 Exchange) 則可以完全略過伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="612b8-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="612b8-126">如果已為 Lync Server 2013 和其他應用程式設定伺服器對伺服器驗證，則不需要為商務用 Skype Server 重新執行。</span><span class="sxs-lookup"><span data-stu-id="612b8-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="612b8-127">不過，如果您想要使用商務用 Skype Server 中的某些功能（例如「整合連絡人存放區」），則需要伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="612b8-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="612b8-128">使用整合連絡人存放區，商務用 Skype Server 連絡人資訊會儲存在 Exchange 中，而不是儲存在商務用 Skype Server 中;這可讓使用者從商務用 Skype、Outlook 或 Outlook Web Access 中輕鬆存取一組連絡人。</span><span class="sxs-lookup"><span data-stu-id="612b8-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; this enables users to have a single set of contacts that is readily accessible from within Skype for Business, Outlook, or Outlook Web Access.</span></span> <span data-ttu-id="612b8-129">由於整合連絡人存放區要求商務用 Skype Server 與 Exchange 共用資訊，因此您必須使用伺服器對伺服器驗證，才能部署該功能。</span><span class="sxs-lookup"><span data-stu-id="612b8-129">Because the unified contact store requires Skype for Business Server to share information with Exchange, you must use server-to-server authentication in order to deploy the feature.</span></span> <span data-ttu-id="612b8-130">如果您選擇使用 Exchange 封存（其中立即訊息會話的記錄會儲存為 Exchange 電子郵件，而不是個別的資料庫記錄），則也需要伺服器對伺服器驗證。</span><span class="sxs-lookup"><span data-stu-id="612b8-130">Server-to-server authentication is also required if you choose to use Exchange archiving, in which the transcripts of instant messaging sessions are saved as Exchange emails rather than as individual database records.</span></span>
  
<span data-ttu-id="612b8-131">若要讓 Microsoft 365 或 Office 365 版本的商務用 Skype Server 與其 Exchange 對應，商務用 Skype 伺服器必須先取得授權伺服器的安全性權杖。</span><span class="sxs-lookup"><span data-stu-id="612b8-131">For the Microsoft 365 or Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="612b8-132">然後，商務用 Skype 伺服器會使用該安全性權杖來識別 Exchange 本身。</span><span class="sxs-lookup"><span data-stu-id="612b8-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="612b8-133">Microsoft 365 或 Office 365 版本的 Exchange 必須經過相同的程式，才能與商務用 Skype 伺服器進行通訊。</span><span class="sxs-lookup"><span data-stu-id="612b8-133">The Microsoft 365 or Office 365 versions of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="612b8-134">不過，對於兩部 Microsoft 伺服器之間的內部部署伺服器對伺服器驗證，則不需要使用協力廠商權杖伺服器。</span><span class="sxs-lookup"><span data-stu-id="612b8-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="612b8-135">伺服器產品（例如商務用 Skype Server 和 Exchange）具有內建的權杖伺服器，可用於與其他 Microsoft 伺服器 (，例如支援伺服器對伺服器驗證的 SharePoint 伺服器) 。</span><span class="sxs-lookup"><span data-stu-id="612b8-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="612b8-136">例如，商務用 Skype Server 可以自行發行和簽署安全性權杖，然後使用該權杖與 Exchange 通訊。</span><span class="sxs-lookup"><span data-stu-id="612b8-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="612b8-137">在這種情況下，便不需要協力廠商權杖伺服器。</span><span class="sxs-lookup"><span data-stu-id="612b8-137">In a case like this, there is no need for a third-party token server.</span></span>
