# <a name="response-group-usage-report-in-skype-for-business-server"></a>商務用 Skype Server 中的回應群組使用方式報告

**摘要:** 瞭解商務用 Skype Server 中的回應群組應用程式。

[回應群組] 應用程式提供一種方式, 讓商務用 Skype Server 根據所撥打的號碼, 或在來電者對一系列問題的回復, 接聽電話。 通常, 回應群組通話不會傳送給個別人, 而是傳送給稱為「代理人」群組的人員小組。 例如, 如果某人撥打電話號碼給您的技術支援人員, 商務用 Skype Server 可以自動將呼叫傳送到第一個可用的技術支援中心代理程式。 或者, 商務用 Skype Server 可能會詢問一系列問題 (如果您遇到硬體問題, 請按1。 如果您遇到軟體問題, 請按2。 如果您有網路問題, 請按 3, 然後根據這些問題的答案, 將呼叫路由至最合適的技術支援人員。

[回應群組使用方式] 報告詳細瞭解所有回應群組工作流程接收到的電話數, 然後將這些通話中斷為更有限的類別, 例如提供的通話、接聽的通話, 以及已放棄的通話。

使用回應群組使用方式報告的關鍵是瞭解報告通話類型之間的差異:

- **已接收來電**。 所有回應群組應用程式實例接收到的呼叫總數。

- **成功的通話**。 回應群組應用程式所挑選的通話總數目。

- 已**提供來電**。 傳送給回應群組代理程式的呼叫總數。

- 已**接聽通話**。 已由回應群組代理程式實際接聽的通話總數目。

- 已**放棄通話的百分比**。 回應群組應用程式接收, 但沒有由代理程式接聽的通話百分比。 此值的計算方法是從收到的電話減去接聽的通話, 然後根據收到的通話數來除以該值。 例如, 如果您收到10個通話, 且已接聽7個, 您會從10減去 7, 然後離開3個未回復的通話。 這個值將除以 10, 從而讓您放棄通話百分比 30%。

- **轉接來電**。 由於佇列超時或佇列溢出而轉移之回應群組通話的總數。

如果您正在查看 [回應群組使用量] 報告, 而且不記得任何這些撥號類型的定義, 只要將滑鼠放在適當的通話類型標籤上即可。 隨即會出現工具提示, 其中提供通話類型的簡短描述。

[回應群組使用方式] 報告可讓您篩選工作流程 URI (與該工作流程相關聯的 SIP 位址)。 不過, 工作流程 Uri 實際上不會出現在報表本身。 如果您想知道哪些工作流程正在接聽最多來電, 或是哪些工作流程遇到最大的通話, 請按一下適當的度量單位, 以開啟該特定時段的回應群組通話清單報告。 該報告會列出工作流程 Uri。

## <a name="accessing-the-response-group-usage-report"></a>存取回應群組使用方式報告

[回應群組使用方式] 報告是從 [監控報告] 首頁存取。 您可以按一下下列任何一種度量,[在商務用 Skype Server 中向下切入 [回應群組通話清單] 報告](call-list-report.md):

- 已接聽通話

- 成功的通話

- 已提供通話

- 已接聽通話

- 已轉移通話

## <a name="making-the-best-use-of-the-response-group-usage-report"></a>充分利用 [回應群組使用量] 報告

[回應群組使用量] 報告的其中一個較有趣的用途, 可能不容易看出: 能夠針對單一回應群組工作流程檢索使用方式資訊。

> [!CAUTION]
> 回應群組工作流程基本上是一組指示在使用者撥打特定電話號碼時的商務用 Skype 伺服器所做的指示。 如此一來, 每個工作流程都會與電話號碼建立唯一的關聯。 當某人呼叫該號碼時, 工作流程會判斷通話的處理方式。 例如, 工作流程可能會將呼叫路由到一系列的互動式語音回應 (IVR) 問題, 提示來電者輸入其他資訊 (「按1取得硬體支援。 按下2取得軟體支援。 或者, 工作流程可能會導致呼叫放在佇列中, 而呼叫者則會停留在通話中, 直到有可用的工程師接聽通話。 您也可以使用工作流程來設定接聽電話的代理程式的可用性: [工作流程] 是用來設定兩個上班時間 (一周中的天數, 以及可供代理接聽之電話的時間) 和假日 (沒有代理程式可供應答的天數)。通話)。 任何時候只要您撥打屬於回應群組應用程式的電話號碼, 您就會實質呼叫回應群組工作流程。 

雖然工作流程 Uri 不會出現在 [回應群組使用量] 報告中, 但仍可以查看單一工作流程的使用方式統計資料, 這項功能通常相當有用。 例如, 假設您最近 unveiled 新的廣告活動, 而且想知道人員是否正在撥打電話給該產品。 如果您已將回應群組工作流程與廣告行銷活動中指定的電話號碼相關聯, 您可以輕鬆檢查是否有多少人 (如果有) 撥打該號碼。

您也可以使用類似的方法來測量由您的內部問訊台或客戶服務部門處理的通話數量。

若要查看特定工作流程的使用方式統計資料, 請在 [工作流程 URI] 方塊中輸入工作流程 URI。 當然, 如所述, 工作流程 Uri (與工作流程相關聯的 SIP 位址) 不會出現在報表上。 這表示您需要尋找一些其他方法來判斷工作流程的 URI。 其中一種方法是使用 Windows PowerShell 和商務用 Skype Server 管理命令介面。 例如, 這個命令會傳回所有回應群組工作流程的 Uri:

```
Get-CsRgsWorkflow | Select-Object Name, PrimaryUri
```

這樣會傳回如下所示的資料:

<pre>
Name                            PrimaryUri
----                            ----------
Customer Support                sip:support@litwareinc.com
Help Desk                       sip:helpdesk@litwareinc.com
New Ad Campaign                 sip:newads@litwareinc.com
</pre>

這個命令會傳回單一工作流程的資訊, 其中一個工作流程是 [新廣告活動] 的名稱:

```
Get-CsRgsWorkflow -Name "New Ad Campaign" | Select-Object Name, PrimaryUri
```
