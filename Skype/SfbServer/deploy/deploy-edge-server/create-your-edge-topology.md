# <a name="create-your-edge-topology-for-skype-for-business-server"></a>為商務用 Skype Server 建立邊緣拓撲
 
**摘要:** 瞭解如何在商務用 Skype Server 中建立、發佈及匯出 Edge 伺服器拓撲。
  
[拓撲建立器] 是您在建立邊緣伺服器拓撲時所需使用的工具, 就與商務用 Skype Server 的任何拓撲元件搭配使用一樣。 在遵循下列步驟之前, 您必須先設定至少一個前端池或標準版伺服器。
  
我們會在本文中涵蓋下列主題:
  
- 建立您的邊緣伺服器拓撲
    
- 發佈 Edge 伺服器拓撲
    
- 匯出邊緣伺服器拓撲
    
  > [!NOTE]
  > 若要遵循下列步驟, 您必須登入如下所述的網域伺服器, 就像是下列網域群組成員的使用者: 
  
- RTCUniversalServerAdmins
    
- 網域管理員
    
## <a name="build-your-edge-server-topology"></a>建立您的邊緣伺服器拓撲

第一個部署步驟是建立您的商務用 Skype Server Edge 伺服器拓撲, 其中包含三個選項的其中一個:
  
- 單一邊緣伺服器
    
- DNS 負載平衡的邊緣池 (一或多個伺服器)
    
- 硬體負載平衡的邊緣池 (一或多個伺服器)
    
如果您不確定所需的專案, 請在開始執行下列步驟之前, 請先走完規劃檔。 否則, 讓我們開始吧。
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>定義單一邊緣伺服器的拓撲

1. 登入您的商務用 Skype Server 標準版伺服器, 或商務用 Skype Server 前端池。
    
2. 在該處開啟 [**商務用 Skype 伺服器拓撲**建立器]。
    
3. 在主控台樹中, 展開您要部署邊緣伺服器的網站。
    
4. 以滑鼠右鍵按一下 [**邊緣池**], 然後按一下 [**新增邊緣池**]。
    
5. 按一下 [**定義新的邊緣池**] 畫面上的 **[下一步]** 。
    
6. 在 [**定義 edge 池 FQDN** ] 畫面上, 輸入邊緣伺服器要使用的內部完整功能變數名稱 (FQDN), 然後選取 [**單一電腦] 池**, 然後在 [完成時, 按一下 **[下一步]** 。
    
7. 在 [**選取功能**] 畫面上, 您可以選擇:
    
   - 您可能想要針對 SIP 存取服務、商務用 Skype Server Web 會議服務和您的 A/V 邊緣服務使用相同的 FQDN 和 IP 位址。 如果是這樣, 您必須選擇 [**使用單一 FQDN 和 IP 位址] 核取方塊**(並在下面的步驟9中繼續)
    
   - 如果您打算啟用同盟, 請選擇 [**針對此 Edge 池啟用同盟 (埠 5061)** ] 核取方塊。
    
8. 按一下 **[下一步]** 之後, 您會在 [ **IP 選項**] 畫面上找到自己。 選項如下所示:
    
   - 在內部介面上啟用 IPv4
    
   - 在內部介面上啟用 IPv6
    
   - 在外部介面上啟用 IPv4
    
   - 在外部介面上啟用 IPv6
    
   無論您使用的是 IPv4 或 IPv6 位址, 這些位址都一目了然, 而您是要在邊緣伺服器 (內部或外部) 套用這些位址 (在步驟10中, 您將需要記住這一點)。 您也可以選擇將 Edge 伺服器或 Edge 池設定為使用網路位址轉譯 (NAT) 位址作為外部 IP 位址。 您可以選擇 [**此 Edge 池的外部 IP 位址是由 NAT 轉譯**] 核取方塊來執行此動作。 準備就緒時, 按一下 **[下一步]** 。
    
9. 在 [外部 Fqdn] 畫面上, 您的選項取決於您在上述步驟7中所做的選取。
    
   - 如果您已核取 [**使用單一 FQDN 和 IP 位址**] 核取方塊, 您必須在 [ **SIP 存取**] 方塊中輸入單一的外部 FQDN。 接著, 您必須針對每個 edge 服務輸入不同的埠號碼, 以允許它們彼此獨立連線。 我們建議將5061用於**SIP 存取**邊緣服務、444的**網路會議**Edge 服務, 以及**A/V**邊緣服務的443。 完成時, 請按 **[下一步]** 。
    
   - 如果您沒有核取 [**使用單一 FQDN 和 IP 位址**] 核取方塊, 您現在必須為**SIP 存取**邊緣服務、**網路會議**Edge 服務和**a/V**邊緣服務輸入三個外部 fqdn。 完成時, 請按 **[下一步]** 。
    
10. 您現在已在 [**定義內部 IP 位址**] 畫面上。 在這裡, 您會在 [**內部 IPv4 位址**] 和 [**內部 IPv6 位址**] 文字方塊中輸入 Edge 伺服器的 IP 位址, 視您在步驟8中所做的選擇而定。 準備就緒時, 按一下 **[下一步]** 。
    
11. 在 [**定義外部 IP 位址**] 畫面上, 您可以選擇幾個選項 (視您先前的選項而定):
    
    - 您可能會在所有服務中使用單一 FQDN。 如果是, 請在 [ **SIP 存取**] 文字方塊中輸入您的外部 IPv4 或 IPv6 位址 (無論您使用的是哪一個), 然後按 **[下一步]**。
    
    - 您可能已選擇針對服務使用三個不同的 Fqdn 和 IP 位址。 如果是這種情況, 請為**SIP 存取**邊緣服務、**網路會議**Edge 服務和**A/V**邊緣服務輸入您的外部 IPv4 或 IPv6 位址, 然後按 **[下一步]**。
    
    > [!NOTE]
    > 如果您先前沒有選擇啟用並指派 IPv6 定址, 您就不會看到這個對話方塊。 那是正常的, 請移至下一個步驟。 
  
12. 如果您選擇在步驟8中使用 NAT, 您現在會看到一個畫面, 其中包含**公用 IP 位址**textbox。 您必須輸入您為 A/V Edge 服務設定的公用 IPv4 和/或 IPv6 位址, 才能由 NAT 進行翻譯。 然後按一下 **[下一步]**。
    
13. [下一個畫面向上] 是 [**定義下一個躍點]**。 在 [**下一個躍點池]** 方塊中, 選取您的內部池名稱, 這可能是前端池或獨立池。 如果您的環境中有主管, 您應該選擇控制器。 然後按一下 **[下一步]**。
    
14. 在 [**關聯前端池**] 畫面上, 您必須指定一個或多個內部池 (包括前端池和標準版伺服器), 才能與此 Edge 伺服器建立關聯。 只要選擇您想要使用此 Edge 伺服器與支援的外部使用者通訊的內部池名稱即可。 按一下 **[下一步]**。
    
    > [!NOTE]
    > 請記住, 如果您的內部池或獨立伺服器已使用不同的商務用 Skype Server Edge 伺服器, 則它們不能有多個關聯性。 如果您選擇在這種情況下的內部池或獨立伺服器, 您會看到一個警告, 告訴您其他邊緣伺服器, 您可以決定是否要繼續進行。 如果您要繼續這個新的關聯, 則與其他邊緣伺服器的連線將會停止。 
  
15. 按一下下一個畫面上的 **[完成]** 。
    
16. 現在, 您可以發佈此更新的技術, 並依照[商務用 Skype Server 中部署邊緣](deploy-edge-servers.md)伺服器中的指示, 從這裡部署到您的 Edge 伺服器。

12. 在 [**定義外部 IP 位址**] 畫面上, 您可以選擇幾個選項 (視您先前的選項而定):
    
    - 您可能會在所有服務中使用單一 FQDN。 如果是, 請在 [ **SIP 存取**] 文字方塊中輸入您的外部 IPv4 或 IPv6 位址 (無論您使用的是哪一個), 然後按 **[下一步]**。
    
    - 您可能已選擇針對服務使用三個不同的 Fqdn 和 IP 位址。 如果是這種情況, 請為**SIP 存取**邊緣服務、**網路會議**Edge 服務和**A/V**邊緣服務輸入您的外部 IPv4 或 IPv6 位址, 然後按 **[下一步]**。
    
    > [!NOTE]
    > 如果您先前沒有選擇啟用並指派 IPv6 定址, 您就不會看到這個對話方塊。 那是正常的, 請移至下一個步驟。 
  
13. 按一下 **[完成]**。 您剛建立的邊緣伺服器現在應該會列在 [**定義此池中的電腦**] 對話方塊中。
    
14. 當仍在 [**定義此 pool 的電腦**] 畫面中, 請再次按一下 [**新增**] 按鈕, 然後重複步驟11到 13, 直到您已新增您想在此池中擁有的所有邊緣伺服器為止。 完成後, 請按 **[下一步]**。
    
15. [下一個畫面向上] 是 [**定義下一個躍點]**。 在 [**下一個躍點池]** 方塊中, 選取您的內部池名稱, 這可能是前端池或獨立池。 如果您的環境中有主管, 您應該選擇控制器。 然後按一下 **[下一步]**。
    
16. 在 [**關聯前端池**] 畫面上, 您必須指定一個或多個內部池 (包括前端池和標準版池), 才能與此 Edge 伺服器建立關聯。 只要選擇您想要使用此 Edge 伺服器與支援的外部使用者通訊的內部池名稱即可。 按一下 **[下一步]**。
    
    > [!NOTE]
    > 請記住, 如果您的內部池或獨立伺服器已使用不同的商務用 Skype Server Edge 伺服器, 則它們不能有多個關聯性。 如果您選擇在這種情況下的內部池或獨立伺服器, 您會看到一個警告, 告訴您其他邊緣伺服器, 您可以決定是否要繼續進行。 如果您要繼續這個新的關聯, 則與其他邊緣伺服器的連線將會停止。 
  
17. 按一下下一個畫面上的 **[完成]** 。
    
18. 現在, 您可以發佈此更新的技術, 並依照[商務用 Skype Server 中部署邊緣](deploy-edge-servers.md)伺服器中的指示, 從這裡部署到您的 Edge 伺服器。
    
## <a name="publish-your-edge-server-topology"></a>發佈 Edge 伺服器拓撲

完成上述步驟後, 就可以發佈這個新的拓朴, 這也會讓您將它匯出到商務用 Skype Server Edge 伺服器或 Edge 池。 請依照下列步驟進行:
  
1. 啟動**拓撲**建立器 (如果尚未從上一個程式開始)。
    
2. 在**拓撲**建立器中, 于主控台樹狀結構中, 以滑鼠右鍵按一下 [**商務用 skype 伺服器**], 然後按一下 [**商務用 skype server 拓撲**建立器]。
    
3. 在嚮導的 [**歡迎**] 頁面上, 按一下 **[下一步]**。
    
4. 在 [**建立其他資料庫**] 頁面上, 按一下 **[下一步]**。
    
5. 當狀態指出資料庫建立成功時, 請執行下列動作:
    
    - 若要查看記錄, 請按一下 [**查看記錄**]。
    
    - 若要關閉嚮導, 請按一下 **[完成**]。
    
## <a name="export-your-edge-server-topology"></a>匯出邊緣伺服器拓撲

若要成功部署, 商務用 Skype Server 部署嚮導需要存取中央管理儲存區資料。 對於網域或樹林中的內部伺服器來說, 這通常相當簡單。 Edge 伺服器位於網域以外, 因此您必須手動將拓撲檔案匯出到 Edge 伺服器位置, 通常是在物理媒體上。 此匯出是透過 PowerShell 完成:
  
1. 啟動**商務用 Skype Server 管理命令**介面。
    
2. 在**商務用 Skype Server Management 命令**介面中, 執行下列動作:
    
   ```
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. 將匯出的檔案複製到外部媒體 (例如, 您可以從邊緣伺服器位置存取的 USB 磁片磁碟機或網路共用)。
