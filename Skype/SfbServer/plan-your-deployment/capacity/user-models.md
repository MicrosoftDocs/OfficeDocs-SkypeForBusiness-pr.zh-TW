# <a name="user-models-in-skype-for-business-server"></a>商務用 Skype Server 中的使用者模型
 
此處所述的使用者模型針對[商務用 Skype Server 的容量規劃使用者模型使用量](user-model.md)中所述的容量規劃測量與建議，提供基礎。
  
## <a name="skype-for-business-server-user-models"></a>商務用 Skype Server 使用者模型

下表描述 [註冊]、[連絡人]、[立即訊息（IM）] 和 [商務用 Skype 伺服器的目前狀態] 使用者模型。
  
**環境與註冊使用者模型**

|**類別**|**說明**|
|:-----|:-----|
|部署大小與發佈  <br/> |我們會針對具有三個中心網站的大型部署建模，每個網站都有一個前端池。  <br/> |
|Active Directory 使用者的百分比  <br/> |我們假設組織中所有 Active Directory 使用者的70% 都已啟用商務用 Skype Server。 80% 的使用者每天都會登入商務用 Skype Server （80% concurrency）。 併發使用者是本節其餘部分中的數位的基礎。  <br/> |
|Active Directory 變更  <br/> |我們假設在每週的 Active Directory 中為商務用 Skype 建立並啟用總使用者的0.5%，0.5 而使用者的總數是從 Active Directory 和商務用 Skype 中停用。 5% 的使用者每週至少有一個 Active Directory 屬性已變更。  <br/> |
|Active Directory 通訊群組  <br/> |我們假設組織中的 Active Directory 通訊群組數目等於 Active Directory 中所有使用者數的三倍。 通訊群組具有下列大小：  <br/> •64% 有2-30 個使用者  <br/> •13% 有31-50 使用者  <br/> •10% 有51-100 使用者  <br/> •13% 有101-500 使用者  <br/> |
|[語音 over IP （VoIP）] 使用者  <br/> |商務用 Skype Server 使用者的60% 已啟用統一通訊（UC）（也就是他們的電話號碼是由商務用 Skype Server 所擁有）。  <br/> |
|已註冊的用戶端分配  <br/> |65% 的用戶端執行商務用 Skype 軟體，包括商務用 Skype 和 Lync 手機版。  <br/> 從舊版 Lync 執行用戶端軟體之客戶的30%。  <br/> 5% 的用戶端使用商務用 Skype Web App。  <br/> 如果行動已啟用，我們會假設40% 的使用者正在與其他先前提及的註冊用戶端選項同時使用行動。 在這種情況下，用戶端的多重狀態點（MPOP）比率為1：1.9。 如果行動已停用，則 MPOP 比率為1：1.5。  <br/> |

## <a name="busy-hour"></a>Busy 小時

對於點對點工作階段，峰值負載是使用繁忙時間的呼叫企圖（BHCA）來計算。 這個語音產業字假設當天所有通話的50% 將在20% 的時間內完成。 它是使用下列公式來計算：
  
 `BHCA=(total calls * 0.5) / 1.6`
  
在執行 VoIP 及其他點對點工作階段時，請在24小時內的每日至少1.6 小時負載中執行 VoIP 及其他點對點工作階段，以進行效能測試。
  
「會議峰值負載」假設所有會議的75% 是在4個小時的時間內完成。 這些尖峰時間的平均會議負載是1.5。
  
## <a name="enterprise-voice-to-pstn-calls"></a>企業語音到 PSTN 通話

下列假設適用于企業語音通話：
  
- 60% 的使用者已啟用企業語音功能，而這些使用者的60% 已啟用 PSTN 通話。
    
- 每個啟用 PSTN 通話的使用者都會在繁忙時間內撥打4個 PSTN 通話。 每個通話持續時間為3分鐘。
    
- 這些 PSTN 語音通話的65% 使用媒體旁路。
    
## <a name="mobility"></a>行動性

系統會假設已登錄使用者的40% 是啟用行動。 針對已啟用行動裝置的每位使用者，我們假設行動用戶端的活動會累加至該使用者的其他 MPOP 實例，除了會議互動以外，其行動用戶端只是另一個用戶端類型，可以用來參與會議。
  
## <a name="persistent-chat"></a>常設聊天室

我們假設有25% 的註冊使用者將參與持續聊天會話，且具有下列特性：
  
- 每位使用者1.5 聊天室的平均值
    
- 每個聊天室會每小時產生12個輪詢要求，平均目標為10個使用者
    
## <a name="response-group-and-call-park"></a>回應群組和通話駐留

我們假設已註冊的使用者0.15% 是屬於回應群組。 我們假設0.02% 的已註冊使用者已在任何指定時間點停用通話。
