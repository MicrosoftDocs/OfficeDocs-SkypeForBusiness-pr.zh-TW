# <a name="plan-for-monitoring-in-skype-for-business-server"></a>在商務用 Skype Server 中規劃監視

**摘要：** 在規劃商務用 Skype Server 中的 [監視服務] 時，請複習本主題。

商務用 Skype Server 中的 [監視服務] 提供一種方式，可讓系統管理員收集組織中發生之通訊會話的使用量與品質資料，讓他們能夠識別趨勢與問題。 持續監控您的部署可讓您儘早發現問題，並讓貴組織的使用者感到滿意。

商務用 Skype Server 的監控不需要個別的伺服器角色（在舊版 Lync 版本中的情況一樣）;相反地，監視服務是內嵌在每個前端伺服器中。 在商務用 Skype Server 中，預設不會啟用監視。 本文將協助您決定是否要在初始商務用 Skype 伺服器設定期間或之後啟用監視，以及需要哪些 SQL 資源才能支援監視活動。 如果您不確定是否完全符合或未監視，以及監控有何説明，請參閱監視的[基本知識](monitoring.md#Basics)。 若要開始規劃程式，請移至[定義您的監視需求](monitoring.md#requirements)。 如需監視之 SQL 需求的詳細資訊，請參閱[監視的 sql 需求](monitoring.md#topologies)。

## <a name="basics-about-monitoring"></a>關於監視的基本概念
<a name="Basics"> </a>

會話是使用者連線至下列專案的一般期限：

- 會議

- 音訊/視頻或應用程式共用等會議工具

- 透過對等交談（例如立即訊息或語音通話）的另一個使用者

> [!NOTE]
> 商務用 Skype 伺服器會保持追蹤每個會話的相關資訊：誰稱為誰;會話中使用了哪些端點;此會話的持續時間為多久;會話的感知品質為何;等等。 商務用 Skype 伺服器不會錄製並儲存實際通話本身。 包括立即訊息會話：雖然商務用 Skype 伺服器會記錄立即訊息會話的相關資訊，但它不會維護在會話期間傳送的每一條立即訊息的記錄。

每個會話的商務用 Skype Server 所收集的基本呼叫詳細資訊可用於：

- **投資回報率（ROI）** 分析。 系統管理員可以將使用資料與針對其先前的電話系統所收集的類似資料進行比較，以顯示成本節約，並協助論證商務用 Skype Server 的部署。

- **裝置庫存管理**。 資產管理資訊可協助系統管理員識別仍在使用中的舊裝置，這些裝置需要進行取代，且找出未使用或未使用的大量裝置。

- **技術支援人員**。 疑難排解資料可協助工程師判斷使用者的呼叫失敗的原因，而不需要收集伺服器或用戶端記錄。 對於不具備商務用 Skype 用戶端及商務用 Skype Server 相關技術知識的支援人員，您可以隨時存取及理解此資訊。

- **系統疑難排解**。 讓系統管理員能夠偵測到可能造成使用者無法執行基本工作（例如加入會議、建立通話或傳送立即訊息）的主要問題。

[監視] 也提供允許 SIP 端點（例如商務用 Skype）提供管理員不需存取的疑難排解資訊的機制：

- **影響品質的媒體度量**單位。 這些度量單位會涉及來電本身的實際傳輸;它們提供一種旅行記錄做為網路上的呼叫 journeys。 這些標準（包括資料包遺失、抖動及往返行程時間等）會提供通話在撥打電話給其他人終點的時間時，所發生問題的相關資訊。

- **向最終使用者報告的問題**。 這些指標包括在下列情況下，商務用 Skype 向最終使用者顯示的品質較差的通知：聲音太遠、出現太柔、網路連線較差或品質不佳，因為其他程式在電腦正在佔用可用的資源。

- **環境資訊**。 這些指標詳細說明通話品質因素，例如使用麥克風與喇叭類型、使用者是否透過 VPN 連線，以及使用者是否在無線連線。

每次通話結束時，SIP 相容端點都會將這項資訊傳送到主持通話的前端伺服器。 您不需要執行任何動作，即可取得端點以傳送該資訊;該行為是在 SIP 通訊協定內建。 不過，如果您想要收集並儲存該資訊，您必須安裝並啟用監視。 如果您已安裝並啟用監視，則呼叫資訊是由在前端伺服器上執行並中繼到一組 SQL Server 資料庫的代理程式所收集。 監視服務（以 "整合資料集合代理程式" 的形式）是 collocated 到所有前端伺服器。

## <a name="define-your-requirements-for-monitoring"></a>定義監視的需求
<a name="requirements"> </a>

在開始使用商務用 Skype 伺服器進行安裝與設定監視之前，仍需要解決幾個主要問題：

 **您何時要安裝監視？** 您可以在安裝和設定商務用 Skype Server 的同時安裝及設定監視;商務用 Skype Server 部署嚮導將提供機會，讓您能夠在安裝期間將前端池與監視資料庫建立關聯。 或者，您也可以在已安裝商務用 Skype Server 本身之後安裝監視;您可以使用拓撲建立器，將前端池和伺服器與監視資料庫建立關聯，然後發佈已修改的拓撲。

請記住，必須先安裝並設定 SQL Server，然後才能部署和設定監視。 不過，您只需要部署 SQL Server 本身，當您發佈商務用 Skype Server 拓撲時，系統會為您建立監視資料庫。

 **您想要監視何種類型的資料？** 商務用 Skype Server 可讓您監視兩種一般類型的資料：通話詳細錄製（CDR）資料和體驗品質（QoE）資料。 通話詳細資料錄製提供一種方式，讓您追蹤商務用 Skype 伺服器功能（例如，語音 over IP （VoIP）電話通話）的使用方式;立即訊息（IM）;檔案傳輸;音訊/視頻（A/V）會議;與應用程式共用會話。 這項資訊可協助您瞭解使用的是哪一種商務用 Skype Server 功能（以及哪些不是），也會提供這些功能的使用時機資訊。 經驗品質資料可讓您維護貴組織中音訊和視頻通話品質的記錄，包括網路資料包遺失、背景雜色及「抖動」（資料包延遲的差異）等專案。

如果您選擇要在商務用 Skype Server 中啟用監視，您可以同時啟用 CDR 監視與 QoE 監視，或者，您也可以選擇啟用一種監視，同時讓其他類型保持停用。 例如，假設您的使用者只使用立即訊息和檔案傳輸，且不進行音訊或視頻通話。 在這種情況下，可能無需啟用 QoE 監視。 同樣地，商務用 Skype 伺服器可讓您輕鬆啟用並在部署監視後停用監視。 例如，您可以選擇部署監視，但不停停 QoE 監視。 如果您的使用者開始遇到音訊或視頻通話的問題，您可以啟用 QoE 監視，並使用該資料來協助您疑難排解並解決這些問題。

在安裝商務用 Skype server 之後，當您安裝 [商務用 Skype Server] 時，沒有特殊的優點（或缺點），無法安裝監視。 要記住的一點是，在您安裝監視之前，您必須先選取電腦來託管後端監視存放區，而且在該電腦上必須安裝並設定受支援版本的 SQL Server，才能使用該電腦進行監視. 如果您已在電腦上安裝 SQL Server，且該電腦已準備好使用，您可以在安裝商務用 Skype Server 時安裝監視。 如果您沒有後端電腦準備就緒，您可以繼續自行安裝商務用 Skype Server，然後在後端電腦準備好使用時安裝監視。

 **您需要多少個後端監視資料庫？** 據估計，監控和封存的 collocated 資料庫可能支援240000商務用 Skype Server 使用者）。 此外，單一監視資料庫可由多個前端池使用;如果您的組織有三個前端池，您可以將這三個池與同一個後端書店建立關聯。

對於許多組織而言，決定所需的後端監視資料庫數量時，資料庫容量不會是決定因素。 相反地，更重要的考慮因素可能是網路速度。 假設您有三個前端池，但是其中一個池是在緩慢的網路連線中找到。 在這種情況下，您可能會想要使用兩個監視資料庫：一個資料庫以使用良好的網路連線來為兩個池提供服務，並使用一個單獨的資料庫，以較慢的網路連線提供該池的服務。

您也應該考慮到商務用 Skype 伺服器支援使用鏡像資料庫。 「資料庫鏡像」可讓您同時維護資料庫的兩份複本，並讓每個資料庫都駐留在不同的伺服器上。 任何時候，都會將資料寫入到主要資料庫，而這種資料也會寫入到鏡像資料庫中。 如果主資料庫應該失敗或無法使用，您可以使用簡單的商務用 Skype Server PowerShell 命令，將 [容錯移轉] 到鏡像資料庫。 例如：

```PowerShell
Invoke-CsDatabaseFailover -PoolFqdn atl-cs-001.litwareinc.com -DatabaseType "Monitoring" -NewPrincipal "Mirror"
```

這對於規劃的目的非常重要，因為鏡像會要求您將所需的資料庫數加倍：除了每個主資料庫之外，您還需要第二個資料庫作為鏡像。

 **您的商務用 Skype 伺服器網站是否需要自己的自訂監視設定？** 當您安裝商務用 Skype Server 時，也會安裝 CDR 及 QoE 設定的全域集合;這些全域集合讓您能夠將相同的 CDR 與 QoE 設定套用到整個組織。 在許多情況下，這會足夠：您通常會想要為所有的使用者啟用 CDR 監視。

不過，您也可能會想要將不同的設定套用至不同的網站。 例如，您可能想要在雷德蒙網站中使用 CDR 與 QoE 監視，但只在您的都柏林網站中使用 CDR 監視。 同樣地，您可能想要在雷德蒙者網站保留60天的監控資料，但只需要在都柏林網站中維護此類型的資料。 商務用 Skype Server 可讓您在網站範圍中建立不同的 CDR 及 QoE 設定設定;這可讓您以不同方式管理每個網站。 （這包括啟用和停用監控，以及設定管理設定，例如要保留多久資料。）

請注意，您可以在部署監視之前或在部署監視之後進行此決策。 例如，您可以部署監視，然後使用 [全域設定] 管理整個組織。 如果您稍後改變主意，您可以針對雷德蒙者的網站建立一組不同的設定，然後使用這些設定來管理雷德蒙者的監視。 （在網站範圍套用的設定，總是優先于在全域範圍中套用的設定。）如果您再次改變主意，您可以直接刪除套用至雷蒙德網站的設定設定。 移除網站設定的集合之後，全域設定集合就會自動套用到該網站。

## <a name="sql-requirements-for-monitoring"></a>監視的 SQL 需求
<a name="topologies"> </a>

當您啟用監視時，系統會自動在每個前端伺服器上安裝並啟用統一資料收集代理程式。 如需支援的 SQL Server 版本及其他詳細資料，請參閱[商務用 Skype server 2015 的伺服器需求](requirements-for-your-environment/server-requirements.md)

監視資料可以與其他類型的資料共用 SQL Server 實例。 通常，通話詳細資料記錄資料庫（LcsCdr）和體驗資料庫品質（QoEMetrics）會共用相同的 SQL 實例;在相同的 SQL 實例與封存資料庫（LcsLog）相同的情況下，通常也是這兩個監視資料庫。 關於 SQL Server 實例唯一的真正需求是，任何一個 SQL Server 實例都僅限於下列專案：

- 商務用 Skype Server 2015 後端資料庫的一個實例。 （作為一般規則，建議您不要在相同的 SQL 實例（甚至是同一台電腦）上 collocated 您的監視資料庫，就像是後端資料庫。 雖然技術上有可能，您可以使用後端資料庫所需的磁碟空間來執行監視資料庫的風險。

- [通話詳細資料記錄] 資料庫的一個實例。

- 體驗資料庫品質的一個實例。

- 存檔資料庫的一個實例。

換句話說，在同一個 SQL Server 實例中，您無法使用 LcsCdr 資料庫的兩個實例。 如果您需要 LcsCdr 資料庫的多個實例，則需要設定多個 SQL Server 實例。
