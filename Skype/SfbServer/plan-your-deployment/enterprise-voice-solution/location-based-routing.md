# <a name="plan-for-location-based-routing-in-skype-for-business"></a>在商務用 Skype 中規劃以位置為基礎的路由

規劃商務用 Skype Server Enterprise Voice 中以位置為基礎的路由, 包括與同時撥打和委派的互動, 以及支援位置的路由案例。

以位置為基礎的路由可以根據通話中各方的位置, 限制 VoIP 端點與 PSTN 端點之間的通話路由。 以位置為基礎的路由是通話管理的功能, 可控制如何透過商務用 Skype 伺服器路由通話。 它會根據商務用 Skype 來電者的地理位置, 強制呼叫授權規則, 以瞭解通話是否可以路由到 PBX 或 PSTN 端點。

以位置為基礎的路由引進了一組新的規則, 可修改國內和國際 PSTN 通話的路由, 以防止避開長途電話。 以位置為基礎的路由提供了將這些規則限定為特定區域、特定閘道或只有特定使用者組的彈性。

下列案例說明主要可以強制執行的限制式基礎路由類型:

- 出局通話-以位置為基礎的路由會將撥出電話強制到與呼叫者避免 PSTN 免付費旁路之區域的 PSTN 閘道, 這可防止撥出到與來電者位於不同區域的 PSTN 閘道。

- 入口通話-位置路由可以防止撥出 PSTN 呼叫撥打商務用 Skype 端點, 如果 PSTN 閘道路由撥入通話不在與已呼叫的商務用 Skype 使用者相同的區域中。

- 未知區域-以位置為基礎的路由會將傳入和傳出 PSTN 呼叫傳送給與不確定位置的使用者 (亦即從網際網路連線或位於未知區域的遠端使用者)。

- 國際區域-以位置為基礎的路由如果找不到使用者位置的閘道, 就會透過國際 PSTN 閘道來傳送撥出通話。

## <a name="guidance-for-where-to-apply-location-based-routing"></a>在何處套用以位置為基礎的路由的指導方針

視情況而定, 以位置為基礎的路由, 可以套用在使用者的端點網路網站位置或 PSTN 閘道的網路網站位置。 本主題提供如何套用以位置為基礎的路由的指導方針。

### <a name="applying-location-based-routing-at-the-users-location"></a>在使用者的位置套用以位置為基礎的路由

以位置為基礎的路由利用由 E9 使用的商務用 Skype 伺服器 (由 1-1、CAC 和媒體旁路) 所定義的相同網路區域、網站和子網, 來套用呼叫路由限制, 以避免 PSTN 免付費旁路。 使用者的位置是由使用者的商務用 Skype 端點的 IP 子系連線所決定。 每個 IP 子網都與一個網路網站相關聯, 該網路網站匯總成系統管理員定義的網路區域。 根據使用者的網路網站, 強制執行以位置為基礎的路由。

根據每個網路網站, 會套用以位置為基礎的路由規則, 這表示一組指定的規則將會套用至在相同網路網站內以位置為基礎的路由所啟用的所有端點。 系統管理員可以將以位置為基礎的路由套用到需要它的網路網站。

您可以在每個網路網站上定義語音路由策略, 以定義特定 PSTN 閘道, 該區域應該由位於網路網站中的所有使用者使用, 以呼叫 PSTN 電話號碼。 當使用者位於已啟用位置路由的網路網站時, 此類語音路由策略會優先于使用者語音原則所定義的路由, 而且會防止透過其他已啟用的 PSTN 閘道進行呼叫路由以位置為基礎的路由。 當商務用 Skype 使用者發出 PSTN 通話時, 使用者的語音原則會判斷使用者是否有權撥打電話。 如果使用者的語音原則允許使用者撥打電話, 則以位置為基礎的路由決定要傳出撥打電話的 PSTN 閘道。 以位置為基礎的路由依據使用者的位置來決定此判斷。

使用者位置可以使用下列方式進行分類:

- 使用者位於已啟用位置路由的已知網路網站, 且其已執行 (直向內撥號) 號碼會在位於相同網路網站 (亦即辦公室) 的 PSTN 閘道上終止。 [撥出電話] 的路由將是透過使用者所在之網路網站的語音路由原則。 對使用者的內向 PSTN 呼叫會路由到與 PSTN 閘道位於相同網路網站的端點。

- 使用者位於與 PSTN 閘道所在網路網站不同的已知網路網站中。 (亦即, 使用者出差至其他公司辦公室)。 [撥出電話] 的路由將是使用使用者所在網路網站的語音路由策略。 對使用者的內向 PSTN 呼叫將不會路由至 PSTN 閘道以外的端點, 以避免 PSTN 長途電話。

- 當使用者位於商務用 Skype Server 部署無法使用的網路網站時, 傳出通話的路由將根據指派給使用者的語音原則, 而不是系結到以位置為基礎的路由限制。 傳入的 PSTN 呼叫將不會路由至位於未知網路網站的端點, 以避免 PSTN 長途電話繞過。

### <a name="applying-location-based-routing-at-the-pstn-gateways-location"></a>在 PSTN 閘道的位置套用以位置為基礎的路由

透過 PSTN 閘道和 Pbx 路由的通話, 可能需要以位置為基礎的路由限制 (視這類系統的位置而定)。 在每個幹線的細微性, 都可以啟用以位置為基礎的路由。

在主幹上啟用時, 以位置為基礎的路由會引入下列一組規則:

- 針對每個幹線啟用位置路由時, 在該主幹上定義的規則將只會套用到透過該主幹路由的通話。

- 若要避免 PSTN tolls 繞過從網路網站產生的位置, 而該網路網站是 PSTN 閘道所在的網路網站, 則以位置為基礎的路由會引入網路網站與指定主幹的關聯性。 這會定義允許呼叫路由至指定主幹的網路網站。

您可以透過以下兩種方式, 為基於位置的路由啟用 Trunks:

- 幹線是針對 egresses 對 PSTN 的呼叫的 PSTN 閘道所定義。 由這種類型的幹線路由的撥入通話將只路由到與幹線位於相同網路網站內的端點。

- 主幹是針對不在靜態位置 (即 PBX 手機) 中使用舊版手機的中繼伺服器對等進行的中繼伺服器對等定義。 針對這個特殊設定, 由這種類型的幹線路由的所有撥出電話, 都會被認為是來自同一個網路網站作為主幹。 來自 PBX 使用者的呼叫將會與與主幹相同網路網站中的商務用 Skype 使用者擁有相同的以位置為基礎的路由強制執行。 如果在個別網路網站中的兩個 PBX 系統是透過商務用 Skype 伺服器連線, 則以位置為基礎的路由將允許從一個網路網站的一個 PBX 端點路由到其他網路網站中的另一個 PBX 端點。 這個案例不會受到以位置為基礎的路由封鎖。 除了這種情況以及在相同位置中與商務用 Skype 使用者類似的方式, 與此設定連線到中繼伺服器對等的端點, 都能撥打或接聽其他不路由呼叫 t 的中繼伺服器對等電話o PSTN (亦即連接至不同 PBX 的端點), 不論是與中繼伺服器對等相關的網路網站。 所有的撥入通話、撥出通話、來電轉接與涉及 PSTN 端點的呼叫轉寄, 都會受到基於位置的路由, 只使用定義為本機到此類中繼伺服器對等的 PSTN 閘道。

## <a name="scenarios-for-location-based-routing"></a>位置基礎路由案例

以位置為基礎的路由在下列情況下進行路由通話時, 會套用下列一般規則。

### <a name="outgoing-calls"></a>撥出電話

針對位置路由的使用者的出站呼叫路由會受到使用者端點的網路位置的影響。 下表說明根據呼叫者終點的位置, 以位置為基礎的路由會如何影響傳出通話的路由。

**來電者撥入到 PSTN 的電話**

||**使用者端點位於已啟用位置路由的網路網站中**|**使用者端點位於未知的網路網站中, 或未啟用以位置為基礎的路由**|
|:-----|:-----|:-----|
|撥出通話的授權  <br/> |根據使用者的語音原則, 進行撥號授權  <br/> |根據使用者的語音原則, 進行撥號授權  <br/> |
|傳送撥出通話  <br/> |根據網路網站的語音路由策略來傳送通話  <br/> |通話是依據使用者的語音原則傳送, 只能透過 trunks (如果有的話) 啟用。  <br/> |

### <a name="incoming-calls"></a>來電

將來電路由至啟用位置路由的使用者, 視使用者端點的位置而定。 傳入通話的路由會受到下列方式的影響。 如果使用者有來電至位於以位置為基礎的路由的網路網站中的端點, 且端點與 PSTN 閘道位於相同的網路網站中, 通話就會路由。 如果使用者有來電至位於支援位置路由的網路網站中的端點, 且端點所在的網路網站與 PSTN 閘道不同, 則不會路由該通話。 當使用者沒有位於與來電所在的 PSTN 閘道相同網路網站中的端點時, 撥入電話會直接路由至使用者的語音信箱, 而未接來電通知將會傳送給呼叫方。

已啟用針對位置路由的使用者的來電轉接設定將會繼續生效, 不過, 轉寄的呼叫將會受到使用者的位置式路由限制。

下表說明根據被呼叫者終點的位置, 以位置為基礎的路由會如何影響輸入通話的路由。 PSTN 閘道的網路網站是針對位置路由而啟用, 且以位置為基礎的路由只允許將 PSTN 呼叫路由至同一網路網站中的端點。

**被呼叫者從 PSTN 接收入站通話**

||**被呼叫者的端點位於與 PSTN 閘道相同的網路網站中**|**被呼叫者的端點與 PSTN 閘道不在同一個網路網站中**|**被呼叫方的端點位於未知的網路網站, 或未啟用以位置為基礎的路由**|
|:-----|:-----|:-----|:-----|
|傳送進貨 PSTN 通話  <br/> |來電是路由到被叫方的端點  <br/> |來電未路由至被叫方的端點  <br/> |來電未路由至被叫方的端點  <br/> |

|商務用 Skype Mobile  <br/> |聯盟使用者  <br/> |同盟使用者是針對商務用 Skype Mobile 位置的電話路由使用者進行語音通話;商務用 Skype Mobile 參與方升級至 PSTN 使用者。  <br/> |通話遭到封鎖。  <br/> |
|商務用 Skype Mobile  <br/> |聯盟使用者  <br/> |聯盟使用者是針對商務用 Skype Mobile 位置的 VoIP 呼叫, 也就是其中一個參與方升級至 CAA 連絡人。  <br/> |已升級的通話遭到封鎖, 且有適當的錯誤訊息。  <br/> |
|商務用 Skype Mobile  <br/> |聯盟使用者  <br/> |同盟使用者是針對商務用 Skype Mobile 位置的 VoIP 呼叫, 且同盟使用者升級至 PSTN 使用者。  <br/> |根據聯盟使用者的位置路由, 允許或不允許進行升級。 商務用 Skype Mobile 位置的路由使用者應用程式不會執行任何動作。  <br/> |

### <a name="delegation"></a>委派

商務用 Skype 中的委派功能會依下列方式受到以位置為基礎的路由影響:

- 當您為以位置為基礎的路由啟用代理人代表管理員進行呼叫時, 代理人的語音原則會用來授權通話, 而代理人的網站語音路由原則則將用於路由通話

- 針對主管的內送 PSTN 呼叫, 適用于來電轉接或同時撥打的相同規則將按照來電轉接與轉移及同時撥打主題中的說明進行。

- 當代理人將 PSTN 端點設定為同時振鈴目標時, 針對來電給 manager, 就會使用與內向幹線相關的網站語音路由原則, 將呼叫路由到代理人的 PSTN 端點。

- 若是委派, 建議管理員與他的相關委派通常位於同一個網路網站中。

## <a name="other-planning-considerations"></a>其他規劃考慮

規劃位置路由時, 您應該考慮下列案例的影響。

### <a name="disaster-recovery"></a>災害復原

在從主要池到備份池的容錯移轉期間, 以及將一般作業還原到主要池時, 在災難與恢復程式期間一直都要執行位置式路由。

### <a name="survivable-branch-appliance"></a>Survivable 分支裝置

設定以位置為基礎的路由會影響您部署與 Survivable 分支裝置相關聯之閘道的位置規劃。 與您的 SBA 關聯的閘道必須與您的 Survivable 分支裝置位於同一個網路網站;否則, 駐留在您 Survivable 分支裝置的使用者將不能在以位置為基礎的路由設定的情況下, 發出出站通話。 當您的 Survivable 分支裝置與中央網站之間的 WAN 連線為關閉狀態時, 會繼續強制執行以位置為基礎的路由限制。

## <a name="capabilities-not-supported-by-location-based-routing"></a>未受位置基礎路由支援的功能

以位置為基礎的路由不適用於下列類型的互動。 當商務用 Skype 端點與使用這些功能的 PSTN 端點互動時, 不會強制執行位置路由。

- PSTN 撥入會議

- 透過回應群組撥入和撥出 PSTN 電話

- 透過通話駐留撥打電話給寄存或檢索 PSTN 電話

- 打入的 PSTN 通話至宣告服務

- 透過群組通話挑選來檢索的打入 PSTN 電話

若要針對下列清單中的互動類型強制執行以位置為基礎的路由規則, 您必須為會議啟用位置路由:

- 從會議進行 PSTN 撥出

- 從對等音訊交談升級至涉及 PSTN 端點的會議

- 涉及 PSTN 端點的顧問式轉移

若要啟用會議的位置路由, 請參閱[會議的位置式](https://technet.microsoft.com/library/e1acb1ba-0ed2-4abf-8a7b-1ca3049e95e3.aspx)路由。
